# 은행 해킹대응 솔루션 PoC 결과 보고서 요약

## 0️⃣ 제안 요약

이번 은행 PoC를 통해, 당초 설정한 3개 목표가 모두 충족되었으며, 웹 패킷 분석 기반의 추가 가시성이 기존 침입차단시스템(IPS) 중심 운영을 효과적으로 보완할 수 있음을 확인하였습니다.

특히 PoC 결과 보고가 진행되던 중 실제 웹 공격이 발생하였고, 이 과정에서 기존 운영 체계의 대응 한계와 함께, PLURA의 추가 분석 가시성과 운영 판단 지원 가치가 현장에서 확인되었습니다.

이번 PoC를 통해 확인된 핵심은 다음 4가지입니다.

**1. PoC 목표 3개 항목이 모두 완료되었습니다.**  
웹 패킷 분석 기반 IPS 보완 가능성, 크리덴셜 스터핑 공격 탐지, APT 공격 진행 상황의 시각화 제공이라는 주요 검증 목표가 모두 충족되었습니다.

**2. 기존 차단 중심 보안 체계를 보완할 수 있는 추가 가시성이 확인되었습니다.**  
기존 체계는 차단 여부 중심의 정보를 제공하는 데 강점이 있었고, PLURA는 전체 로그, 탐지 로그, 사후 탐지 로그까지 포함한 보다 확장된 분석 정보를 제공하였습니다.

**3. 실제 공격 상황에서 운영 판단 지원 가치가 확인되었습니다.**  
PoC 결과 보고 당일 실제 웹 공격이 탐지되었으며, 이를 통해 단순 탐지를 넘어 공격 흐름 이해와 대응 우선순위 판단 지원의 중요성이 다시 확인되었습니다.

**4. 금융권 환경에서 웹 구간 가시성 확보의 필요성이 확인되었습니다.**  
금융 서비스 환경에서는 단순 차단만으로 충분하지 않으며, 공격 행위와 흐름을 더 구체적으로 파악할 수 있는 분석 기반이 필요함을 보여주었습니다.

## Executive Summary

이번 PoC는 단순한 기능 시연이 아니라, 은행 웹 서비스 환경에서 해킹 대응 솔루션이 실제로 어떤 보완 효과를 제공할 수 있는지를 검증한 과정이었습니다.

PoC 결과, 웹 패킷 분석을 통한 기존 IPS 보완 가능성이 확인되었고, 크리덴셜 스터핑 공격 탐지와 APT 공격 진행 상황 시각화 또한 유의미한 수준으로 검증되었습니다. 또한 결과 보고 당일 실제 웹 공격이 발생하면서, 기존 체계와 비교한 추가 가시성 및 운영 판단 지원 측면에서 PLURA의 차별성이 보다 명확히 드러났습니다.

이는 금융권과 같이 안정성, 신속한 대응 판단, 운영 가시성이 중요한 환경에서, 단순 차단 중심 체계를 넘어 보다 정교한 분석 기반 보안 운영이 필요함을 보여주는 결과입니다.

## 2️⃣ PoC 추진 배경과 목적

PoC 킥오프 미팅에서 은행 측은, 정보보안 관련 다단계 시스템을 구축·운영하고 있음에도 외부 공격에 대한 지속적인 위협이 존재하며, 이에 대한 상시적 경계와 대비가 필요하다는 점을 강조하였습니다.

이러한 배경에서 이번 PoC는 인터넷 및 모바일 뱅킹 구간의 패킷을 분석하여 해킹 공격을 탐지하고, 이를 통해 기존 침입차단시스템(IPS)을 보완할 수 있는지를 확인하는 데 목적을 두고 진행되었습니다.

즉, 이번 PoC는 단순한 제품 기능 검토가 아니라, 다음과 같은 운영 질문에 답하는 검증 과정이었습니다.

- 기존 보안 체계를 보완할 수 있는가
- 자동화 공격과 지능형 공격을 보다 명확하게 식별할 수 있는가
- 보안 운영자가 공격의 진행 흐름을 더 빠르고 직관적으로 파악할 수 있는가

특히 금융권 환경에서는 보안 시스템이 단순히 차단 기능만 수행하는 것이 아니라, 공격의 실체와 흐름을 보다 명확하게 보여주어 운영 판단을 지원하는 역할까지 수행할 수 있어야 합니다.

## 3️⃣ 1차 결과 보고 상세

### 3-1) 웹 패킷 분석 기반 IPS 보완 가능성 확인

웹 패킷 분석을 통해 공격 탐지 및 상세 내역 제공이 가능함을 확인하였고, 이를 통해 기존 침입차단시스템(IPS)의 보완재로서 타당성을 검증하였습니다.

이는 차단 여부 중심의 정보만으로는 확보하기 어려운 추가 분석 가시성을 제공할 수 있음을 의미합니다.

### 3-2) 크리덴셜 스터핑 공격 탐지 가능성 확인

모의 해킹 크리덴셜 스터핑 공격에 대해 공격자 IP 주소 등을 식별함으로써, 자동화 공격 탐지 가능성을 입증하였습니다.

이는 단순 이상 트래픽 인지 수준을 넘어, 실제 공격 행위를 보다 구체적으로 식별할 수 있는 기반이 있음을 보여줍니다.

### 3-3) 공격 진행 상황의 시각화 제공 가능성 확인

APT 공격 진행 상황을 MITRE ATT&CK 매트릭스 형태로 표시하여, 이상징후 탐지와 공격 흐름 파악이 가능한 대시보드 제공 가능성을 확인하였습니다.

이는 보안 운영자 입장에서 단일 이벤트 중심이 아니라, 공격의 전체 맥락을 보다 입체적으로 이해할 수 있게 한다는 점에서 의미가 있습니다.

## 4️⃣ 실제 공격 탐지와 운영상 시사점

PoC 결과 보고가 진행되던 2023년 11월 14일 오후, PoC 대상 구간에서 실제 웹 공격이 탐지되었습니다.

공격은 주로 SQL 인젝션 유형으로 파악되었으며, 심각성이 즉시 공유되고 대응 요청이 이루어졌습니다. 다만 실제 차단은 오후 4시 53분경 이루어졌습니다.

이 사례는 다음과 같은 운영상 시사점을 보여줍니다.

### 4-1) 탐지와 차단은 항상 동시에 이루어지지 않습니다

실제 운영 환경에서는 탐지 이후에도 보고 체계, 승인 절차, 운영 의사결정이 개입되므로 차단까지 시간이 소요될 수 있습니다.

따라서 보안 운영에서는 단순 탐지 여부뿐 아니라, 공격의 지속성, 심각성, 흐름을 보다 명확하게 보여주는 추가 분석 정보가 중요합니다.

### 4-2) 운영 판단을 지원하는 가시성이 중요합니다

공격이 발생했을 때 보안 담당자는 “막았는가”뿐 아니라 “어떤 유형의 공격이 어느 정도로 진행되고 있는가”를 빠르게 이해해야 합니다.

이번 사례는 단순 차단 중심 정보만으로는 충분하지 않을 수 있으며, 추가적인 탐지 로그와 분석 정보가 운영 판단에 실질적인 도움이 될 수 있음을 보여주었습니다.

### 4-3) 금융권 보안 운영은 안정성과 신속성의 균형이 중요합니다

금융 서비스 환경에서는 공격 대응이 중요하지만, 동시에 운영 안정성도 함께 고려해야 합니다.

이러한 환경에서는 차단 기능과 더불어, 이상행위 탐지와 공격 흐름 가시성을 통해 운영자가 보다 정확하게 상황을 이해하고 대응 우선순위를 정할 수 있어야 합니다.

## 6️⃣ 이번 PoC를 통해 확인된 핵심 의미

이번 PoC를 통해 확인된 핵심 의미는 다음과 같습니다.

### 6-1) 웹 패킷 분석 기반 보안의 실효성 확인

웹 패킷 분석을 통해 기존 IPS 보완 가능성이 확인되었습니다.  
이는 웹 서비스 구간이 핵심인 금융권 환경에서 의미가 큽니다.

### 6-2) 자동화 공격 탐지 가능성 확인

크리덴셜 스터핑과 같은 자동화 공격에 대해, 공격자 IP 및 주요 행위를 식별할 수 있음을 확인하였습니다.

이는 사용자 인증 구간을 노리는 공격에 대해 보다 정교한 탐지 기반을 마련할 수 있음을 시사합니다.

### 6-3) 공격 흐름 가시화의 실무적 가치 확인

MITRE ATT&CK 기반 대시보드를 통해 APT 공격 진행 상황을 시각화할 수 있음을 확인하였습니다.

이는 보안 담당자가 단일 이벤트 수준을 넘어 전체 공격 흐름을 이해하고 보고할 수 있게 해 준다는 점에서 운영상 가치가 있습니다.

### 6-4) 실제 운영 환경에서의 한계와 보완 방향 확인

PoC 결과 보고 당일 실제 공격이 탐지된 사례는, 금융권 보안 운영에서 탐지 이후의 대응 판단과 운영 체계가 얼마나 중요한지를 보여주었습니다.

이 과정에서 보다 확장된 탐지·분석 정보의 필요성이 다시 확인되었습니다.

### 6-5) 금융권 보안 운영에 필요한 가시성 확보 방향 제시

이번 PoC는 금융권 보안 운영에서 요구되는 공격 가시성 확보, 이상행위 식별, 운영 판단 지원 체계 강화 측면에서 참고 가능한 의미를 가집니다.

## 7️⃣ 제안 방향

은행과 같은 금융권 환경에서는 기존 보안 장비의 차단 기능만으로 모든 공격 상황을 충분히 설명하기 어렵습니다.  
특히 웹 서비스 구간에서는 정상 트래픽과 유사한 공격, 자동화 공격, 지능형 공격 흐름 등에 대해 보다 정교한 탐지와 분석이 필요합니다.

이에 따라 향후 방향은 다음과 같이 정리할 수 있습니다.

### 7-1) 웹 구간 분석 가시성 강화

웹 요청과 응답 구간에 대한 충분한 분석 데이터를 확보하여, 단순 차단 여부를 넘어 공격의 실제 흐름을 파악할 수 있어야 합니다.

### 7-2) 탐지와 차단 사이의 운영 연결 강화

공격 탐지 이후 실제 차단과 대응 판단으로 이어지는 운영 절차를 보다 신속하고 명확하게 정리할 필요가 있습니다.

### 7-3) 대시보드 기반 상황 인식 체계 강화

APT 및 이상행위의 진행 상황을 시각적으로 제공함으로써, 보안 운영자가 공격의 전체 흐름을 보다 직관적으로 파악할 수 있어야 합니다.

### 7-4) 기존 보안 체계 보완형 도입 검토

PLURA는 기존 보안 체계를 대체하기보다, 기존 체계가 제공하지 못하는 추가 가시성과 분석 기능을 보완하는 방향으로 검토할 수 있습니다.

이는 금융권 환경에서 보다 현실적인 도입 방향이 될 수 있습니다.

이번 PoC는 단순한 기능 검증을 넘어, 실제 금융 서비스 환경에서 다음 사항을 확인한 의미 있는 사례였습니다.

- PLURA의 차별적 분석 가치와 운영 지원 가능성
