# PLURA-XDR 제안서
## NHN Cloud와 PLURA-XDR 공동 사업 및 통합 보안 경쟁력 강화 제안

**미토스 기반 AI 신위협 대응**  
> “앤트로픽의 Mythos Preview 사례가 보여주듯, AI가 제로데이 취약점 탐지와 공격 자동화 속도를 높이는 시대가 시작되고 있습니다. 이런 환경에서는 시그니처 중심의 개별 보안 도구만으로는 한계가 있습니다. PLURA-XDR은 **요청·응답 본문 정밀 분석 + 웹-서버 XDR 상관분석 + 자동 대응**을 통해, 알려지지 않은 공격까지 더 빠르게 식별하고 대응할 수 있는 구조를 제공합니다.”

---

## 0️⃣ 제안 요약

NHN Cloud는 강력한 IaaS 사업 기반 위에 다양한 정보보안 서비스를 함께 제공하고 있습니다.

당사의 제안은 NHN Cloud가 단순히 PLURA-XDR을 내부에서 사용하라는 의미가 아닙니다.  
더 큰 목적은 **NHN Cloud의 IaaS와 PLURA-XDR을 결합하여, NHN Cloud 기존 고객사와 외부 신규 잠재 고객사에 함께 제안할 수 있는 경쟁력 있는 사이버보안 상품 구조를 만들자**는 것입니다.

즉, 이 제안의 핵심은 “도입”보다 **공동 사업**입니다.

NHN Cloud는 IaaS 판매 경쟁력을 높일 수 있고,  
PLURA-XDR은 NHN Cloud 기반으로 상품과 고객 저변을 확대할 수 있습니다.  
고객 입장에서도 인프라와 보안을 함께 이해하기 쉬운 형태로 도입할 수 있습니다.

이 구조는 NHN Cloud와 PLURA-XDR 모두에게 이익이 되는 상호 확장 모델입니다.

- NHN Cloud는 IaaS + 보안 결합 상품으로 경쟁력을 높일 수 있음
- PLURA-XDR은 NHN Cloud 기반으로 상품과 고객사를 지속 확대할 수 있음
- 고객은 인프라와 보안을 함께 검토할 수 있어 도입 구조가 단순해짐
- 양측 모두 중장기적으로 더 강한 사업 구조를 만들 수 있음

그리고 이 공동 사업 구조가 정착되면,  
그 다음 단계로 NHN Cloud 내부 보안 체계에도 PLURA-XDR을 적용하여  
이처럼 위험한 시기에 더 강한 보안 운영 체계를 구축할 수 있습니다.

### PLURA-XDR의 핵심 차별점

PLURA-XDR은 단순한 웹방화벽이나 EDR이 아닙니다.  
다음 기능을 **하나의 운영 체계**로 제공합니다.

- **제로데이 공격 탐지 기술**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **웹과 서버를 연결한 상관분석**
- **자동 대응과 포렌식까지 이어지는 운영 구조**

그리고 이러한 탐지·분석·대응의 기준은  
단순 기능 나열이 아니라 **MITRE ATT&CK 기반 위협 행위 체계**로 정리됩니다.

즉, PLURA-XDR은

> **MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, 랜섬웨어, APT까지 하나의 흐름으로 대응하는 통합 정보보안 플랫폼**입니다.

### 제안 핵심 수치 예시

아래 수치는 제안 단계의 **예시 KPI**입니다.

| 항목 | 목표 |
| --- | --- |
| 평균 탐지 시간 | 기존 대비 **50% 이상 단축** |
| 평균 초동 대응 시간 | 기존 대비 **40% 이상 단축** |
| 웹-서버 연계 분석 시간 | 기존 수동 분석 대비 **70% 이상 절감** |
| 콘솔 전환 횟수 | 사고 1건당 **3개 이상 → 1개 수준** |
| 크리덴셜 스터핑 탐지율 | 테스트 시나리오 기준 **90% 이상** |
| 포렌식 증적 확보율 | 테스트 이벤트 기준 **100% 수집** |

※ 실제 수치는 고객 환경, 적용 범위, 테스트 시나리오에 따라 PoC 착수 시 공동 조정합니다.

## 1️⃣ 왜 이 제안이 중요한가: NHN Cloud의 IaaS 경쟁력을 보안 상품으로 확장할 수 있습니다

NHN Cloud는 IaaS 사업자입니다.  
즉, 핵심은 인프라를 제공하고, 그 위에 다양한 서비스와 상품을 결합해 고객 가치를 높이는 것입니다.

보안도 같은 관점에서 봐야 합니다.

현재 NHN Cloud는 여러 보안 기능을 제공하고 있지만,  
고객 입장에서는 이를 각각 이해하고 선택해야 할 수 있습니다.  
이 과정에서 다음과 같은 부담이 생깁니다.

- 어떤 보안 기능을 선택해야 하는지 판단이 어렵다
- 여러 기능이 서로 어떻게 연결되는지 이해하기 어렵다
- 운영 화면과 대응 체계가 분산될 수 있다
- 기능이 늘수록 비용과 운영 부담이 함께 증가한다

즉, 고객이 원하는 것은  
“여러 기능의 목록”보다  
**실제로 이해하고 도입할 수 있는 통합 보안 상품**입니다.

이 지점에서 NHN Cloud와 PLURA-XDR의 결합은 큰 의미를 가집니다.

### 고객이 실제로 원하는 것

오늘날 공격은 단일 이벤트로 끝나지 않습니다.

- 웹 취약점 공격
- 크리덴셜 스터핑
- 권한 상승
- 내부 확산
- 데이터 유출
- 랜섬웨어
- APT

이처럼 공격은 하나의 흐름으로 이어집니다.  
따라서 고객이 원하는 것은 개별 기능이 아니라  
**공격 흐름 전체를 하나로 보고 대응할 수 있는 운영 체계**입니다.

즉,

> 고객은 “어떤 보안 기능이 있나”보다  
> “실제 해킹에 어떻게 대응할 수 있나”를 더 중요하게 봅니다.

### NHN Cloud에 주는 의미

이 제안은 NHN Cloud의 보안 기능을 비판하는 제안이 아닙니다.  
오히려 NHN Cloud가 이미 보유한 IaaS 경쟁력을 기반으로,  
**더 강한 보안 상품 경쟁력을 만들 수 있는 공동 제안**입니다.

경영진 관점에서 보면 핵심은 두 가지입니다.

**1) 고객 제안력이 더 강해집니다.**  
IaaS와 보안을 따로 설명하는 것이 아니라,  
함께 제안 가능한 통합 상품으로 만들 수 있습니다.

**2) 수익 구조가 확장됩니다.**  
NHN Cloud의 IaaS 판매가 늘수록 PLURA-XDR의 적용 기회도 늘어나고,  
PLURA-XDR의 경쟁력이 커질수록 NHN Cloud의 IaaS 제안도 강해집니다.

즉,

> **NHN Cloud와 PLURA-XDR은 함께 팔수록 더 강해지는 구조**입니다.

## 2️⃣ 공동 사업 관점에서의 제안: NHN Cloud + PLURA-XDR 결합 상품

이번 제안의 본질은  
PLURA-XDR을 단순 솔루션으로 도입하자는 것이 아니라,  
**NHN Cloud의 IaaS와 결합한 경쟁력 있는 사이버보안 상품**으로 함께 시장에 제안하자는 것입니다.

### 2-1) 제안 방향

다음 두 축으로 정리할 수 있습니다.

**1. 외부 고객 대상 공동 사업**
- NHN Cloud 기존 고객사 대상 공동 제안
- 외부 신규 잠재 고객사 대상 공동 영업
- IaaS + 통합 보안 구조로 제안 단순화
- 고객이 이해하기 쉬운 상품 구조 확보

**2. 내부 보안 고도화로의 확장**
- 외부 고객 대상 공동 상품 구조 정착
- 이후 NHN Cloud 내부 보안 체계에도 적용 확대
- NHN Cloud 자체 보안 운영 경쟁력 강화

즉, 순서는 명확합니다.

> **1차는 외부 고객 대상 공동 판매 구조**이고,  
> **2차는 NHN Cloud 내부 보안 적용 확대**입니다.

### 2-2) 왜 이 구조가 경쟁력이 있는가

고객은 여러 보안 기능을 개별적으로 선택할수록  
선택 부담과 비용 부담을 동시에 느끼게 됩니다.

반면 NHN Cloud의 IaaS와 PLURA-XDR을 함께 제안하면,  
고객은 이를 **기술 조합**이 아니라 **하나의 사용 가능한 상품**으로 이해할 수 있습니다.

즉, 고객은 아래처럼 판단하게 됩니다.

| 구분 | 기존 다중 보안 서비스 선택 구조 | NHN Cloud + PLURA-XDR |
| --- | --- | --- |
| 도입 방식 | 기능별 개별 선택 | 통합 상품 관점 |
| 운영 화면 | 다중 콘솔 | 통합 흐름 중심 |
| 분석 방식 | 운영자 수동 연계 | 웹-서버 상관분석 |
| 도입 판단 | 기능별 선택 부담 큼 | 제품 단위로 이해 쉬움 |
| 비용 설명 | 항목별 누적 | 단순한 제품 비용 구조 |

즉,

> **NHN Cloud + PLURA-XDR은 고객에게 더 설명하기 쉽고, 더 팔기 쉬운 구조**입니다.

### 2-3) 상호 이익 구조

이 구조는 한쪽만 좋은 제안이 아닙니다.

**NHN Cloud 측면**
- IaaS 상품 경쟁력 강화
- 보안 결합 제안력 강화
- 신규 고객 제안 시 차별화 가능
- 기존 고객 업셀링 가능

**PLURA-XDR 측면**
- NHN Cloud 기반 고객 확대
- 상품 적용 범위 확대
- 지속적인 보안 상품 확장 가능
- 안정적인 공동 영업 기반 확보

즉,

> **NHN Cloud가 많이 팔리면 PLURA-XDR도 커지고,  
> PLURA-XDR이 커질수록 NHN Cloud의 IaaS 제안도 더 강해집니다.**

## 3️⃣ 왜 PLURA-XDR이 공동 상품으로 적합한가

PLURA-XDR은 단순 기능형 솔루션이 아니라  
**공격 흐름 전체를 하나로 연결하는 운영 체계**라는 점에서 공동 상품에 적합합니다.

### 3-1) 핵심 차별점

PLURA-XDR의 핵심 차별점은 다음과 같습니다.

- **제로데이 공격 탐지**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **웹-서버 상관분석**
- **자동 대응과 포렌식 연계**

즉, 단순히 알려진 공격을 막는 기능이 아니라,  
**공격 흐름 전체를 해석하고 운영하는 체계**입니다.

### 3-2) 모든 기준은 MITRE ATT&CK

PLURA-XDR의 중요한 차이는  
탐지 항목을 기능 중심이 아니라  
**MITRE ATT&CK 기준으로 해석하고 운영한다는 점**입니다.

이 기준에서 보면,

- 크리덴셜 스터핑은 **Initial Access / Credential Access**
- 랜섬웨어는 **Impact**
- APT는 **여러 ATT&CK 단계가 장기간 연결된 공격 흐름**

입니다.

즉,

> **PLURA-XDR은 MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, 데이터 유출, 랜섬웨어, APT까지 하나의 흐름으로 분석하고 대응합니다.**

### 3-3) 왜 기존 일반 WAF/EDR보다 강한가

PLURA-XDR은 아래 차이를 가집니다.

| 구분 | 일반 웹방화벽 | 일반 XDR/EDR | NHN 기존 서비스 조합 | PLURA-XDR |
| --- | --- | --- | --- | --- |
| 알려진 공격 탐지 | 가능 | 일부 가능 | 가능 | 가능 |
| 제로데이 공격 탐지 | 제한적 | 서버 행위 중심 | 제한적 | 웹+서버 연계 |
| 요청 본문 분석 | 제한적 | 없음 | 제한적 | 가능 |
| 응답 본문 분석 | 드묾 | 없음 | 제한적 | 가능 |
| 크리덴셜 스터핑 대응 | 제한적 | 제한적 | 분산 대응 | 세션 흐름 기반 |
| 랜섬웨어 대응 | 제한적 | 일부 가능 | 분산 대응 | 웹-서버 연계 가능 |
| APT 대응 | 제한적 | 일부 가능 | 분산 대응 | ATT&CK 기준 상관분석 |
| 자동 대응+포렌식 연계 | 부분적 | 부분적 | 부분적 | 통합 제공 |

즉,

> **PLURA-XDR은 알려진 공격뿐 아니라,  
> 계정 기반 공격, 랜섬웨어, APT, 데이터 유출까지 연결해 대응하는 통합 플랫폼**입니다.

### 3-4) 민감정보 탐지 차별성

민감정보탐지 기능을 말할 때 중요한 것은  
단순 키워드 매칭이 아니라 **응답 본문 분석 여부**입니다.

응답 본문 분석이 없다면 실질적인 데이터 유출 탐지에는 한계가 있습니다.

> 만약 제공되는 민감정보탐지 기능이 응답 본문 분석까지 포함한다면,  
> 이는 당사가 보유한 응답 본문 기반 보안 분석 특허와의 관계를 별도로 검토할 필요가 있습니다.

## 4️⃣ NHN Cloud 환경에서의 적용 구조

이번 제안은 NHN Cloud의 기존 보안 서비스를 부정하는 것이 아니라,  
기존 IaaS 및 보안 기능 위에 **PLURA-XDR을 통합 계층으로 결합**하는 구조입니다.

### 4-1) 적용 구조

```text
인터넷/외부 공격
   ↓
NHN Cloud IaaS / 서비스 구간
   ↓
PLURA-Web
 - 요청/응답 본문 분석
 - 제로데이 공격 탐지
 - 크리덴셜 스터핑 탐지
   ↓
PLURA-XDR
 - 웹 이벤트 상관분석
 - 인증/세션 이상행위 분석
 - 기존 보안 이벤트 연계
 - MITRE ATT&CK 기준 행위 분석
   ↓
PLURA-EDR
 - 프로세스/파일/네트워크 행위 탐지
 - 웹셸/확산 행위 탐지
 - 랜섬웨어/APT 후속 행위 탐지
   ↓
자동 대응 / 알림 / 포렌식
```

### 4-2) 적용 방식

PLURA-XDR은 다음 두 가지 방식으로 적용할 수 있습니다.

| 적용 방식                    | 설명                                                                                         |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| Co-exist 모델              | 기존 NHN Cloud Web Firewall, SIEM, Security Monitoring을 유지하면서 PLURA-XDR이 정밀 분석 및 상관분석 계층을 추가 |
| Replace / Consolidate 모델 | 고객 환경에 따라 일부 개별 기능을 PLURA-XDR로 통합해 운영 화면과 대응 체계를 단순화                                       |

즉, NHN Cloud와 PLURA-XDR은  
대체 관계만이 아니라 **공존과 단계적 통합 전환이 가능한 구조**입니다.

### 4-3) 로그 수집 및 연계 방식

PLURA-XDR은 다음 데이터를 중심으로 통합 분석합니다.

| 데이터 유형 | 수집/분석 대상                            | 방식       |
| ------ | ----------------------------------- | -------- |
| 웹 구간   | 요청 본문, 응답 본문, URL, 세션, 인증 이벤트       | 웹 구간 분석  |
| 서버 구간  | 프로세스, 파일, 네트워크, 계정 행위               | EDR 에이전트 |
| 운영 이벤트 | 정책 위반, 차단 이벤트, 알림, 포렌식 증적           | XDR 연계   |
| 연계 이벤트 | 기존 SIEM / Security Monitoring 상위 경보 | 이벤트 연동   |

이를 통해 NHN Cloud가 제공하는 상위 가시성에 더해,  
PLURA-XDR은 **실제 공격의 세부 흐름**을 분석하는 계층 역할을 수행합니다.

### 4-4) 서버 및 Kubernetes 배포 방식

PLURA-EDR은 NHN Cloud의 다음 환경에서도 단계적으로 적용 가능합니다.

| 환경                     | 적용 방식          |
| ---------------------- | -------------- |
| VM 인스턴스                | 에이전트 직접 설치     |
| Baremetal 서버           | 에이전트 직접 설치     |
| Kubernetes Worker Node | 노드 단위 배포 및 운영  |
| Windows / Linux 혼합     | OS별 에이전트 정책 적용 |
| Hybrid 구성 서버           | 단계별 확대 적용      |

즉, NHN Cloud의 IaaS 환경과 결합하기에 현실적인 구조를 가집니다.

## 5️⃣ 어떻게 시작할 것인가: 작게 시작하고 함께 키웁니다

이 제안의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다.

핵심은 **작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 것**입니다.

### 5-1) 단계별 적용 로드맵

| 단계  | 기간(예시) | 주요 내용       | 성공 기준                         |
| --- | ------ | ----------- | ----------------------------- |
| 1단계 | 2주     | 웹 구간 파일럿 적용 | 요청/응답 분석, 제로데이·크리덴셜 스터핑 탐지 확인 |
| 2단계 | 4주     | EDR 확대 적용   | 핵심 서버 배포, 웹-서버 상관분석 활성화       |
| 3단계 | 2주     | 자동 대응 정책 적용 | 알림→차단→격리 정책 단계적 적용            |
| 4단계 | 4주     | 운영 정착 및 확산  | 운영팀/SOC 단독 활용, 확대 여부 결정       |

### 5-2) 초기 적용 범위

* 인터넷 노출 웹 서비스
* 핵심 API 서버
* 인증/로그인 구간
* 민감정보 조회 가능 구간
* 관리자 기능 포함 구간

### 5-3) PoC 시작 방식 제안

초기 PoC는 아래와 같이 시작하는 것이 현실적입니다.

1. 인터넷 노출 웹 서비스 1개 선정
2. 핵심 API 또는 로그인 구간 1개 선정
3. 관련 서버 2~5대 수준 EDR 적용
4. 크리덴셜 스터핑 및 웹-서버 연계 시나리오 검증
5. 랜섬웨어/APT 전 단계 행위 검증
6. 운영 콘솔, 알림, 포렌식 확보 흐름 확인

즉, 처음부터 전체 전환이 아니라
**작게 시작해 빠르게 검증하고 단계적으로 확대하는 방식**이 적합합니다.

## 6️⃣ 비용 모델과 공동 상품 관점의 의미

PLURA-XDR의 가격 모델은 고객이 이해하기 쉬운 형태로 제안하는 것이 중요합니다.

### 6-1) 기본 가격 구조 예시

* 웹 서비스 수 또는 처리 구간 기준
* 서버 대수 기준 EDR 과금
* 통합 XDR 분석 기능 포함
* 초기 구축/온보딩 비용 별도 또는 프로모션 포함

### 6-2) 고객 관점의 비용 메시지

고객은 여러 보안 서비스를 각각 선택할수록  
기능별 과금, 운영 인력, 분석 시간, 교육 비용까지 함께 부담하게 됩니다.

반면 PLURA-XDR은

* 웹 구간 분석
* 서버 행위 분석
* 통합 XDR 상관분석
* 자동 대응
* 포렌식
* 랜섬웨어/APT 대응 관점의 통합 운영

을 하나의 체계로 제공하므로,  
고객은 이를 **복수 서비스의 합산 비용**이 아니라  
**바로 사용할 수 있는 통합 정보보안 제품 비용**으로 이해할 수 있습니다.

### 6-3) 비용 절감 표현 예시

제안 단계에서는 아래처럼 설명할 수 있습니다.

> 고객 환경에 따라 차이가 있으나, 개별 보안 서비스 다중 선택 구조 대비  
> **도입 항목 단순화와 운영 인력 절감을 통해 총비용 기준 30~50% 수준의 효율 개선 가능성**을 기대할 수 있습니다.

※ 위 수치는 제안용 예시 범위이며, 실제 절감률은 대상 범위와 기존 운영 구조에 따라 별도 산정합니다.

즉, NHN Cloud는  
**IaaS + 통합 보안**을 하나의 경쟁력 있는 상품 구조로 제안할 수 있게 됩니다.

## 7️⃣ 2차 확장: NHN Cloud 내부 보안에도 적용 가능

이 제안의 1차 목적은  
외부 고객 대상 **공동 상품 및 공동 판매 구조**입니다.

그러나 그 구조가 정착되면,  
NHN Cloud 내부 보안 체계에도 PLURA-XDR을 적용할 수 있습니다.

이런 시기에는 NHN Cloud 자체도 다음이 필요합니다.

* 제로데이 공격 대응
* 랜섬웨어 대응
* APT 공격 대응
* MITRE ATT&CK 기준 위협 운영
* 웹과 서버를 함께 보는 통합 가시성
* 자동 대응과 포렌식 체계

즉,

> 외부 고객에게 경쟁력 있는 구조라면,  
> NHN Cloud 내부 보안에도 동일한 가치가 적용될 수 있습니다.

이것이 2차 확장 포인트입니다.

이 제안의 본질은  
NHN Cloud가 PLURA-XDR을 단순히 “사용하라”는 것이 아닙니다.

핵심은

* **NHN Cloud의 IaaS와 PLURA-XDR을 결합한 공동 사업 구조**
* **기존 고객과 외부 신규 고객에게 함께 판매할 수 있는 경쟁력 있는 보안 상품**
* **NHN Cloud가 많이 팔릴수록 PLURA-XDR도 확장되고, PLURA-XDR이 강해질수록 NHN Cloud IaaS 제안도 강해지는 상호 확장 모델**
* **그 연장선에서 NHN Cloud 내부 보안 고도화 가능**

에 있습니다.

즉,

> **NHN Cloud와 PLURA-XDR은 함께 팔수록 더 강해지는 관계**입니다.

이것이 본 제안의 핵심입니다.

* NHN Cloud는 IaaS 사업 기반 위에 다양한 보안 기능을 제공하고 있습니다.
* 이번 제안의 핵심은 NHN Cloud가 PLURA-XDR을 단순히 도입하라는 것이 아니라, **NHN Cloud의 IaaS와 PLURA-XDR을 결합해 함께 팔자**는 것입니다.
* 고객은 여러 기능의 목록보다, 실제로 이해하고 도입할 수 있는 통합 보안 상품을 원합니다.

* 이는 단순 도입 제안이 아니라 **공동 성장 제안**입니다.
