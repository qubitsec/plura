# PLURA-XDR 제안서
## NHN Cloud와 PLURA-XDR 공동 사업 및 보안 고도화 제안

**미토스 기반 AI 신위협 대응**  
> “앤트로픽의 Mythos Preview 사례가 보여주듯, AI가 제로데이 취약점 탐지와 공격 자동화 속도를 높이는 시대가 시작되고 있습니다. 이런 환경에서는 시그니처 중심의 개별 보안 도구만으로는 한계가 있습니다. PLURA-XDR은 **요청·응답 본문 정밀 분석 + 웹-서버 XDR 상관분석 + 자동 대응**을 통해, 알려지지 않은 공격까지 더 빠르게 식별하고 대응할 수 있는 구조를 제공합니다.”

---

## 0️⃣ 제안 요약

NHN Cloud는 App Security Check, Server Security Check, Web Firewall, DDoS Guard, SIEM, Bastion, Vaccine, Security Monitoring 등 다양한 정보보안 서비스를 제공하고 있습니다.

본 제안은 크게 두 가지입니다.

**제안 1**은 NHN Cloud와 PLURA-XDR을 결합하여,  
NHN Cloud 기존 고객사와 외부 신규 잠재 고객사에 함께 제안할 수 있는  
경쟁력 있는 통합 보안 상품 구조를 만드는 것입니다.

**제안 2**는 이러한 구조를 바탕으로,  
NHN Cloud 내부 보안 체계에도 PLURA-XDR을 적용하여  
제로데이, 크리덴셜 스터핑, 랜섬웨어, APT 공격까지 더 강하게 대응할 수 있는  
보안 운영 체계를 구축하는 것입니다.

즉, 이 제안의 핵심은  
단순히 새로운 보안 기능을 추가하자는 것이 아니라,  
**외부 고객에게 더 강한 상품 경쟁력을 만들고, 그 연장선에서 내부 보안 체계도 고도화하자**는 것입니다.

### PLURA-XDR의 핵심 차별점

PLURA-XDR은 단순한 웹방화벽이나 EDR이 아닙니다.  
다음 기능을 **하나의 운영 체계**로 제공합니다.

- **제로데이 공격 탐지 기술**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **웹과 서버를 연결한 상관분석**
- **자동 대응과 포렌식까지 이어지는 운영 구조**

특히 최근과 같이 AI가 제로데이 취약점 탐지와 공격 자동화 속도를 높이는 환경에서는,  
단순 시그니처 기반 방어만으로는 한계가 있습니다.  
PLURA-XDR은 **요청·응답 본문 정밀 분석 + 웹-서버 XDR 상관분석 + 자동 대응**을 통해,  
이러한 신종 위협까지 더 빠르게 식별하고 대응할 수 있는 경쟁력을 제공합니다.

그리고 이러한 탐지·분석·대응의 기준은  
단순 기능 나열이 아니라 **MITRE ATT&CK 기반 위협 행위 체계**로 정리됩니다.

즉, PLURA-XDR은

> **MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, 랜섬웨어, APT까지 하나의 흐름으로 대응하는 통합 정보보안 플랫폼**입니다.

## 1️⃣ 제안 1: NHN Cloud + PLURA-XDR 공동 사업 제안

### 1-1) 왜 공동 상품 구조가 필요한가

NHN Cloud는 클라우드 사업자로서 다양한 고객 환경과 보안 수요를 함께 상대하고 있습니다.

이때 중요한 것은 개별 보안 기능을 각각 설명하는 것이 아니라,  
고객이 실제로 이해하고 도입할 수 있는 **통합 상품 구조**를 제안하는 것입니다.

현재 NHN Cloud는 여러 보안 기능을 제공하고 있지만,  
고객 입장에서는 이를 각각 이해하고 선택해야 할 수 있습니다.  
이 과정에서 다음과 같은 부담이 생깁니다.

- 어떤 보안 기능을 선택해야 하는지 판단이 어렵다
- 여러 기능이 서로 어떻게 연결되는지 이해하기 어렵다
- 운영 화면과 대응 체계가 분산될 수 있다
- 기능이 늘수록 비용과 운영 부담이 함께 증가한다

즉, 고객이 원하는 것은  
“여러 기능의 목록”보다  
**실제로 이해하고 도입할 수 있는 통합 보안 상품**입니다.

### 1-2) 고객이 실제로 원하는 것은 기능 목록이 아니라 대응 체계입니다

오늘날 공격은 단일 이벤트로 끝나지 않습니다.

- 웹 취약점 공격
- 크리덴셜 스터핑
- 권한 상승
- 내부 확산
- 데이터 유출
- 랜섬웨어
- APT

이처럼 공격은 하나의 흐름으로 이어집니다.  
따라서 고객이 원하는 것은 개별 기능이 아니라  
**공격 흐름 전체를 하나로 보고 대응할 수 있는 운영 체계**입니다.

즉,

> 고객은 “어떤 보안 기능이 있나”보다  
> “실제 해킹에 어떻게 대응할 수 있나”를 더 중요하게 봅니다.

### 1-3) NHN Cloud와 PLURA-XDR의 결합이 주는 의미

이번 제안은 NHN Cloud의 기존 보안 기능을 부정하는 제안이 아닙니다.  
오히려 NHN Cloud가 이미 제공하고 있는 클라우드 서비스와 보안 서비스 위에,  
**더 강한 상품 경쟁력을 만들 수 있는 공동 제안**입니다.

경영진 관점에서 보면 핵심은 두 가지입니다.

**1) 고객 제안력이 더 강해집니다.**  
클라우드와 보안을 따로 설명하는 것이 아니라,  
함께 제안 가능한 통합 상품으로 만들 수 있습니다.

**2) 수익 구조가 확장됩니다.**  
NHN Cloud의 고객 기반이 확대될수록 PLURA-XDR의 적용 기회도 늘어나고,  
PLURA-XDR의 경쟁력이 커질수록 NHN Cloud의 클라우드 제안도 강해집니다.

즉,

> **NHN Cloud와 PLURA-XDR은 함께 팔수록 더 강해지는 구조**입니다.

### 1-4) 왜 PLURA-XDR이 공동 상품으로 적합한가

PLURA-XDR은 단순 기능형 솔루션이 아니라  
**공격 흐름 전체를 하나로 연결하는 운영 체계**라는 점에서 공동 상품에 적합합니다.

핵심 차별점은 다음과 같습니다.

- **제로데이 공격 탐지**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **웹-서버 상관분석**
- **자동 대응과 포렌식 연계**

즉, 단순히 알려진 공격을 막는 기능이 아니라,  
**공격 흐름 전체를 해석하고 운영하는 체계**입니다.

### 1-5) 모든 기준은 MITRE ATT&CK입니다

PLURA-XDR의 중요한 차이는  
탐지 항목을 기능 중심이 아니라  
**MITRE ATT&CK 기준으로 해석하고 운영한다는 점**입니다.

이 기준에서 보면,

- 크리덴셜 스터핑은 **Initial Access / Credential Access**
- 랜섬웨어는 **Impact**
- APT는 **여러 ATT&CK 단계가 장기간 연결된 공격 흐름**

입니다.

즉,

> **PLURA-XDR은 MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, 데이터 유출, 랜섬웨어, APT까지 하나의 흐름으로 분석하고 대응합니다.**

### 1-6) 미토스 기반 AI 신위협 대응: 외부 고객 제안 시 가장 강한 차별점

최근에는 앤트로픽의 Mythos Preview 사례가 보여주듯, AI가 제로데이 취약점을 더 빠르게 찾아내고 공격 가능성을 높이는 방향으로 발전하고 있습니다.  
이는 단순히 새로운 공격 도구가 하나 더 생긴 것이 아니라, **공격의 속도와 변형 능력 자체가 달라지고 있다**는 의미입니다.

이런 환경에서는 기존 시그니처 중심의 웹방화벽이나 개별 보안 도구만으로는 대응에 한계가 있습니다.  
알려진 패턴을 탐지하는 방식은 이미 공개된 공격이나 반복되는 변형에는 유효할 수 있지만,  
AI가 만들어내는 새로운 우회 방식, 비정형 요청, 예상하지 못한 응답 흐름까지 빠르게 따라가기는 어렵습니다.

특히 중요한 점은 공격이 더 이상 한 구간에서 끝나지 않는다는 것입니다.  
AI 기반 신위협은 웹 요청 단계에서 시작해, 인증 우회, 계정 탈취, 내부 확산, 데이터 수집, 랜섬웨어 또는 APT형 후속 행위까지 연결될 수 있습니다.  
즉, 단일 룰이나 단일 장비가 아니라 **요청·응답·세션·서버 행위 전체를 하나의 흐름으로 연결해 보는 체계**가 필요합니다.

PLURA-XDR은 바로 이 지점에서 경쟁력을 가집니다.

- **요청·응답 본문 기반 정밀 분석**으로 알려지지 않은 공격 징후를 더 빠르게 식별할 수 있고
- **웹-서버 XDR 상관분석**으로 공격이 실제 서버 행위로 이어졌는지 추적할 수 있으며
- **자동 대응과 포렌식 연계**를 통해 탐지 이후 조치와 증적 확보까지 즉시 이어갈 수 있습니다.

또한 이러한 탐지·분석·대응은 MITRE ATT&CK 기준으로 정리될 수 있기 때문에,  
단순히 “이상하다”는 수준이 아니라 **어떤 공격 단계에서 어떤 행위가 발생했는지**를 체계적으로 설명할 수 있습니다.

결국 미토스와 같은 AI 기반 신위협 시대에는  
보안의 기준이 “얼마나 많은 기능이 있는가”가 아니라  
**얼마나 빠르게 새로운 공격을 식별하고, 실제 침해 흐름으로 연결해 대응할 수 있는가**로 바뀌고 있습니다.

이 점에서 PLURA-XDR은 NHN Cloud가 외부 고객에게 제안할 때,  
단순한 보안 기능 추가가 아니라 **미토스급 AI 신위협까지 대응 가능한 차세대 통합 보안 상품**이라는 점을 분명히 보여 줄 수 있습니다.

### 1-7) 공동 상품 구조의 경쟁력

고객은 여러 보안 기능을 개별적으로 선택할수록  
선택 부담과 비용 부담을 동시에 느끼게 됩니다.

반면 NHN Cloud의 클라우드 서비스와 PLURA-XDR을 함께 제안하면,  
고객은 이를 **기술 조합**이 아니라 **하나의 사용 가능한 상품**으로 이해할 수 있습니다.

| 구분 | 기존 다중 보안 서비스 선택 구조 | NHN Cloud + PLURA-XDR |
| --- | --- | --- |
| 도입 방식 | 기능별 개별 선택 | 통합 상품 관점 |
| 운영 화면 | 다중 콘솔 | 통합 흐름 중심 |
| 분석 방식 | 운영자 수동 연계 | 웹-서버 상관분석 |
| 도입 판단 | 기능별 선택 부담 큼 | 제품 단위로 이해 쉬움 |
| 비용 설명 | 항목별 누적 | 단순한 제품 비용 구조 |

즉,

> **NHN Cloud + PLURA-XDR은 고객에게 더 설명하기 쉽고, 더 팔기 쉬운 구조**입니다.

### 1-8) 비용 구조 측면의 의미

고객은 여러 보안 서비스를 각각 선택할수록  
기능별 과금, 운영 인력, 분석 시간, 교육 비용까지 함께 부담하게 됩니다.

반면 PLURA-XDR은

- 웹 구간 분석
- 서버 행위 분석
- 통합 XDR 상관분석
- 자동 대응
- 포렌식
- 랜섬웨어/APT 대응 관점의 통합 운영

을 하나의 체계로 제공하므로,  
고객은 이를 **복수 서비스의 합산 비용**이 아니라  
**바로 사용할 수 있는 통합 정보보안 제품 비용**으로 이해할 수 있습니다.

제안 단계에서는 아래처럼 설명할 수 있습니다.

> 고객 환경에 따라 차이가 있으나, 개별 보안 서비스 다중 선택 구조 대비  
> **도입 항목 단순화와 운영 인력 절감을 통해 총비용 기준 30~50% 수준의 효율 개선 가능성**을 기대할 수 있습니다.

※ 위 수치는 제안용 예시 범위이며, 실제 절감률은 대상 범위와 기존 운영 구조에 따라 별도 산정합니다.

## 2️⃣ 제안 2: NHN Cloud 내부 보안 고도화 제안

### 2-1) 왜 지금 내부 보안 고도화가 중요한가

외부 고객에게 경쟁력 있는 통합 보안 구조라면,  
그 가치는 NHN Cloud 내부 보안 체계에도 그대로 적용될 수 있습니다.

지금과 같은 위협 환경에서는 단순한 기능 중심 보안이 아니라,  
제로데이 공격, 크리덴셜 스터핑, 랜섬웨어, APT 공격까지 하나의 흐름으로 보고 대응할 수 있는 운영 체계가 필요합니다.

즉, NHN Cloud 자체도 다음이 필요합니다.

- 제로데이 공격 대응
- 크리덴셜 스터핑 대응
- 랜섬웨어 대응
- APT 공격 대응
- MITRE ATT&CK 기준 위협 운영
- 웹과 서버를 함께 보는 통합 가시성
- 자동 대응과 포렌식 체계

또한 이와 같은 AI 기반 신종 위협은 외부 고객 환경뿐 아니라 NHN Cloud 자체 운영 환경에도 동일하게 적용되는 문제입니다.  
따라서 외부 고객에게 경쟁력 있는 대응 체계를 제안할 수 있다면,  
그 구조는 NHN Cloud 내부 보안 고도화에도 그대로 활용될 수 있습니다.

### 2-2) 왜 기존 일반 WAF/EDR보다 강한가

PLURA-XDR은 아래 차이를 가집니다.

| 구분 | 일반 웹방화벽 | 일반 XDR/EDR | NHN 기존 서비스 조합 | PLURA-XDR |
| --- | --- | --- | --- | --- |
| 알려진 공격 탐지 | 가능 | 일부 가능 | 가능 | 가능 |
| 제로데이 공격 탐지 | 제한적 | 서버 행위 중심 | 제한적 | 웹+서버 연계 |
| 요청 본문 분석 | 제한적 | 없음 | 제한적 | 가능 |
| 응답 본문 분석 | 드묾 | 없음 | 제한적 | 가능 |
| 크리덴셜 스터핑 대응 | 제한적 | 제한적 | 분산 대응 | 세션 흐름 기반 |
| 랜섬웨어 대응 | 제한적 | 일부 가능 | 분산 대응 | 웹-서버 연계 가능 |
| APT 대응 | 제한적 | 일부 가능 | 분산 대응 | ATT&CK 기준 상관분석 |
| 자동 대응+포렌식 연계 | 부분적 | 부분적 | 부분적 | 통합 제공 |

즉,

> **PLURA-XDR은 알려진 공격뿐 아니라,  
> 계정 기반 공격, 랜섬웨어, APT, 데이터 유출까지 연결해 대응하는 통합 플랫폼**입니다.

### 2-3) 민감정보 탐지 차별성

민감정보탐지 기능을 말할 때 중요한 것은  
단순 키워드 매칭이 아니라 **응답 본문 분석 여부**입니다.

응답 본문 분석이 없다면 실질적인 데이터 유출 탐지에는 한계가 있습니다.

> 만약 제공되는 민감정보탐지 기능이 응답 본문 분석까지 포함한다면,  
> 이는 당사가 보유한 응답 본문 기반 보안 분석 특허와의 관계를 별도로 검토할 필요가 있습니다.

### 2-4) NHN Cloud 환경에서의 기술 적용 방식

이번 제안은 NHN Cloud의 기존 보안 서비스를 부정하는 것이 아니라,  
기존 클라우드 서비스 및 보안 기능 위에 **PLURA-XDR을 통합 계층으로 결합**하는 구조입니다.

#### 2-4-1) 적용 구조

```text
인터넷/외부 공격
   ↓
NHN Cloud 서비스 구간
   ↓
PLURA-Web
 - 요청/응답 본문 분석
 - 제로데이 공격 탐지
 - 크리덴셜 스터핑 탐지
   ↓
PLURA-XDR
 - 웹 이벤트 상관분석
 - 인증/세션 이상행위 분석
 - 기존 보안 이벤트 연계
 - MITRE ATT&CK 기준 행위 분석
   ↓
PLURA-EDR
 - 프로세스/파일/네트워크 행위 탐지
 - 웹셸/확산 행위 탐지
 - 랜섬웨어/APT 후속 행위 탐지
   ↓
자동 대응 / 알림 / 포렌식
```

#### 2-4-2) 적용 방식

PLURA-XDR은 다음 두 가지 방식으로 적용할 수 있습니다.

| 적용 방식 | 설명 |
| --- | --- |
| Co-exist 모델 | 기존 NHN Cloud Web Firewall, SIEM, Security Monitoring을 유지하면서 PLURA-XDR이 정밀 분석 및 상관분석 계층을 추가 |
| Replace / Consolidate 모델 | 고객 환경에 따라 일부 개별 기능을 PLURA-XDR로 통합해 운영 화면과 대응 체계를 단순화 |

즉, NHN Cloud와 PLURA-XDR은  
대체 관계만이 아니라 **공존과 단계적 통합 전환이 가능한 구조**입니다.

#### 2-4-3) 로그 수집 및 연계 방식

PLURA-XDR은 다음 데이터를 중심으로 통합 분석합니다.

| 데이터 유형 | 수집/분석 대상 | 방식 |
| --- | --- | --- |
| 웹 구간 | 요청 본문, 응답 본문, URL, 세션, 인증 이벤트 | 웹 구간 분석 |
| 서버 구간 | 프로세스, 파일, 네트워크, 계정 행위 | EDR 에이전트 |
| 운영 이벤트 | 정책 위반, 차단 이벤트, 알림, 포렌식 증적 | XDR 연계 |
| 연계 이벤트 | 기존 SIEM / Security Monitoring 상위 경보 | 이벤트 연동 |

이를 통해 NHN Cloud가 제공하는 상위 가시성에 더해,  
PLURA-XDR은 **실제 공격의 세부 흐름**을 분석하는 계층 역할을 수행합니다.

#### 2-4-4) 콘솔과 운영 연계

운영 방식은 다음과 같이 제안할 수 있습니다.

- 1차 운영: PLURA-XDR 통합 콘솔에서 웹-서버 상관분석 수행
- 2차 연계: NHN Cloud 운영팀 또는 고객 SOC에 알림 전달
- 3차 대응: 정책 기반 자동 차단 또는 운영 승인 후 조치

즉, NHN Cloud 콘솔을 무조건 대체하는 것이 아니라  
**실제 대응을 위한 전문 분석 콘솔**로 PLURA-XDR을 위치시킬 수 있습니다.

#### 2-4-5) 서버 및 Kubernetes 배포 방식

PLURA-EDR은 NHN Cloud의 다음 환경에서도 단계적으로 적용 가능합니다.

| 환경 | 적용 방식 |
| --- | --- |
| VM 인스턴스 | 에이전트 직접 설치 |
| Baremetal 서버 | 에이전트 직접 설치 |
| Kubernetes Worker Node | 노드 단위 배포 및 운영 |
| Windows / Linux 혼합 | OS별 에이전트 정책 적용 |
| Hybrid 구성 서버 | 단계별 확대 적용 |

즉, NHN Cloud의 클라우드 환경과 결합하기에 현실적인 구조를 가집니다.

### 2-5) 단계적 적용 및 확장

이 제안의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다.

핵심은 **작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 것**입니다.

#### 2-5-1) 단계별 적용 로드맵

| 단계 | 기간(예시) | 주요 내용 | 성공 기준 |
| --- | --- | --- | --- |
| 1단계 | 2주 | 웹 구간 파일럿 적용 | 요청/응답 분석, 제로데이·크리덴셜 스터핑 탐지 확인 |
| 2단계 | 4주 | EDR 확대 적용 | 핵심 서버 배포, 웹-서버 상관분석 활성화 |
| 3단계 | 2주 | 자동 대응 정책 적용 | 알림→차단→격리 정책 단계적 적용 |
| 4단계 | 4주 | 운영 정착 및 확산 | 운영팀/SOC 단독 활용, 확대 여부 결정 |

#### 2-5-2) 초기 적용 범위

- 인터넷 노출 웹 서비스
- 핵심 API 서버
- 인증/로그인 구간
- 민감정보 조회 가능 구간
- 관리자 기능 포함 구간

#### 2-5-3) PoC 시작 방식 제안

초기 PoC는 아래와 같이 시작하는 것이 현실적입니다.

1. 인터넷 노출 웹 서비스 1개 선정
2. 핵심 API 또는 로그인 구간 1개 선정
3. 관련 서버 2~5대 수준 EDR 적용
4. 크리덴셜 스터핑 및 웹-서버 연계 시나리오 검증
5. 랜섬웨어/APT 전 단계 행위 검증
6. 운영 콘솔, 알림, 포렌식 확보 흐름 확인

즉, 처음부터 전체 전환이 아니라  
**작게 시작해 빠르게 검증하고 단계적으로 확대하는 방식**이 적합합니다.

## 결론

본 제안은 두 가지입니다.

**제안 1**은 NHN Cloud와 PLURA-XDR을 결합해  
외부 고객에게 더 경쟁력 있는 통합 보안 상품을 함께 제안하자는 것입니다.

**제안 2**는 이러한 구조를 바탕으로  
NHN Cloud 내부 보안 체계도 더 강한 대응 중심 구조로 고도화하자는 것입니다.

즉, 이 제안은 단순한 솔루션 도입 제안이 아니라,

- **NHN Cloud의 클라우드 서비스와 PLURA-XDR을 결합한 공동 사업 구조**
- **기존 고객과 외부 신규 고객에게 함께 판매할 수 있는 경쟁력 있는 보안 상품**
- **NHN Cloud의 고객 기반이 커질수록 PLURA-XDR도 확장되고, PLURA-XDR이 강해질수록 NHN Cloud의 클라우드 제안도 강해지는 상호 확장 모델**
- **그 연장선에서 NHN Cloud 내부 보안 고도화 가능**

을 함께 담은 구조적 제안입니다.

특히 NHN Cloud는 외부 고객에게 제안할 때도,  
PLURA-XDR을 통해 **AI 기반 신종 위협까지 대응 가능한 차별화된 통합 보안 상품**이라는 메시지를 분명히 제시할 수 있습니다.

즉,

> **NHN Cloud와 PLURA-XDR은 함께 팔수록 더 강해지는 관계**입니다.

이것이 본 제안의 핵심입니다.
