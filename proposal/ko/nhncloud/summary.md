# PLURA-XDR 제안 요약
## NHN Cloud 보안 서비스 통합 경쟁력 강화 방안

# 1. 왜 지금 필요한가

NHN Cloud는 Web Firewall, SIEM, Bastion, Security Monitoring 등 다양한 정보보안 서비스를 이미 제공하고 있습니다.

문제는 **보안 기능의 부재**가 아니라,  
이 기능들이 **각각 산재되어 있어 고객이 직접 선택·이해·조합해야 한다는 점**입니다.

그 결과 고객과 내부 운영 조직은 다음과 같은 부담을 안게 됩니다.

- 여러 보안 기능을 각각 이해하고 선택해야 함
- 운영 화면과 콘솔이 분산되어 있음
- 웹 공격, 계정 공격, 서버 침해를 하나의 흐름으로 보기 어려움
- 탐지 이후 대응과 포렌식까지 연결하기 어려움
- 기능이 늘어날수록 비용과 운영 부담도 함께 증가함

즉,

> **서비스는 많지만, 실제 침해 대응 체계로는 분산되어 있는 구조**입니다.

오늘날 공격은 단일 이벤트로 끝나지 않습니다.

- 웹 취약점 공격
- 크리덴셜 스터핑
- 권한 상승
- 내부 확산
- 데이터 유출
- 랜섬웨어
- APT

이처럼 공격은 하나의 흐름으로 이어집니다.  
따라서 고객이 원하는 것은 기능 목록이 아니라,  
**공격 흐름 전체를 하나로 보고 대응할 수 있는 통합 운영 체계**입니다.

### 경영진 관점 핵심 판단

현재 고객이 느끼는 부담은 크게 두 가지입니다.

**1) 선택 부담**  
무엇을 도입해야 하는지 복잡합니다.

**2) 비용 부담**  
여러 서비스를 따로 선택할수록 도입비와 운영비가 함께 증가합니다.

따라서 NHN Cloud 입장에서도 필요한 것은  
보안 기능을 더 늘리는 것이 아니라,  
**기존 기능을 통합된 가치로 재구성하는 것**입니다.

# 2. PLURA-XDR이 제공하는 차별적 가치

PLURA-XDR은 단순한 웹방화벽이나 EDR이 아닙니다.  
핵심은 **웹 + 서버 + XDR + 자동 대응 + 포렌식**을 하나의 흐름으로 연결하는 데 있습니다.

## 2-1. 무엇이 다른가

PLURA-XDR의 핵심 차별점은 다음과 같습니다.

- **제로데이 공격 탐지**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **웹과 서버를 연결한 상관분석**
- **자동 대응과 포렌식 연계**

즉, 단순히 “알려진 공격을 막는 기능”이 아니라,  
**공격 흐름 전체를 해석하고 운영하는 체계**입니다.

## 2-2. 모든 기준은 MITRE ATT&CK

PLURA-XDR의 중요한 차이는  
탐지 항목을 단순 기능 기준이 아니라,  
**MITRE ATT&CK 기준으로 해석하고 운영한다는 점**입니다.

이 관점에서 보면,

- 크리덴셜 스터핑은 단순 로그인 공격이 아니라  
  **Initial Access / Credential Access**
- 랜섬웨어는 단순 악성코드가 아니라  
  **Impact**
- APT는 단일 파일 문제가 아니라  
  **여러 ATT&CK 단계가 장기간 연결된 공격 흐름**

입니다.

즉,

> **PLURA-XDR은 MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, 데이터 유출, 랜섬웨어, APT까지 하나의 흐름으로 분석하고 대응합니다.**

## 2-3. NHN Cloud에 어떤 경쟁력을 주는가

PLURA-XDR이 추가되면 NHN Cloud는  
“다양한 기능을 가진 클라우드”를 넘어  
**실제 대응 가능한 클라우드 보안 플랫폼**으로 설명할 수 있습니다.

### 기존 구조와 비교

| 구분 | 기존 다중 보안 서비스 | NHN Cloud + PLURA-XDR |
| --- | --- | --- |
| 도입 방식 | 기능별 개별 선택 | 통합 제품 관점 |
| 운영 화면 | 다중 콘솔 | 통합 흐름 중심 |
| 분석 방식 | 운영자 수동 연계 | 웹-서버 상관분석 |
| 제로데이 대응 | 제한적 | 가능 |
| 크리덴셜 스터핑 대응 | 부분적 | 가능 |
| 랜섬웨어/APT 대응 | 분산적 | 통합 가능 |
| 민감정보 유출 탐지 | 제한적 | 응답 본문 기반 가능 |
| 자동 대응 | 부분적 | 통합 적용 |
| 기준 체계 | 기능별 분산 | MITRE ATT&CK 기반 운영 |
| 비용 설명 | 누적 과금 구조 | 단순한 제품 비용 구조 |

즉, PLURA-XDR은 NHN Cloud의 기존 보안 기능을 부정하는 것이 아니라,  
**산재된 기능을 하나의 경쟁력으로 재구성하는 역할**을 합니다.

## 2-4. 비용 측면의 의미

고객은 여러 보안 서비스를 따로 선택할수록  
도입 비용, 운영 비용, 인력 비용, 교육 비용을 함께 부담하게 됩니다.

반면 PLURA-XDR은

- 웹 구간 분석
- 서버 행위 분석
- 통합 XDR 상관분석
- 자동 대응
- 포렌식
- 랜섬웨어/APT 대응 관점의 통합 운영

을 하나의 체계로 제공하므로,  
고객은 이를 **복수 서비스의 합산 비용**이 아니라  
**실제로 사용할 수 있는 통합 정보보안 제품 비용**으로 이해할 수 있습니다.

제안 단계에서는 다음과 같이 설명할 수 있습니다.

> 기존 개별 보안 서비스 다중 선택 구조 대비,  
> 도입 항목 단순화와 운영 인력 절감을 통해  
> **총비용 기준 20~30% 수준의 효율 개선 가능성**을 기대할 수 있습니다.

# 3. 어떻게 시작할 것인가

이 제안의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다.

핵심은 **작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 것**입니다.

## 3-1. 적용 구조

```text
인터넷/외부 공격
   ↓
NHN Cloud 서비스 구간
   ↓
PLURA-Web
 - 요청/응답 본문 분석
 - 제로데이 공격 탐지
 - 크리덴셜 스터핑 탐지
   ↓
PLURA-XDR
 - 웹 이벤트 상관분석
 - 인증/세션 이상행위 분석
 - MITRE ATT&CK 기준 행위 분석
   ↓
PLURA-EDR
 - 프로세스/파일/네트워크 행위 탐지
 - 랜섬웨어/APT 후속 행위 탐지
   ↓
자동 대응 / 알림 / 포렌식
