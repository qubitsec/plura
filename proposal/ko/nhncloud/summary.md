# PLURA-XDR 제안 요약
## NHN Cloud와 PLURA-XDR 공동 사업 및 보안 고도화 제안

**미토스 기반 AI 신위협 대응**  
> “미토스와 같은 AI 기반 신위협 시대에는 시그니처 중심 방어만으로는 부족합니다. PLURA-XDR은 **요청·응답 본문 분석 + 웹-서버 상관분석 + 자동 대응**으로 알려지지 않은 공격에 더 빠르게 대응할 수 있습니다.”

---

# 1. 왜 지금 이 제안이 필요한가

NHN Cloud는 Web Firewall, SIEM, Bastion, Security Monitoring 등 다양한 정보보안 서비스를 이미 제공하고 있습니다. 

문제는 **보안 기능의 부족**이 아니라,  
기능이 **산재되어 있어 고객이 직접 선택·이해·조합해야 한다는 점**입니다. 

그 결과 고객과 내부 운영 조직은 다음 부담을 안게 됩니다.

- 여러 보안 기능을 각각 이해하고 선택해야 함
- 운영 화면과 콘솔이 분산되어 있음
- 웹 공격, 계정 공격, 서버 침해를 하나의 흐름으로 보기 어려움
- 탐지 이후 대응과 포렌식까지 연결하기 어려움
- 기능이 늘수록 비용과 운영 부담도 함께 증가함

즉,

> **서비스는 많지만, 실제 침해 대응 체계로는 분산되어 있는 구조**입니다. 

오늘날 공격은 단일 이벤트로 끝나지 않습니다.

- 웹 취약점 공격
- 크리덴셜 스터핑
- 권한 상승
- 내부 확산
- 데이터 유출
- 랜섬웨어
- APT

따라서 고객이 원하는 것은 기능 목록이 아니라,  
**공격 흐름 전체를 하나로 보고 대응할 수 있는 통합 운영 체계**입니다. 

그리고 후발 클라우드 사업자가 시장 판도를 바꿀 수 있는 가장 현실적인 방법은,  
**최고 수준의 보안 상품을 더 합리적인 비용 구조로 제공하는 것**입니다.

### 이번 제안의 핵심

본 제안은 두 가지입니다.

**제안 1**  
NHN Cloud와 PLURA-XDR을 결합해,  
기존 고객과 외부 신규 잠재 고객에게 함께 제안할 수 있는  
**경쟁력 있는 통합 보안 상품 구조**를 만드는 것입니다.

**제안 2**  
이 구조를 바탕으로,  
NHN Cloud 내부 보안 체계도  
**제로데이, 크리덴셜 스터핑, 랜섬웨어, APT까지 더 강하게 대응할 수 있는 구조**로 고도화하는 것입니다.

즉, 이 제안은 단순한 솔루션 추가가 아니라,  
**외부 시장 확대와 내부 보안 강화를 함께 추진하는 구조적 제안**입니다.

---

# 2. 왜 PLURA-XDR인가

PLURA-XDR은 단순한 웹방화벽이나 EDR이 아닙니다.  
핵심은 **웹 + 서버 + XDR + 자동 대응 + 포렌식**을 하나의 흐름으로 연결하는 데 있습니다. 

## 2-1. 핵심 차별점

- **제로데이 공격 탐지**
- **크리덴셜 스터핑 대응**
- **요청·응답 본문 기반 정밀 분석**
- **민감정보 유출 탐지**
- **랜섬웨어 대응**
- **APT 공격 대응**
- **LOLBAS 악용 행위 대응**
- **웹-서버 상관분석**
- **자동 대응과 포렌식 연계**

즉, 단순히 알려진 공격을 막는 기능이 아니라,  
**공격 흐름 전체를 해석하고 운영하는 체계**입니다.

## 2-2. 모든 기준은 MITRE ATT&CK

PLURA-XDR의 중요한 차이는  
탐지 항목을 기능 중심이 아니라  
**MITRE ATT&CK 기준으로 해석하고 운영한다는 점**입니다. 

이 기준에서 보면,

- 크리덴셜 스터핑은 **Initial Access / Credential Access**
- 랜섬웨어는 **Impact**
- APT는 **여러 ATT&CK 단계가 장기간 연결된 흐름**
- LOLBAS 악용은 **Execution / Defense Evasion / Lateral Movement**와 연결될 수 있는 현실적인 행위 기반 위협

입니다.

즉,

> **PLURA-XDR은 MITRE ATT&CK 기준으로 웹 공격, 계정 공격, 내부 확산, LOLBAS 악용, 랜섬웨어, APT까지 하나의 흐름으로 분석하고 대응합니다.**

## 2-3. 미토스급 AI 신위협 대응 경쟁력

최근 AI는 제로데이 취약점 탐지와 공격 자동화 속도를 높이고 있습니다.  
이런 환경에서는 시그니처 중심의 개별 보안 도구만으로는 대응에 한계가 있습니다. 

PLURA-XDR은 다음 구조로 차별화됩니다.

- **요청·응답 본문 정밀 분석**으로 알려지지 않은 공격 징후 식별
- **웹-서버 XDR 상관분석**으로 실제 서버 행위와 연결 추적
- **자동 대응 + 포렌식**으로 탐지 이후 조치까지 연계

즉, NHN Cloud가 외부 고객에게 제안할 때도  
PLURA-XDR은 **미토스급 AI 신위협까지 대응 가능한 차세대 통합 보안 상품**이라는 메시지를 줄 수 있습니다.

## 2-4. NHN Cloud에 주는 경쟁력

PLURA-XDR이 추가되면 NHN Cloud는  
“다양한 기능을 가진 클라우드”를 넘어  
**실제 대응 가능한 클라우드 보안 플랫폼**으로 설명할 수 있습니다. 

| 구분 | 기존 다중 보안 서비스 | NHN Cloud + PLURA-XDR |
| --- | --- | --- |
| 도입 방식 | 기능별 개별 선택 | 통합 상품 관점 |
| 운영 화면 | 다중 콘솔 | 통합 흐름 중심 |
| 분석 방식 | 운영자 수동 연계 | 웹-서버 상관분석 |
| 제로데이 대응 | 제한적 | 가능 |
| 크리덴셜 스터핑 대응 | 부분적 | 가능 |
| 랜섬웨어/APT 대응 | 분산적 | 통합 가능 |
| LOLBAS 대응 | 제한적 | 행위 기반 대응 가능 |
| 민감정보 유출 탐지 | 제한적 | 응답 본문 기반 가능 |
| 기준 체계 | 기능별 분산 | MITRE ATT&CK 기반 운영 |
| 비용 설명 | 누적 과금 구조 | 단순한 통합 상품 구조 |

즉, PLURA-XDR은 NHN Cloud의 기존 보안 기능을 부정하는 것이 아니라,  
**산재된 기능을 하나의 경쟁력으로 재구성하는 역할**을 합니다.

## 2-5. 비용 측면의 의미

고객은 여러 보안 서비스를 따로 선택할수록  
도입비, 운영비, 인력비, 교육비를 함께 부담하게 됩니다. 

반면 PLURA-XDR은  
웹 구간 분석, 서버 행위 분석, 통합 XDR 상관분석, 자동 대응, 포렌식, 랜섬웨어/APT 대응을 하나의 체계로 제공하므로,  
고객은 이를 **복수 서비스의 합산 비용**이 아니라  
**실제로 사용할 수 있는 통합 정보보안 제품 비용**으로 이해할 수 있습니다.

제안 단계에서는 다음처럼 설명할 수 있습니다.

> 기존 개별 보안 서비스 다중 선택 구조 대비,  
> 도입 항목 단순화와 운영 인력 절감을 통해  
> **총비용 기준 30~50% 수준의 효율 개선 가능성**을 기대할 수 있습니다.

# 3. 어떻게 시작할 것인가

이 제안의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다. 

핵심은 **작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 것**입니다.

## 3-1. 적용 방향

PLURA-XDR은 두 가지 방식으로 적용할 수 있습니다.

**1) Co-exist 모델**  
기존 NHN Cloud Web Firewall, SIEM, Security Monitoring을 유지하면서  
PLURA-XDR이 정밀 분석 및 상관분석 계층을 추가

**2) Replace / Consolidate 모델**  
일부 개별 기능을 PLURA-XDR로 통합하여  
운영 화면과 대응 체계를 단순화

즉, NHN Cloud와 PLURA-XDR은  
대체 관계만이 아니라 **공존과 단계적 통합 전환이 가능한 구조**입니다.

## 3-2. 단계별 시작 방식

- 인터넷 노출 웹 서비스 1개 선정
- 핵심 API 또는 로그인 구간 1개 선정
- 관련 서버에 단계적 EDR 적용
- 크리덴셜 스터핑 및 웹-서버 연계 시나리오 검증
- 랜섬웨어/APT 전 단계 행위 검증
- 운영 콘솔, 알림, 포렌식 확보 흐름 확인

즉, 처음부터 전체 전환이 아니라  
**작게 시작해 빠르게 검증하고 단계적으로 확대하는 방식**이 적합합니다.

## 3-3. 경영진 판단 포인트

이 제안의 의미는 명확합니다.

**첫째, NHN Cloud의 기존 보안 기능을 부정하지 않습니다.**  
산재된 기능을 통합 가치로 재구성합니다.

**둘째, 고객이 이해하기 쉬운 상품 구조를 만듭니다.**  
“여러 기능”이 아니라 **실제 대응 가능한 통합 보안 상품**으로 제안할 수 있습니다.

**셋째, 차별화 포인트가 분명합니다.**  
제로데이, 크리덴셜 스터핑, 응답 본문 기반 민감정보 유출, 랜섬웨어/APT 대응, LOLBAS 악용 행위 대응, MITRE ATT&CK 기반 운영, AI 신위협 대응은 기존 일반 WAF/EDR 제안보다 강합니다.

**넷째, 작게 시작할 수 있습니다.**  
PoC 부담이 낮고, 빠르게 검증 가능합니다.

# 최종 결론

> NHN Cloud의 현재 보안 서비스가 “기능의 집합”이라면,  
> PLURA-XDR은 이를 **제로데이 탐지, 크리덴셜 스터핑 대응, LOLBAS 악용 대응, 랜섬웨어/APT 대응, 민감정보 유출 탐지까지 포함하는 실제 대응 가능한 통합 운영 체계**로 완성하는 제안입니다.

즉, 이 제안의 본질은 단순한 솔루션 추가가 아니라,

- **NHN Cloud 클라우드 경쟁력의 재구성**
- **고객 선택 부담과 운영 부담의 완화**
- **MITRE ATT&CK 기준의 실제 대응 체계 확보**
- **AI 신위협까지 포함한 차세대 보안 상품화**
- **통합 상품 관점의 비용 구조 제시**

에 있습니다.

결국 후발 클라우드 사업자가 시장 판도를 바꿀 수 있는 가장 현실적인 방법은,  
**최고 수준의 보안 상품을 더 합리적인 비용 구조로 제공하는 것**입니다.

이 점에서 NHN Cloud와 PLURA-XDR의 결합은  
기능 추가가 아니라 **시장 경쟁력을 재구성하는 전략적 제안**입니다.
