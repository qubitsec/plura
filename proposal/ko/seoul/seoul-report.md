# 웹/호스트 탐지 운영 현황 보고서

## 1. 보고 요약

웹/호스트 탐지 운영 현황을 점검한 결과, 웹 구간에서는 외부의 자동화된 정찰 및 스캐닝 시도가 지속적으로 발생하고 있으며, 주요 유형은 제한된 파일 접근 시도, 공개 취약 경로 대상 스캔, SQL 인젝션 시도 등으로 확인되었습니다.

이번 점검에서는 **날짜별 주요 탐지 사례를 기준으로 총 3차례 분석 조사**를 수행하였으며, 각 사례별로 공격 유형, 시도 경로, 응답 결과, 침해 성공 여부를 확인하였습니다.

호스트 구간에서는 전반적으로 탐지 빈도가 높지 않았으나, 매월 초 반복적으로 리눅스 계정 비밀번호 변경 관련 이벤트가 다수 발생하는 패턴이 확인되었습니다.

주요 웹 탐지 사례는 대부분 302 또는 404 응답으로 확인되었으며, 보고 시점 기준 실제 침해 성공 정황은 확인되지 않았습니다.

---

## 2. 전체 현황

### 2.1 웹 탐지 현황

- 일별 탐지 규모: 약 100~200건
- 주별 탐지 규모: 적을 때 500건, 많을 때 3,000건 이상
- 주요 유형: LFI 계열 및 제한된 파일 액세스 시도 다수 발생

### 2.2 주요 웹 탐지 필터 현황

#### 1) 디렉토리 접근 공격 (Low)
- 모두 404 응답으로 확인
- 대상 자원 접근 및 공격 성공 정황 없음

#### 2) 제한된 파일 액세스 시도 (High)
- 하루 약 10건 수준 탐지
- 서로 다른 IP에서 반복 발생
- 주로 `/.env` 파일 액세스 시도

#### 3) 외부 링크 IP 주소 사용 (Low)
- 하루 10~30건 발생
- 확인 결과 공격 아님

#### 4) 파일 업로드 시도 (Middle)
- 20~30건 탐지
- 모두 한국 IP
- 확인 결과 정상 요청

#### 5) IMG 태그 공격 (High)
- 하루 2~3건 탐지
- 모두 한국 IP
- 정상적인 게시물 수정 중 이미지 삽입 행위로 판단

## 4. 웹 탐지 종합 판단

주요 웹 탐지는 다음과 같은 특성을 보였습니다.

- `/.env`, `/config`, `xmlrpc.php`, `phpunit eval-stdin`, `wp-config.php` 등 공개된 취약 경로 또는 민감 파일 대상 시도
- 자동화된 정찰 및 스캐닝 성격이 강함
- 응답 결과는 주로 302 또는 404
- 보고 시점 기준 실제 파일 노출, 명령 실행, 침해 성공 정황은 확인되지 않음

종합하면, 외부 공격 시도는 지속적으로 발생하고 있으나 현재까지 확인된 주요 사례는 침해 성공으로 이어지지 않은 것으로 판단됩니다.

## 5. 호스트 탐지 현황

### 5.1 전체 현황

- 호스트 탐지는 거의 발생하지 않음
- 일별 0~18건 수준

### 5.2 반복 발생 패턴

매월 1일, 4~5일경 다음 이벤트가 반복적으로 탐지됨

- 계정 액세스 제거 `[T1531]` (Middle)
- 계정 비밀번호 변경 `(SYSCALL)` (High)
- 계정 비밀번호 변경 `(CONFIG_CHANGE)` (High)

### 5.3 판단

- 반복 발생 패턴으로 보아 정기 운영 작업 또는 계정 관리 작업과의 연관성 검토 필요
- 현재 기준 특이 침해 징후보다는 주기적 운영 이벤트 가능성이 높음

## 6. 로그 업로드 용량 현황

### 6.1 호스트 로그
- 25~27MB

### 6.2 웹 로그
- 적을 때: 200MB
- 평균: 300~400MB
- 많을 때: 700~800MB
- 최대 발생일: 2026/04/02
- 최대 용량: 814.59MB

### 6.3 판단
- 로그 업로드 용량은 웹 로그 비중이 매우 높음
- 웹 탐지 증가 시 로그 저장 및 분석 리소스에 대한 지속적 관리 필요

## 7. 종합 의견

현재 운영 환경에서는 외부의 자동화된 취약 경로 스캔과 정찰 시도가 지속적으로 발생하고 있습니다. 다만 주요 탐지 사례를 검토한 결과, 대부분 302 또는 404 응답으로 확인되어 실제 침해 성공 정황은 확인되지 않았습니다.

호스트 구간은 전반적으로 안정적인 수준이나, 월초 반복 발생하는 계정 관련 이벤트에 대해서는 운영성 이벤트 여부를 지속적으로 확인할 필요가 있습니다.

웹 로그 비중이 높고 탐지량 변동폭도 큰 만큼, 웹 구간 중심의 지속적인 모니터링과 로그 관리가 중요합니다.

---

## 8. 결론

현재 웹/호스트 운영 현황을 종합하면, 외부 공격 시도는 지속적으로 존재하나 주요 사례 대부분은 탐지 단계 또는 무효화된 시도로 확인되었습니다.

따라서 현 시점에서는 웹 구간에 대한 지속적인 모니터링, 반복 탐지 유형에 대한 추적 관리, 호스트 계정 이벤트의 운영성 검토를 중심으로 운영하는 것이 적절합니다.
