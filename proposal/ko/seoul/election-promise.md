# 1. 왜 지금 서울시 사이버안보 대전환인가

서울은 더 안전하고 성숙한 스마트시티로 발전해야 합니다.  
스마트시티의 핵심은 편리함만이 아닙니다. 시민의 개인정보가 안전하고, 행정서비스가 멈추지 않으며, 위기 상황에서도 도시 기능이 유지되는 것입니다.

서울의 행정, 교통, 복지, 의료, 상수도, 재난안전, 시민 민원 서비스는 이미 디지털 시스템 위에서 운영되고 있습니다. 따라서 사이버공격은 더 이상 단순한 전산 장애가 아닙니다. 시민의 개인정보, 행정서비스, 도시 기능을 위협하는 새로운 안전사고이자 도시 재난입니다.

특히 AI 공격은 이미 현실화되고 있습니다.  
취약점 탐지, 공격 경로 분석, 악성코드 변형, 계정 탈취, 자동화된 침투 시도가 과거보다 훨씬 빠른 속도로 이루어질 수 있습니다. 미토스급 AI 해킹은 기존의 사후 보고, 개별 장비, 기관별 대응 중심의 보안체계로는 대응하기 어렵습니다.

이제 서울시는 사이버안보 분야의 근본적인 전환을 추진해야 합니다.

그 출발점은 **서울시 표준로깅 시스템**입니다.  
로그가 없으면 공격 원인도 알 수 없고, 피해 규모도 산정할 수 없으며, 시민에게 어떤 정보가 영향을 받았는지도 설명할 수 없습니다. 지금까지 정보보안은 장비 도입과 관제 확대에 집중해 왔지만, 정작 가장 중요한 로그에 대해서는 제대로 된 기준과 지침이 부족했습니다.

서울시는 이제 다음 단계로 나아가야 합니다.

**서울시청, 25개 자치구, 산하기관, 투자·출연기관, 시립병원, 복지시설의 보안 기록을 하나의 기준으로 통일하고, 이를 기반으로 서울시 전체 통합 정보보안 플랫폼을 구축하겠습니다.**

그 이유는 **책임 행정** 때문입니다.  
사이버사고가 발생했을 때 시민은 “어디에서 시작됐는지”, “어떤 정보가 영향을 받았는지”, “누가 언제 어떤 조치를 했는지”, “왜 피해 확산을 막지 못했는지”를 묻습니다. 서울시청, 자치구, 산하기관이 서로 다른 기준으로 기록하고 서로 다른 절차로 대응하면, 사고 이후 원인 분석도 피해 규모 산정도 책임 있는 설명도 어려워집니다. 서울 전체가 같은 기준으로 기록하고, 같은 플랫폼에서 분석하고, 같은 절차로 대응해야 시민에게 설명할 수 있습니다.

이 플랫폼은 단순한 보안장비의 집합이 아닙니다.  
서울 전체의 로그, 위협, 취약점, 패치, 사고 대응, 선조치·후보고 이력을 하나의 기준으로 관리하는 **서울형 사이버안보 운영체계**입니다.

서울을 더 성숙한 스마트시티로 발전시키기 위해서는 안전사고 없는 디지털 기반이 필요합니다. 그리고 그 기반은 **표준화된 로그, 통합된 플랫폼, 등급별 대응 체계, 긴급 시 선조치·후보고가 가능한 제도**에서 시작됩니다.

**기록하고, 분석하고, 즉시 대응하는 서울.**  
**AI 공격에도 흔들리지 않는 안전한 스마트시티 서울.**  
그 변화를 시작하겠습니다.

# 4. 핵심 공약 1: 서울시 표준로깅 시스템 도입

## “보안의 출발점은 기록입니다.”

사이버보안에서 가장 중요한 것은 로그입니다.  
공격이 발생했을 때 무엇이 들어왔고, 어떤 계정이 사용되었고, 어떤 서버에서 어떤 행위가 일어났으며, 어떤 데이터가 접근·변경·유출되었는지를 확인할 수 있어야 합니다.

그러나 현재 많은 기관의 정보보안 체계는 로그를 충분히 남기지 못하거나, 남기더라도 기관별·시스템별 기준이 달라 공격의 전체 흐름을 재구성하기 어렵습니다.

로그 기준이 부실하면 다음과 같은 문제가 발생합니다.

| 문제 | 결과 |
|---|---|
| 로그 수집 기준 부재 | 어떤 이벤트를 남겨야 하는지 기관마다 다름 |
| 로그 형식 불일치 | 서울시청, 자치구, 산하기관 간 통합 분석 어려움 |
| 중요 로그 누락 | 공격 원인 분석 실패 |
| 보존 기간 불명확 | 사고 이후 증거 확보 실패 |
| 개인정보 접근 로그 미흡 | 피해 규모 산정 실패 |
| 웹 요청·계정·서버 로그 분리 | 공격 흐름 파악 실패 |
| 수동 보고 중심 | 대응 방안 도출 지연 |

결과적으로 사고가 발생해도 **공격 원인 분석, 피해 규모 산정, 재발 방지 대책 수립, 책임 소재 확인, 시민 안내**가 모두 늦어질 수밖에 없습니다.

따라서 서울시는 통합 정보보안 플랫폼 구축에 앞서, 그 기반이 되는 **서울시 표준로깅 시스템**을 도입해야 합니다.

# 7. 표준로깅 시스템의 핵심 원칙

## 1. 공격 흐름을 재구성할 수 있어야 합니다

단일 로그만으로는 공격을 이해할 수 없습니다.  
웹 요청, 계정 행위, 서버 이벤트, 개인정보 접근 기록이 하나의 흐름으로 연결되어야 합니다.

예를 들어 다음과 같은 분석이 가능해야 합니다.

```text
외부 공격 요청
→ 로그인 시도
→ 계정 탈취
→ 관리자 권한 상승
→ 서버 명령 실행
→ 내부 이동
→ 개인정보 조회
→ 외부 전송
→ 차단 및 복구
```

이 흐름을 확인하지 못하면 사고 대응은 추측에 의존하게 됩니다.

## 2. 서울시 전체가 같은 로그 기준을 사용해야 합니다

서울시청은 A 방식, 자치구는 B 방식, 산하기관은 C 방식으로 로그를 남기면 통합 분석이 불가능합니다.

따라서 서울시는 다음 기준을 통일해야 합니다.

| 항목 | 표준화 내용 |
|---|---|
| 로그 항목 | 반드시 남겨야 할 필수 로그 정의 |
| 로그 형식 | 기관 간 비교 가능한 공통 포맷 적용 |
| 로그 시간 | 표준 시간 동기화 |
| 로그 보존 | 중요도별 보존 기간 기준 마련 |
| 로그 무결성 | 위변조 방지 및 원본 보존 |
| 로그 접근권한 | 열람·수정·삭제 권한 통제 |
| 로그 검색 | 사고 시 즉시 검색 가능한 구조 |
| 로그 연계 | 통합 정보보안 플랫폼과 자동 연동 |

## 3. 피해 규모를 산정할 수 있어야 합니다

보안사고 이후 가장 중요한 질문은 다음입니다.

```text
누가 공격했는가?
어디로 들어왔는가?
어떤 계정이 사용되었는가?
어떤 시스템이 영향을 받았는가?
어떤 개인정보가 조회되었는가?
얼마나 많은 시민이 영향을 받았는가?
현재도 공격이 진행 중인가?
어떤 조치를 해야 재발을 막을 수 있는가?
```

이 질문에 답하려면 표준화된 로그가 있어야 합니다.  
로그가 없으면 피해 규모도, 시민 안내도, 재발 방지도 제대로 할 수 없습니다.

## 4. 미토스급 AI 해킹 대응에 맞는 로그여야 합니다

미토스급 AI 해킹은 빠르고, 자동화되고, 여러 취약점을 연쇄적으로 악용할 수 있습니다.  
따라서 단순한 접속 기록만으로는 부족합니다.

서울시 표준로깅 시스템은 다음을 확인할 수 있어야 합니다.

| 대응 항목 | 필요한 로그 |
|---|---|
| AI 기반 취약점 공격 | 웹 요청, 요청 본문, 응답 상태, 공격 패턴 |
| 계정 탈취 | 로그인 실패·성공, 접속 위치, 세션 변화 |
| 내부 이동 | 서버 접속, 원격 명령, 권한 상승 |
| 랜섬웨어 | 파일 변경, 대량 암호화, 프로세스 실행 |
| 개인정보 유출 | 대량 조회, 다운로드, 외부 전송 |
| 관리자 권한 악용 | 정책 변경, 계정 생성, 권한 변경 |
| 선조치·후보고 | 차단, 격리, 계정 잠금, 조치 이력 |

# 8. 통합 정보보안 플랫폼과 표준로깅 시스템의 관계

## “표준로깅 시스템이 있어야 통합 플랫폼이 완성됩니다.”

서울시 통합 정보보안 플랫폼은 표준로깅 시스템 위에서 완성됩니다.

로그 기준이 없으면 통합 플랫폼은 단순히 여러 장비의 알림을 모아 보는 수준에 머물게 됩니다.  
반대로 표준로깅 시스템이 구축되면 서울시 전체의 보안 상황을 하나의 기준으로 분석할 수 있습니다.

```text
서울시 표준로깅 시스템
→ 서울시청·자치구·산하기관 로그 표준화
→ 통합 정보보안 플랫폼 연계
→ 공격 흐름 분석
→ 피해 규모 산정
→ 등급별 대응
→ 선조치·후보고
→ 재발 방지
→ 시민 신뢰 회복
```

따라서 공약의 구조는 다음처럼 추진합니다.

```text
1단계: 서울시 표준로깅 시스템 도입
2단계: 통합 정보보안 플랫폼 구축
3단계: 미토스급 해킹 등급별 대응 체계 운영
4단계: 선조치·후보고 제도화
5단계: 분기별 점검과 공개 토론으로 제도 정착
```

# 9. 핵심 공약 2: 서울시 통합 정보보안 플랫폼 구축

## “서울시청, 자치구, 산하기관의 보안을 하나로 묶겠습니다.”

그 이유는 **책임 행정** 때문입니다.

서울시청, 25개 자치구, 산하기관, 투자·출연기관, 시립병원, 복지시설이 각각 따로 움직이는 보안체계로는 미토스급 AI 해킹에 대응할 수 없습니다. 더 중요한 것은, 사고가 발생했을 때 시민에게 책임 있게 설명할 수 없다는 점입니다.

기관마다 다른 로그 기준, 다른 보안장비, 다른 보고 절차, 다른 대응 권한으로 운영되면 다음 질문에 답하기 어렵습니다.

```text
어디에서 공격이 시작되었는가?
어느 기관의 어떤 시스템이 영향을 받았는가?
어떤 개인정보가 조회되거나 유출되었는가?
누가 언제 어떤 조치를 했는가?
왜 피해 확산을 막지 못했는가?
앞으로 어떻게 재발을 막을 것인가?
```

서울시는 표준로깅 시스템을 기반으로 **서울시 통합 정보보안 플랫폼**을 구축하겠습니다.  
서울 전체가 같은 기준으로 기록하고, 같은 플랫폼에서 분석하고, 같은 절차로 대응해야 시민에게 설명할 수 있습니다.

**보안 통합은 기술 문제가 아니라 책임 행정의 문제입니다.**

이 플랫폼은 다음을 통합합니다.

| 구분 | 내용 |
|---|---|
| 통합 관제 | 서울시청·자치구·산하기관 보안 이벤트 통합 수집 |
| 통합 분석 | 웹 요청, 시스템 이벤트, 계정 행위, 서버 로그, 개인정보 접근 기록 연계 분석 |
| AI 대응 | 미토스급 자동화 공격 징후 분석 및 대응 우선순위 판단 |
| 자동 대응 | 악성 IP 차단, 계정 잠금, 서버 격리, 위험 세션 차단 |
| 패치 관리 | 기관별 보안 패치 현황, 지연 현황, 조치 이력 관리 |
| 사고 관리 | 선조치, 후보고, 재발 방지, 책임기관 지정 |
| 정책 관리 | 서울시 전체 보안 기준, 예외 승인, 점검 결과 통합 관리 |
| 공개 점검 | 분기별 이행 현황 점검 및 공개 토론 운영 |

# 10. 핵심 공약 3: 레거시 정보보안 체계 제거

## “불필요한 보안 낭비를 줄이고, 실제 AI 해킹 대응 역량에 집중하겠습니다.”

서울시의 정보보안은 더 이상 기관별 장비 도입, 개별 관제, 수동 보고, 사후 점검 방식에 머물러서는 안 됩니다.

서울시는 레거시 정보보안 체계를 단계적으로 제거하고, 통합 정보보안 플랫폼 중심으로 전환하겠습니다. 특히 레거시 제품들에 대한 실효성 평가를 실시해 불필요하고 의미 없이 예산이 낭비되는 요소를 제거하겠습니다. 법령과 필수 기준은 준수하되, 단지 인증 항목을 채우기 위한 이유만으로 사용되고 있는 제품과 절차는 과감히 걷어내겠습니다.

## 추진 원칙

1. **신규 레거시 보안장비 도입 제한**  
   통합 플랫폼과 연계되지 않는 개별 장비 도입을 제한합니다.

2. **기존 보안장비 실효성 평가**  
   기존 제품은 탐지 성과, 사고 대응 기여도, 로그 품질, 통합 연계 가능성, 유지비용을 기준으로 평가합니다.

3. **인증 항목 중심 제품 정리**  
   인증 항목 충족을 위해 형식적으로 유지되는 제품은 실제 공격 대응 효과를 검증하고, 실효성이 낮은 경우 통합·대체·폐지합니다.

4. **기존 보안장비 단계적 통합**  
   기존 장비는 즉시 폐기하는 것이 아니라, 로그·정책·대응 절차를 통합 플랫폼으로 흡수합니다.

5. **중복 기능 정리**  
   기관별로 중복 운영되는 관제, 보고, 점검 기능을 통합합니다.

6. **수동 보고 축소**  
   사고 보고, 패치 현황, 조치 이력은 플랫폼에서 자동 관리되도록 전환합니다.

7. **서울시 표준 보안 운영 절차 적용**  
   자치구와 산하기관도 동일한 대응 기준을 사용하도록 합니다.

## 평가 기준

| 평가 항목 | 판단 기준 |
|---|---|
| 공격 대응성 | 미토스급 AI 해킹, 계정 탈취, 랜섬웨어, 개인정보 유출 대응에 실제 기여하는가 |
| 로그 품질 | 원인 분석과 피해 규모 산정에 필요한 로그를 제공하는가 |
| 통합 가능성 | 서울시 통합 정보보안 플랫폼과 연계 가능한가 |
| 운영 효율성 | 중복 기능이나 과도한 수동 운영 부담이 없는가 |
| 비용 적정성 | 유지비용 대비 실질적인 보안 효과가 있는가 |
| 책임성 | 사고 발생 시 누가 어떤 조치를 했는지 추적 가능한가 |

# 13. 핵심 공약 6: 사이버보안 통합 정책 담당관 지정과 실무 중심 자문위원회 구축

## “정책과 실무를 분리하고, 독단이 아닌 검증 가능한 보안 정책을 만들겠습니다.”

사이버보안은 실무 관제만으로 해결되지 않습니다.  
예산, 조례, 계약, 감사, 개인정보, 기관 간 조정, 사고 공개 기준까지 포함하는 정책 영역입니다.

서울시 사이버보안 정책을 총괄하는 **사이버보안 통합 정책 담당관**을 별도로 지정하겠습니다. 정책 수립, 예산 조정, 기관 협의, 이행 점검을 전문적으로 수행하도록 하겠습니다.

동시에 정책이 특정 부서나 일부 전문가의 판단에만 의존하지 않도록 **서울시 사이버보안 실무 자문위원회**를 구축하겠습니다. 자문위원회는 단순한 명망형 위원회가 아니라, 실제 침해사고 대응, 로그 분석, 클라우드 보안, 보안관제, 개인정보 보호, 디지털 포렌식, 패치 관리, 보안 제품 평가 경험을 갖춘 실무 전문가 중심으로 구성하겠습니다.

교수 중심의 형식적 자문에 머무르지 않겠습니다. 학계의 연구 전문성은 필요한 영역에서 활용하되, 위원회의 중심은 실제 현장에서 공격을 분석하고 대응해 본 보안 전문가, 침해대응 전문가, 보안 아키텍트, 로그·포렌식 전문가, 개인정보보호 실무자, 공공 정보화 사업 보안 경험자에게 두겠습니다.

| 구분 | 담당 역할 |
|---|---|
| 사이버보안 통합 정책 담당관 | 정책 수립, 제도 개선, 예산 조정, 기관 협의, 성과 평가 |
| 사이버안전센터 | 실시간 관제, 침해 분석, 기술 대응, 사고 조치 |

# 16. 후보 메시지

존경하는 서울시민 여러분,

서울은 더 안전하고 성숙한 스마트시티로 발전해야 합니다.  
그 핵심은 편리함을 넘어, 시민의 개인정보와 행정서비스를 안전하게 지키는 것입니다.

AI 공격은 이미 현실화되고 있습니다.  
미토스급 AI 해킹은 취약점 탐지와 공격 자동화를 빠르게 만들고 있으며, 기존의 사후 보고 중심 보안체계로는 대응이 어렵습니다. 서울시장 후보는 이제 “보안을 강화하겠다”는 추상적 약속이 아니라, AI 해킹에 어떻게 대응할 것인지 구체적인 체계를 제시해야 합니다.

저는 서울시 사이버안보 체계를 근본적으로 전환하겠습니다.

첫째, **서울시 표준로깅 시스템**을 도입하겠습니다.  
로그가 없으면 공격 원인도, 피해 규모도, 대응 방안도 확인할 수 없습니다. 서울시청, 자치구, 산하기관의 보안 로그를 하나의 기준으로 통일하겠습니다.

둘째, **서울시 통합 정보보안 플랫폼**을 구축하겠습니다.  
표준화된 로그를 기반으로 서울 전체의 보안 상황을 하나의 기준으로 분석하고, 기관 간 협의와 대응이 가능하도록 하겠습니다. 이것은 단순한 시스템 통합이 아니라 시민에게 원인, 피해, 조치, 재발 방지를 설명하기 위한 책임 행정의 기반입니다.

셋째, **미토스급 AI 해킹 대응 프로세스**를 마련하겠습니다.  
위협을 등급화하고, 긴급 상황에서는 시민 피해를 막기 위해 간소화된 절차로 선조치·후보고가 가능하도록 제도화하겠습니다.

넷째, **레거시 정보보안 체계를 평가하고 정리하겠습니다.**  
실제 공격 대응에 기여하지 못하고, 인증 항목 충족만을 위해 유지되는 제품과 절차는 과감히 걷어내겠습니다. 예산은 장비 유지가 아니라 AI 해킹 대응 역량 강화에 집중하겠습니다.

다섯째, **사이버보안 통합 정책 담당관과 실무 중심 자문위원회**를 두겠습니다.  
정책과 실무를 분리하고, 서울시 전체의 보안 기준, 예산, 제도, 이행 점검을 책임 있게 관리하겠습니다. 동시에 침해대응, 로그 분석, 보안관제, 포렌식, 클라우드 보안 등 실제 현장 경험을 가진 전문가가 정책을 검증하도록 하겠습니다.

여섯째, **분기별 점검과 공개 토론**을 운영하겠습니다.  
새로운 제도가 실제 현장에 정착되는지 시민, 실무 보안 전문가, 산업계와 함께 점검하고 개선하겠습니다.

서울의 사이버안보는 더 이상 기관별로 따로 움직여서는 안 됩니다.  
서울시청, 자치구, 산하기관이 하나의 기준으로 기록하고, 하나의 플랫폼으로 분석하고, 하나의 체계로 대응해야 합니다.

저는 AI 해킹 대응을 준비한 시장 후보로서, 서울시 사이버안보를 책임 행정의 관점에서 다시 설계하겠습니다.

**기록하고, 분석하고, 즉시 대응하는 서울.**  
**시민에게 설명할 수 있는 책임 있는 보안 행정.**  
**AI 공격에도 흔들리지 않는 안전한 스마트시티 서울을 만들겠습니다.**

# 17. 선거공보용 짧은 문구

**보안의 출발점은 로그입니다. 서울시 표준로깅 시스템을 도입해 서울시청·자치구·산하기관의 보안 기록을 하나로 통일하고, 이를 기반으로 서울시 통합 정보보안 플랫폼을 완성하겠습니다.**

**로그가 없으면 보안도 없습니다. 서울시 표준로깅 시스템으로 공격 원인, 피해 규모, 대응 방안을 즉시 확인할 수 있는 통합 보안 도시를 만들겠습니다.**

**서울시청·자치구·산하기관의 보안을 하나로 통합하고, 미토스급 AI 해킹에는 선조치·후보고가 가능한 서울형 사이버안보 체계를 만들겠습니다.**

**보안 통합은 기술 문제가 아니라 책임 행정의 문제입니다. 서울시 전체가 같은 기준으로 기록하고, 같은 플랫폼에서 분석하고, 같은 절차로 대응하겠습니다.**

**AI 해킹 대응을 준비한 시장 후보로서, 레거시 보안 낭비를 줄이고 실효성 있는 통합 보안 행정을 추진하겠습니다.**
