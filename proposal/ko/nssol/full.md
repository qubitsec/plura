# PLURA-XDR 제안서
## NS Solutions와 PLURA-XDR 협력을 통한 일본 제조·공급망 보안 고도화 제안

---

## 제안의 출발점

미토스 AI와 같은 신종 위협이 등장하는 시대에,  
현재의 보안 운영 방식만으로 충분하다고 말할 수 있을까요?

일본의 제조업과 공급망 보안은  
앞으로 어떤 구조와 운영 모델 위에서 더 높은 신뢰를 가질 수 있을까요?

공격은 더 빨라지고, 더 정교해지고 있으며,  
특히 공급망으로 연결된 중견·중소 기업까지 포함하면  
기존의 개별 보안 제품 중심 운영만으로는  
전체 위험을 효과적으로 관리하기가 점점 어려워지고 있습니다.

이제 필요한 것은  
웹, 계정, 서버, 데이터 유출, 대응, 포렌식을  
하나의 흐름으로 연결할 수 있는 운영 체계입니다.

즉, 필요한 것은  
개별 보안 기능의 추가가 아니라  
클라우드 기반 통합 정보보안 플랫폼입니다.

NS Solutions는 인프라 구축과 운영 경험을 바탕으로,  
고객의 현실적인 과제를 가장 잘 이해할 수 있는 위치에 있습니다.

그리고 PLURA-XDR은  
웹 요청·응답 분석, 계정 이상행위 탐지, 서버 행위 분석,  
데이터 유출 탐지, 자동 대응, 포렌식을  
하나의 운영 체계로 연결할 수 있는 플랫폼입니다.

본 제안은  
이 두 역량의 결합을 통해  
일본의 제조업 및 공급망 고객에게  
보다 현실적이고 운영 가능한 통합 보안 모델을 제시하고자 하는 데 목적이 있습니다.

---

## 0️⃣ 제안 요약

본 제안의 핵심은 단순합니다.

일본의 제조업과 공급망 고객은  
더 이상 개별 보안 제품을 각각 도입하고 운영하는 방식만으로  
현실적인 대응 체계를 만들기 어렵습니다.

웹 공격은 계정 공격으로 이어지고,  
계정 공격은 서버 침해와 내부 확산으로 이어지며,  
그 이후에는 데이터 유출, 랜섬웨어, 장기 잠복형 공격으로 확장될 수 있습니다.

따라서 고객이 실제로 필요로 하는 것은  
보안 기능의 나열이 아니라  
공격 흐름 전체를 하나로 보고 대응할 수 있는 운영 구조입니다.

이 점에서 NS Solutions와 PLURA-XDR의 결합은  
매우 현실적인 의미를 가질 수 있습니다.

- NS Solutions는 일본 시장에서 축적한 인프라 구축 및 운영 경험을 보유하고 있습니다.
- PLURA-XDR은 웹, 계정, 서버, 데이터 유출, 포렌식을 하나의 흐름으로 연결하는 통합 정보보안 플랫폼입니다.
- 양측이 함께한다면, 일본 제조업과 공급망 고객에게 보다 설명하기 쉽고, 운영하기 쉬우며, 실제 대응까지 가능한 보안 모델을 제안할 수 있습니다.

본 제안은 크게 두 가지입니다.

**제안 1**은  
NS Solutions와 PLURA-XDR이 함께  
일본 제조업 및 공급망 고객에게 제안할 수 있는  
통합 보안 서비스 구조를 만드는 것입니다.

**제안 2**는  
이 구조를 바탕으로  
일본의 중견·중소 제조 고객까지 확장 가능한  
클라우드 기반 통합 정보보안 운영 모델을 제시하는 것입니다.

즉, 이 제안의 핵심은  
새로운 보안 기능 하나를 추가하자는 것이 아니라,

**일본 제조·공급망 고객이 실제로 운영 가능한 통합 보안 구조를 함께 만들자**는 것입니다.

## 1️⃣ 왜 지금 이 논의가 필요한가

### 1-1) 공급망 보안은 이제 일부 대기업만의 문제가 아닙니다

공급망 공격은  
가장 강한 기업을 정면으로 공격하기보다,  
연결된 조직 가운데 상대적으로 취약한 지점을 통해  
더 큰 목표로 확장되는 경우가 많습니다.

제조업 환경에서는 특히 다음 요소가 동시에 연결되어 있습니다.

- 외부 노출 웹 서비스
- 계정 및 인증 체계
- 생산 및 업무 시스템
- 협력사 및 외부 접속
- 내부 서버와 운영 환경
- 중요 데이터와 기술 정보

따라서 어느 한 지점의 약점도  
개별 사고로 끝나지 않고  
공급망 전체 신뢰 문제로 이어질 수 있습니다.

즉, 공급망 보안은  
더 이상 한 기업 내부의 IT 문제만이 아니라,  
연결된 전체 운영 체계의 문제입니다.

### 1-2) 일본의 중견·중소 기업은 보안 필요성과 운영 현실 사이의 간극이 큽니다

많은 기업이 보안의 중요성을 인식하고 있습니다.  
그러나 실제 운영 현장에서는 다음과 같은 제약이 반복됩니다.

- 전담 보안 인력이 충분하지 않음
- 여러 보안 제품을 각각 관리하기 어려움
- 경보는 많지만 실제 대응은 늦어짐
- 침해 이후 원인 분석과 증적 확보가 어렵음
- 중견·중소 기업일수록 운영 복잡도를 감당하기 어려움

이러한 현실에서는  
보안 수준을 높이기 위해 제품을 계속 추가하는 방식보다,  
운영 단순화와 대응 일관성을 함께 확보할 수 있는 구조가 더 중요합니다.

### 1-3) 신종 위협은 기존 운영 모델의 한계를 더 빠르게 드러내고 있습니다

최근 위협 환경에서 중요한 변화는  
공격이 더 빨라지고, 더 정교해지며, 더 자동화된다는 점입니다.

특히 미토스 AI와 같은 신종 위협이 상징하는 변화는  
단순히 새로운 공격 기법 하나가 늘어난 것이 아니라,  
취약점 탐색, 공격 변형, 우회 속도가 전반적으로 빨라질 수 있다는 점입니다.

이런 환경에서는  
시그니처 중심의 개별 보안 도구만으로는  
새로운 공격 흐름 전체를 충분히 따라가기 어렵습니다.

즉, 지금 필요한 것은  
“무엇을 더 도입할 것인가”보다  
“무엇을 하나의 흐름으로 연결해 운영할 것인가”에 대한 답입니다.

## 2️⃣ 고객이 실제로 원하는 것은 기능 목록이 아니라 대응 체계입니다

많은 제안서는  
웹방화벽, EDR, SIEM, 모니터링, 포렌식, 대응 기능을  
각각 나열하는 방식으로 구성됩니다.

하지만 실제 고객이 궁금해하는 것은 다음입니다.

- 실제 공격이 발생하면 얼마나 빨리 알 수 있는가
- 웹 공격이 서버 침해로 이어졌는지 연결해 볼 수 있는가
- 계정 공격과 데이터 유출 징후를 함께 볼 수 있는가
- 사고 이후 대응과 증적 확보까지 이어질 수 있는가
- 여러 제품을 따로 보는 것이 아니라 하나의 흐름으로 관리할 수 있는가

즉, 고객이 원하는 것은  
“어떤 기능이 있나”보다  
“실제 해킹에 어떻게 대응할 수 있나”입니다.

이 점에서  
통합 운영 체계가 없는 보안은  
기능은 많아도 대응은 분산될 가능성이 높습니다.

반대로  
웹, 계정, 서버, 데이터 유출, 대응, 포렌식이  
하나의 흐름으로 연결되어 있다면,  
운영자는 더 적은 콘솔과 더 적은 해석 비용으로도  
보다 빠른 판단과 조치를 할 수 있습니다.

## 3️⃣ 왜 NS Solutions와의 협력이 의미가 있는가

### 3-1) NS Solutions는 고객의 현실을 이해할 수 있는 위치에 있습니다

NS Solutions는  
단순한 제품 판매보다  
고객의 IT 인프라와 운영을 함께 고민해 온 위치에 있습니다.

이 점은 매우 중요합니다.

보안은 제품만으로 완성되지 않습니다.  
실제 고객 환경에 맞게 구축되고, 운영되고, 유지되어야 합니다.

특히 일본 제조업과 공급망 고객은  
단순한 기능 소개보다  
자신들의 운영 환경에 맞는 현실적인 구조를 원합니다.

이때 NS Solutions는  
고객이 어떤 방식으로 시스템을 사용하고,  
어떤 제약 속에서 운영하며,  
어떤 구조에서 보안을 받아들일 수 있는지를  
가장 잘 이해할 수 있는 파트너입니다.

### 3-2) NS Solutions에는 서비스형 보안 모델이 더 잘 어울립니다

일본 시장에서 특히 중요한 것은  
보안 제품을 하나 더 소개하는 방식보다  
운영 가능한 서비스 구조로 정리해 제안하는 방식입니다.

예를 들어 고객은 다음과 같은 구조를 더 쉽게 받아들일 수 있습니다.

- 클라우드 기반 통합 보안 서비스
- 웹과 서버를 함께 보는 관리형 보안 서비스
- 제조업 및 공급망 환경에 맞춘 보안 운영 패키지
- 중견·중소 고객도 적용 가능한 경량형 서비스 모델

즉, NS Solutions에게 필요한 것은  
제품 카탈로그 확대가 아니라  
고객에게 이해하기 쉬운 서비스형 통합보안 모델입니다.

## 4️⃣ 왜 PLURA-XDR이 적합한가

### 4-1) PLURA-XDR은 단일 기능형 솔루션이 아닙니다

PLURA-XDR은  
단순한 웹방화벽이나 단순한 EDR이 아닙니다.

핵심은 다음 기능을  
하나의 운영 체계로 연결한다는 데 있습니다.

- 웹 요청 분석
- 웹 응답 분석
- 계정 및 인증 이상행위 분석
- 서버 행위 분석
- 데이터 유출 탐지
- 자동 대응
- 포렌식 연계
- 웹-서버 상관분석
- MITRE ATT&CK 기준 위협 해석

즉, PLURA-XDR은  
개별 탐지 기능의 집합이 아니라  
공격 흐름 전체를 하나로 연결하는 통합 정보보안 플랫폼입니다.

### 4-2) 공격은 한 구간에서 끝나지 않기 때문에 통합 플랫폼이 필요합니다

오늘날 공격은 하나의 이벤트로 끝나지 않습니다.

예를 들면 다음과 같은 흐름이 이어질 수 있습니다.

1. 웹 공격 시도
2. 인증 우회 또는 계정 공격
3. 내부 서버 행위 발생
4. 데이터 수집 또는 유출 시도
5. 후속 확산 또는 랜섬웨어 실행
6. 사고 이후 포렌식과 원인 분석 필요

이 전체 흐름이 분리되어 있다면  
각 단계는 보일 수 있어도  
사건 전체를 이해하기 어렵습니다.

반대로 이를 하나의 흐름으로 연결하면  
공격의 의미와 우선순위를 더 빠르게 판단할 수 있습니다.

### 4-3) PLURA-XDR의 핵심 차별점

PLURA-XDR의 차별점은 다음과 같이 정리할 수 있습니다.

- **요청·응답 본문 기반 정밀 분석**
- **웹과 서버를 연결한 상관분석**
- **계정 공격과 데이터 유출 징후까지 함께 해석**
- **자동 대응과 포렌식까지 이어지는 운영 구조**
- **MITRE ATT&CK 기반 위협 해석과 설명 가능성**

즉, PLURA-XDR은  
알려진 공격을 막는 도구를 넘어  
실제 침해 흐름을 이해하고 대응하는 데 초점을 둔 플랫폼입니다.

## 5️⃣ 어떤 고객에게 가장 큰 의미가 있는가

### 5-1) 일본 제조업 고객

제조업은 다음 특성을 동시에 가집니다.

- 외부와 연결된 웹 서비스
- 복잡한 계정 체계
- 중요 업무 서버 및 내부 시스템
- 협력사와 연결된 운영 구조
- 민감한 기술 정보와 데이터

따라서 단일 장비 중심 보안보다  
공격 흐름 전체를 이해하는 체계가 더 중요합니다.

### 5-2) 공급망 전체를 관리해야 하는 고객

공급망형 보안에서는  
중심 기업 한 곳만 보호해서는 충분하지 않습니다.

협력사, 연결 기업, 지역 법인, 외부 운영 조직 등  
다양한 접점이 동시에 관리되어야 합니다.

이때 고객은  
복잡한 다중 도구보다  
표준화된 운영 모델을 원합니다.

PLURA-XDR 기반 구조는  
이러한 공급망 고객에게  
공통된 운영 원칙과 대응 모델을 제시하는 데 유리합니다.

### 5-3) 중견·중소 제조 고객

중견·중소 기업은  
보안 필요성은 높지만  
전담 인력과 복잡한 운영을 감당하기 어려운 경우가 많습니다.

이 고객군에게는  
여러 제품을 조합해 직접 운영하는 방식보다  
클라우드 기반 통합 정보보안 서비스가 더 현실적입니다.

즉, PLURA-XDR은  
대기업뿐 아니라  
중견·중소 제조 고객까지 포괄할 수 있는 구조를 가집니다.

## 6️⃣ NS Solutions와 PLURA-XDR이 함께 만들 수 있는 구조

### 6-1) 기본 방향

핵심은 대체가 아니라 결합입니다.

NS Solutions가 가진 구축·운영 역량 위에  
PLURA-XDR의 통합 보안 플랫폼을 결합함으로써,  
일본 제조업과 공급망 고객에게  
보다 경쟁력 있는 서비스형 보안 모델을 제시할 수 있습니다.

### 6-2) 제안 가능한 기본 구조

```text
외부 사용자 / 협력사 / 인터넷
        ↓
웹 서비스 / 인증 구간 / API 구간
        ↓
```

### 6-3) 운영 모델의 의미

이 구조의 장점은 다음과 같습니다.

* 웹과 서버를 따로 보지 않고 함께 볼 수 있음
* 계정 공격과 침해 이후 행위를 연결해 해석할 수 있음
* 데이터 유출 가능성과 후속 대응을 같은 흐름에서 볼 수 있음
* 자동 대응과 포렌식을 운영 프로세스에 포함할 수 있음
* 고객은 여러 제품 목록보다 하나의 운영 모델로 이해할 수 있음

즉, 제안의 핵심은  
기술 조합이 아니라  
실제 운영 가능한 구조를 만드는 데 있습니다.

## 7️⃣ 공동 사업 관점에서의 의미

### 7-1) 고객에게 더 설명하기 쉬운 구조가 됩니다

개별 솔루션을 각각 설명할수록  
고객은 무엇을 왜 선택해야 하는지 더 어려워집니다.

반면 NS Solutions와 PLURA-XDR을 함께 제안하면  
고객은 이를 기능 조합이 아니라  
하나의 통합 서비스로 이해할 수 있습니다.

예를 들어 고객은 다음과 같이 받아들일 수 있습니다.

* 제조업 대상 통합 보안 서비스
* 공급망 환경 대응형 클라우드 보안 서비스
* 웹-계정-서버-유출 대응을 묶은 관리형 보안 모델

### 7-2) 중견·중소 고객까지 확장 가능한 구조가 됩니다

복잡한 엔터프라이즈 보안 구조는  
중견·중소 고객에게 그대로 적용하기 어렵습니다.

하지만 클라우드 기반 통합 플랫폼 구조는  
상대적으로 적은 인력으로도 운영 가능하므로  
더 넓은 고객층으로 확장하기 쉽습니다.

### 7-3) 운영 단순화와 서비스 고도화를 함께 기대할 수 있습니다

고객 입장에서 중요한 것은  
도입 후 실제로 운영 가능한가입니다.

이 구조는 다음 가치를 제공합니다.

* 도입 항목 단순화
* 운영 화면 단순화
* 사건 해석 시간 절감
* 포렌식 및 증적 확보 체계화
* 대응 속도 향상
* 공급망 고객까지 확장 가능한 서비스 구조

즉, 이 제안은  
단순 판매 제안이 아니라  
운영 효율과 서비스 경쟁력을 함께 높이는 구조적 제안입니다.

* 자동 대응과 증적 확보까지 연계 가능

### 시나리오 C. 공급망형 장기 잠복 공격

* MITRE ATT&CK 기준으로 공격 단계를 해석

* 장기 잠복형 공격에 대해서도 체계적인 운영 모델 제공 가능
