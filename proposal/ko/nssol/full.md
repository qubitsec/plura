# PLURA-XDR 제안서
## NS Solutions와 PLURA-XDR 협력을 통한 일본 제조·공급망 보안 고도화 제안

---

## 제안의 출발점

미토스 AI와 같은 신종 위협이 등장하는 시대에,  
현재의 보안 운영 방식만으로 충분하다고 말할 수 있을까요?

일본의 제조업과 공급망 보안은  
앞으로 어떤 구조와 운영 모델 위에서 더 높은 신뢰를 가질 수 있을까요?

공격은 더 빨라지고, 더 정교해지고 있으며,  
특히 공급망으로 연결된 중견·중소 기업까지 포함하면  
기존의 개별 보안 제품 중심 운영만으로는  
전체 위험을 효과적으로 관리하기가 점점 어려워지고 있습니다.

이제 필요한 것은  
웹, 계정, 서버, 데이터 유출, 대응, 포렌식을  
하나의 흐름으로 연결할 수 있는 운영 체계입니다.

즉, 필요한 것은  
개별 보안 기능의 추가가 아니라  
클라우드 기반 통합 정보보안 플랫폼입니다.

NS Solutions는 인프라 구축과 운영 경험을 바탕으로,  
고객의 현실적인 과제를 가장 잘 이해할 수 있는 위치에 있습니다.

그리고 PLURA-XDR은  
웹 요청·응답 분석, 계정 이상행위 탐지, 서버 행위 분석,  
데이터 유출 탐지, 자동 대응, 포렌식을  
하나의 운영 체계로 연결할 수 있는 플랫폼입니다.

본 제안은  
이 두 역량의 결합을 통해  
일본의 제조업 및 공급망 고객에게  
보다 현실적이고 운영 가능한 통합 보안 모델을 제시하고자 하는 데 목적이 있습니다.

---

## 0️⃣ 제안 요약

본 제안의 핵심은 단순합니다.

일본의 제조업과 공급망 고객은  
더 이상 개별 보안 제품을 각각 도입하고 운영하는 방식만으로  
현실적인 대응 체계를 만들기 어렵습니다.

웹 공격은 계정 공격으로 이어지고,  
계정 공격은 서버 침해와 내부 확산으로 이어지며,  
그 이후에는 데이터 유출, 랜섬웨어, 장기 잠복형 공격으로 확장될 수 있습니다.

따라서 고객이 실제로 필요로 하는 것은  
보안 기능의 나열이 아니라  
공격 흐름 전체를 하나로 보고 대응할 수 있는 운영 구조입니다.

이 점에서 NS Solutions와 PLURA-XDR의 결합은  
매우 현실적인 의미를 가질 수 있습니다.

본 제안은 크게 두 가지입니다.

**제안 1**은  
NS Solutions와 PLURA-XDR이 함께  
일본 제조업 및 공급망 고객에게 제안할 수 있는  
통합 보안 서비스 구조를 만드는 것입니다.

**제안 2**는  
이 구조를 바탕으로  
일본의 중견·중소 제조 고객까지 확장 가능한  
클라우드 기반 통합 정보보안 운영 모델을 제시하는 것입니다.

즉, 이 제안의 핵심은  
새로운 보안 기능 하나를 추가하자는 것이 아니라,

**일본 제조·공급망 고객이 실제로 운영 가능한 통합 보안 구조를 함께 만들자**는 것입니다.

## 1️⃣ Why now
## 왜 지금 일본 제조·공급망 보안에 통합 대응 체계가 필요한가

### 1-1) 개별 보안 제품 중심 운영의 한계

공급망 공격은  
가장 강한 기업을 정면으로 공격하기보다,  
연결된 조직 가운데 상대적으로 취약한 지점을 통해  
더 큰 목표로 확장되는 경우가 많습니다.

제조업 환경에서는 특히 다음 요소가 동시에 연결되어 있습니다.

- 외부 노출 웹 서비스
- 계정 및 인증 체계
- 생산 및 업무 시스템
- 협력사 및 외부 접속
- 내부 서버와 운영 환경
- 중요 데이터와 기술 정보

따라서 어느 한 지점의 약점도  
개별 사고로 끝나지 않고  
공급망 전체 신뢰 문제로 이어질 수 있습니다.

### 1-2) 이제 필요한 것은 제품 추가가 아니라 운영 구조의 전환

웹 공격, 계정 공격, 내부 확산, 데이터 유출, 랜섬웨어, 장기 잠복형 공격은  
더 이상 서로 분리된 사건으로 보기 어렵습니다.

따라서 지금 필요한 것은  
개별 제품을 더 도입하는 방식이 아니라,  
공격 흐름 전체를 하나의 사건으로 보고 대응할 수 있는  
통합 운영 구조입니다.

## 2️⃣ Market signal
## Google-Wiz 사례가 보여주는 글로벌 시장의 방향

### 2-1) 클라우드와 보안은 이제 별도 시장이 아닙니다

글로벌 시장은 이미  
클라우드와 보안을 별도로 구매하는 구조보다,  
클라우드 환경 전반을 통합적으로 보호하는 플랫폼 구조로 이동하고 있습니다.

### 2-2) Google의 Wiz 인수는 강한 시장 신호입니다

Google의 Wiz 인수 사례는  
클라우드 보안이 더 이상 부가 기능이 아니라  
핵심 플랫폼 경쟁력으로 재편되고 있음을 보여주는 상징적 신호입니다.

### 2-3) 일본 시장에서도 같은 질문이 필요합니다

일본의 제조업과 공급망 고객에게도  
이제 필요한 것은  
개별 보안 기능의 추가보다  
클라우드 기반 통합 정보보안 플랫폼이 아닌지  
검토할 시점입니다.

## 3️⃣ Japan pain point
## 일본 중소·중견 기업 보안의 현실적 과제

### 3-1) 보안 필요성은 높지만 운영 여력은 충분하지 않습니다

많은 중견·중소 기업이 보안의 중요성을 인식하고 있지만,  
실제 운영 현장에서는 다음과 같은 제약이 반복됩니다.

- 전담 보안 인력이 충분하지 않음
- 여러 보안 제품을 각각 관리하기 어려움
- 경보는 많지만 실제 대응은 늦어짐
- 침해 이후 원인 분석과 증적 확보가 어려움
- 운영 복잡도가 높아질수록 보안 체계가 형식화되기 쉬움

### 3-2) 운영 인력과 체계의 부족은 구조적 문제입니다

이 문제는 단순히 제품이 부족해서가 아니라,  
운영 구조가 분산되어 있기 때문에 더 커집니다.

따라서 중견·중소 고객에게는  
복잡한 다중 도구보다  
보다 단순하고 일관된 통합 운영 모델이 중요합니다.

### 3-3) 운영 부담을 줄이는 보안 모델이 필요합니다

중견·중소 고객에게 현실적인 보안은  
기능을 더하는 방식이 아니라  
운영 부담을 줄이면서도 대응 수준을 높일 수 있는 구조여야 합니다.

따라서 경량 에이전트, 관리형 운영, 자동화 기반 탐지·대응,  
표준화된 운영 절차를 함께 제공하는 모델이 중요합니다.

## 4️⃣ Supply chain risk
## 닛산·혼다와 같은 사례가 보여주는 공급망 리스크

### 4-1) 공급망 보안 수준은 기업 신뢰 전체를 좌우합니다

공급망형 공격은  
대기업 한 곳의 방어력만으로 해결되지 않습니다.

연결된 협력사, 위탁사, 외부 운영 조직, 지역 법인 가운데  
어느 한 지점이 취약해도  
전체 신뢰와 운영 안정성에 영향을 줄 수 있습니다.

### 4-2) 제조업에서는 사고의 파급 범위가 더 큽니다

제조업과 공급망 환경은  
웹, 계정, 서버, 중요 정보, 운영 시스템이 긴밀히 연결되어 있기 때문에,  
침해 사고가 브랜드 신뢰, 공급 차질, 고객 신뢰 저하로까지 이어질 수 있습니다.

### 4-3) 따라서 공급망 보안은 확장 가능한 운영 모델이 중요합니다

공급망 전체를 고려하면  
보안은 본사만의 문제가 아니라  
연결된 조직 전체에 적용 가능한 운영 체계의 문제입니다.

원스톱형 역량을 강조하고 있습니다.

이 점은 보안을 제품 중심이 아니라

### 5-4) NS Solutions의 역할은 서비스 운영의 중심이 될 수 있습니다

이 구조에서 NS Solutions는  
고객 접점, 인프라 설계, 구축, 운영, 서비스 전달, 고객 성공 관리를 담당하는  
핵심 사업 주체가 될 수 있습니다.

즉, 이번 제안은  
단순 리셀링보다  
NS Solutions 주도의 서비스형 보안 모델을 함께 설계하는 방향에 가깝습니다.

## 6️⃣ Why PLURA-XDR
## 왜 PLURA-XDR이 통합 보안 플랫폼으로 적합한가

### 6-1) PLURA-XDR의 핵심 개념

PLURA-XDR은  
단순한 웹방화벽이나 단순한 EDR이 아닙니다.

핵심은 다음 요소를  
하나의 운영 체계로 연결한다는 데 있습니다.

- 웹 요청 분석
- 웹 응답 분석
- 계정 및 인증 이상행위 분석
- 서버 행위 분석
- 데이터 유출 탐지
- 자동 대응
- 포렌식 연계
- 웹-서버 상관분석
- MITRE ATT&CK 기준 위협 해석

### 6-2) 제안 가능한 기본 구조

```text
외부 사용자 / 협력사 / 인터넷
        ↓
웹 서비스 / 인증 구간 / API 구간
        ↓
PLURA-Web
- 요청·응답 분석
- 이상 행위 탐지
- 계정 공격 및 비정상 흐름 분석
        ↓
PLURA-XDR
- 웹 이벤트 상관분석
- 인증 및 계정 이상행위 분석
- MITRE ATT&CK 기준 위협 해석
- 운영자 중심 사건 단위 분석
        ↓
PLURA-EDR
- 서버 프로세스/파일/네트워크 행위 분석
- 내부 확산 및 후속 행위 탐지
- 랜섬웨어/APT 전 단계 행위 탐지
        ↓
자동 대응 / 알림 / 포렌식 / 운영 보고
````

### 6-3) 공격 흐름을 하나로 연결하는 플랫폼

오늘날 공격은 하나의 이벤트로 끝나지 않습니다.

예를 들면 다음과 같은 흐름이 이어질 수 있습니다.

1. 웹 공격 시도
2. 인증 우회 또는 계정 공격
3. 내부 서버 행위 발생
4. 데이터 수집 또는 유출 시도
5. 후속 확산 또는 랜섬웨어 실행
6. 사고 이후 포렌식과 원인 분석 필요

이 전체 흐름이 분리되어 있다면  
각 단계는 보일 수 있어도  
사건 전체를 이해하기 어렵습니다.

반대로 이를 하나의 흐름으로 연결하면  
공격의 의미와 우선순위를 더 빠르게 판단할 수 있습니다.

### 6-4) PLURA-XDR의 차별점

PLURA-XDR의 차별점은 다음과 같이 정리할 수 있습니다.

- 요청·응답 본문 기반 정밀 분석
- 웹과 서버를 연결한 상관분석
- 계정 공격과 데이터 유출 징후까지 함께 해석
- 자동 대응과 포렌식까지 이어지는 운영 구조
- MITRE ATT&CK 기반 위협 해석과 설명 가능성

### 6-5) 경쟁 제품 대비 PLURA-XDR의 설명 포인트

일반적인 XDR이  
주로 엔드포인트와 계정 중심의 상관분석에 강점을 갖는다면,  
PLURA-XDR은 웹 요청·응답, 계정 흐름, 서버 행위, 데이터 유출 징후를  
하나의 사건 흐름으로 연결해 설명할 수 있다는 점에서 차별화됩니다.

특히 제조업과 공급망 환경에서는  
웹에서 시작된 공격이 계정 탈취, 서버 침해, 중요 정보 접근으로 이어질 수 있기 때문에,  
웹 구간까지 포함한 통합 가시성이 중요합니다.

### 6-6) 일본 시장 관점에서 필요한 보강 포인트

일본 시장에서는 단순 기능 설명보다  
실제 도입 사례, 운영 성과, 고객 반응, 현지 지원 가능성,  
일본어 지원과 운영 편의성 같은 요소가 함께 제시될 때  
신뢰도가 더 높아질 수 있습니다.

따라서 향후 제안 단계에서는  
PoC 결과, 운영 사례, 고객 적용 성과, 현지화 지원 계획 등을  
함께 제시하는 것이 바람직합니다.

## 7️⃣ Business model
## 일본 제조 고객 대상 Managed Security / Cloud Security Service 패키지화 방향

### 7-1) 고객에게는 기능 조합보다 서비스 패키지가 더 이해하기 쉽습니다

개별 솔루션을 각각 설명할수록  
고객은 무엇을 왜 선택해야 하는지 더 어려워집니다.

반면 NS Solutions와 PLURA-XDR을 함께 제안하면  
고객은 이를 기능 조합이 아니라  
하나의 통합 서비스로 이해할 수 있습니다.

### 7-2) Managed Security / Cloud Security Service 모델로 정리할 수 있습니다

예를 들어 다음과 같은 패키지 방향을 검토할 수 있습니다.

- 제조업 대상 통합 보안 서비스
- 공급망 환경 대응형 클라우드 보안 서비스
- 웹-계정-서버-유출 대응을 묶은 관리형 보안 모델
- 중견·중소 제조 고객용 경량형 서비스 모델
- 글로벌 거점 및 협력사 확장형 서비스 모델

### 7-3) 역할 분담 구조를 명확히 할 필요가 있습니다

실행 가능성을 높이기 위해서는  
양측의 역할을 보다 선명하게 정리할 필요가 있습니다.

예를 들면 다음과 같은 구조를 검토할 수 있습니다.

- NS Solutions: 고객 창구, 인프라 설계, 구축, 운영, 서비스 제공
- PLURA-XDR: 플랫폼, 탐지·분석 엔진, 위협 분석 기술, 대응 체계
- 공동 운영: 파일럿, 운영 기준 정립, 고도화, 공동 고객 제안

### 7-4) 수익 모델과 서비스 브랜드도 함께 설계할 수 있습니다

이번 협력은  
단순 제품 납품보다  
서비스 패키지 형태로 발전할 가능성이 큽니다.

따라서 향후에는 다음 요소도 함께 검토할 수 있습니다.

- 서비스 패키지 이름
- 과금 구조
- 고객 규모별 상품 구분
- 매출 분담 모델
- 공동 제안 및 공동 마케팅 방식

### 7-5) 단계적 적용과 사업 확장이 가능합니다

이 구조의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다.

작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 방식으로  
서비스 모델을 현실화할 수 있습니다.

### 7-6) 공동 사업 관점의 기대 효과

이 구조는 다음 가치를 제공합니다.

- 고객 설명력 향상
- 운영 단순화
- 사건 해석 시간 절감
- 서비스 포트폴리오 차별화
- 중견·중소 고객까지 확장 가능한 구조 확보
- 공급망 고객 대응력 강화

## 8️⃣ 시나리오

### 8-1) 계정 공격과 비정상 로그인 흐름

**기존 구조**

- 인증 실패와 웹 로그가 분산 기록됨
- 공격과 사용자 오류를 구분하기 어려움
- 차단 여부 판단이 늦어질 수 있음

**통합 구조**

- 요청 흐름, 인증 결과, 세션 패턴을 함께 분석
- 자동화된 계정 공격 징후를 더 빠르게 식별
- 운영자는 사건 단위로 판단 가능

### 8-2) 웹 침투 후 서버 행위 발생

**기존 구조**

- 웹 이벤트와 서버 이벤트가 분리되어 보임
- 실제 침해 여부를 해석하는 데 시간이 걸림

**통합 구조**

- 웹 이벤트와 서버 행위를 하나의 흐름으로 연결
- 침해 이후 행위를 조기에 식별
- 자동 대응과 증적 확보까지 연계 가능

### 8-3) 공급망형 장기 잠복 공격

**기존 구조**

- 단편 이벤트는 보이나 전체 흐름은 이해하기 어려움
- 운영자 숙련도에 따라 대응 품질 차이가 큼

**통합 구조**

- MITRE ATT&CK 기준으로 공격 단계를 해석
- 계정, 서버, 유출 흐름을 하나의 사건으로 설명 가능
- 장기 잠복형 공격에 대해서도 체계적인 운영 모델 제공 가능

## 9️⃣ 단계적 적용 방식 제안

이 제안의 장점은  
처음부터 전체 전환을 요구하지 않는다는 점입니다.

핵심은  
작게 시작하고, 빠르게 검증하고, 단계적으로 확대하는 것입니다.

### 9-1) 1단계: 파일럿 적용

- 인터넷 노출 웹 서비스 1개 선정
- 로그인 또는 핵심 API 구간 1개 선정
- 관련 서버 일부 적용
- 웹-계정-서버 연계 가시성 확인

### 9-2) 2단계: 운영 시나리오 검증

- 계정 공격 대응 시나리오 검증
- 웹 침투 후 서버 행위 분석 검증
- 데이터 유출 징후 탐지 검증
- 자동 대응 및 알림 흐름 검증

### 9-3) 3단계: 서비스 모델화

- 고객 설명용 패키지 정의
- 중견·중소 제조 고객용 경량 모델 정리
- 공급망 고객용 확장 모델 정리
- 운영팀/SOC 기준 대응 절차 정리

### 9-4) 4단계: 확대 적용

- 고객군 확대
- 공급망 고객 모델 확장
- 관리형 서비스 형태 정교화
- 장기적인 공동 사업 구조 검토

## 결론

미토스 AI와 같은 신종 위협이 등장하는 시대에,  
현재의 보안 운영 방식만으로 충분하다고 말할 수 있을까요?

일본의 제조업과 공급망 보안은  
앞으로 어떤 구조와 운영 모델 위에서 더 높은 신뢰를 가질 수 있을까요?

이 질문에 대한 답은  
더 많은 개별 보안 제품을 추가하는 데 있지 않습니다.

필요한 것은  
웹, 계정, 서버, 데이터 유출, 대응, 포렌식을  
하나의 흐름으로 연결할 수 있는  
클라우드 기반 통합 정보보안 플랫폼입니다.

그리고 그 플랫폼은  
고객의 현실을 이해하는 구축·운영 역량과  
실제 침해 흐름을 해석하고 대응할 수 있는 보안 플랫폼이  
함께 결합될 때 더 큰 설득력을 가질 수 있습니다.

이 점에서  
NS Solutions와 PLURA-XDR의 협력은  
일본 제조업과 공급망 고객에게  
보다 현실적이고 운영 가능한 통합 보안 모델을 제시할 수 있는  
의미 있는 방향이 될 수 있습니다.

즉, 본 제안의 핵심은  
새로운 솔루션 하나를 추가하자는 것이 아니라,

**일본의 제조업과 공급망 고객이  
실제로 신뢰하고 운영할 수 있는 보안 구조를 함께 만들자**는 것입니다.

4. Supply chain risk  
   닛산·혼다 같은 사례는 공급망 전체 보안 수준이 기업 신뢰를 좌우함

5. Why NS Solutions  
   NS Solutions는 인프라 구축·운영·클라우드·글로벌 고객 지원 역량이 있어 서비스화에 유리

6. Why PLURA-XDR  
   웹·계정·서버·유출을 하나의 흐름으로 연결하는 클라우드형 통합 정보보안 플랫폼

7. Business model  
   일본 제조 고객 대상 Managed Security / Cloud Security Service로 패키지화 가능
