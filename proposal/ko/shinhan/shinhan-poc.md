# 신한은행 해킹대응 솔루션 PoC 결과 보고서 요약

## 0️⃣ 요약

**PoC 결론:**  
신한은행 환경에서 진행된 PoC를 통해, PLURA는 웹 패킷 분석 기반으로 기존 침입차단시스템(IPS) 보완 가능성을 확인하였고, 크리덴셜 스터핑 공격 탐지와 해킹(APT) 공격 진행 상황의 시각화 제공 측면에서도 유의미한 효과를 입증하였습니다.

특히 PoC 결과 보고가 진행되던 중 실제 웹 공격이 탐지되었고, 해당 과정에서 기존 운영 체계의 대응 한계와 함께, PLURA의 추가 가시성과 분석 가치가 현장에서 확인되었습니다.

이번 PoC를 통해 확인된 핵심은 다음 4가지입니다.

**1. 웹 패킷 분석 기반의 추가 가시성은 기존 IPS 보완에 실질적으로 유효합니다.**  
기존 보안 장비만으로는 놓치기 쉬운 공격 내역과 행위를 보다 상세히 확인할 수 있었습니다.

**2. 크리덴셜 스터핑과 같은 자동화 공격에 대해, 공격자 IP 및 행위 단위의 식별이 가능함을 확인하였습니다.**  
이는 단순 차단 여부를 넘어, 공격의 실체를 보다 명확히 파악할 수 있음을 의미합니다.

**3. 공격 진행 상황을 대시보드 형태로 시각화하여, 이상징후와 공격 흐름을 직관적으로 파악할 수 있음을 확인하였습니다.**  
이는 실무 대응과 보고 체계 모두에 유용한 기반이 됩니다.

**4. 실제 공격 상황에서 기존 운영 체계의 대응 지연 및 한계가 드러났으며, 이에 따라 추가적인 탐지·분석 체계 도입 필요성이 확인되었습니다.**

## 2️⃣ PoC 추진 배경과 의미

PoC 킥오프 미팅에서 신한은행 측은, 정보보안 관련 다단계 시스템을 구축·운영하고 있음에도 외부 공격에 대한 지속적인 위협이 존재하며, 이에 대한 상시적 경계와 대비가 필요하다는 점을 강조하였습니다. :contentReference[oaicite:3]{index=3}

이러한 배경에서, 인터넷 및 모바일 뱅킹 구간의 패킷을 분석하여 해킹 공격을 탐지하고, 이를 통해 기존 침입차단시스템(IPS)을 보완할 수 있는지를 확인하는 것이 이번 PoC의 핵심 목적이었습니다.

즉, 이번 PoC는 단순 기능 시연이 아니라, 실제 금융권 운영 환경에서 다음 질문에 답하는 검증 과정이었습니다.

- 기존 보안 체계를 보완할 수 있는가
- 자동화 공격과 지능형 공격을 보다 잘 식별할 수 있는가
- 보안 운영자가 공격의 진행 흐름을 더 빠르고 명확하게 파악할 수 있는가

## 3️⃣ PoC 결과 요약

2023년 11월 14일 진행된 PoC 결과 보고에서, 신한은행과 큐비트시큐리티는 당초 설정한 3가지 목표에 대해 다음과 같은 결과를 확인하였습니다. :contentReference[oaicite:4]{index=4}

### 3-1) 웹 패킷 분석 기반 IPS 보완 가능성 확인

PoC 결과, 웹 패킷 분석을 통해 공격 탐지 및 상세 내역 제공이 가능함을 확인하였으며, 이를 통해 기존 침입차단시스템(IPS)의 보완재로서 타당성이 입증되었습니다.

이는 기존 차단 중심 장비만으로는 충분히 확보하기 어려운, 공격 맥락과 상세 로그 수준의 가시성을 확보할 수 있음을 의미합니다.

### 3-2) 크리덴셜 스터핑 공격 탐지 가능성 확인

모의 해킹 기반 크리덴셜 스터핑 공격에 대해, 공격자 IP 주소를 포함한 주요 행위를 확인함으로써 탐지 가능성을 입증하였습니다.

이는 단순히 “이상 트래픽이 있었다”는 수준이 아니라, 실제 공격자를 특정하고 공격 시도를 식별할 수 있는 분석 기반이 마련된다는 점에서 의미가 있습니다.

### 3-3) 공격 진행 상황의 시각화 제공 가능성 확인

APT 공격 진행 상황을 MITRE ATT&CK 매트릭스 등으로 표현하여, 이상징후 탐지와 공격 흐름 파악이 가능한 대시보드 제공 가능성을 확인하였습니다.

특히 신한은행 측에서는 이 부분과 관련하여 가시성 데이터의 추가 작성과 보완을 요청하였으며, 이는 단순 탐지 기능을 넘어 운영·보고 체계까지 고려한 활용 가능성을 보여줍니다.

## 4️⃣ 실제 공격 탐지 사례와 운영상 시사점

PoC 결과 보고가 진행되던 2023년 11월 14일 오후, PoC 대상 사이트인 `bank.shinhan.com` 구간에서 실제 웹 공격이 탐지되었습니다. :contentReference[oaicite:5]{index=5}

당시 상황은 다음과 같이 정리할 수 있습니다.

- 웹 공격은 결과 보고가 진행되던 시점에 탐지되었습니다.
- 공격은 주로 SQL 인젝션 유형으로 파악되었으며, 빠른 대응이 필요한 심각한 공격으로 전달되었습니다.
- 공격 심각성은 즉시 공유되었으나, 실제 차단까지는 시간이 소요되었습니다.
- 최종적으로 웹 공격은 오후 4시 53분경 차단되었습니다.

이 사례는 단순한 탐지 이벤트가 아니라, 실제 운영 환경에서 다음과 같은 문제를 드러낸 사례로 볼 수 있습니다.

**첫째, 공격은 탐지되었지만 즉시 차단으로 이어지지 않을 수 있습니다.**  
실제 운영에서는 보고 체계, 승인 절차, 조직 간 지시 체계가 개입되므로 대응 지연이 발생할 수 있습니다.

**둘째, 기존 체계만으로는 공격의 심각성과 지속성을 충분히 구조적으로 보여주기 어려울 수 있습니다.**  
즉, 단순 차단 로그보다 더 넓은 가시성과 상관분석이 필요합니다.

**셋째, 금융권과 같이 민감한 서비스 환경에서는 ‘탐지’와 ‘운영 판단 지원’이 함께 가야 합니다.**  
보안 시스템은 단순히 알림을 발생시키는 것이 아니라, 공격 규모와 진행 양상을 빠르게 이해할 수 있도록 도와야 합니다.

## 6️⃣ PoC를 통해 확인된 핵심 의미

이번 PoC를 통해 확인된 핵심 의미는 다음과 같습니다.

### 6-1) 기존 보안 체계의 공백 보완 필요성 확인

실제 공격이 장시간 지속되었음에도 운영상 즉시 차단되지 못한 사례는, 기존 보안 운영 체계에 공백이 존재할 수 있음을 보여주었습니다.

이는 솔루션의 단순 유무가 아니라, 탐지 정보의 품질과 운영 활용성의 차이에서 비롯되는 문제일 수 있습니다.

### 6-2) 웹 패킷 분석 기반 보안의 실효성 확인

웹 패킷 분석을 통해 기존 IPS 보완재로서의 가능성이 확인되었습니다.  
이는 금융 서비스와 같이 웹 기반 공격이 집중되는 환경에서 의미가 큽니다.

### 6-3) PLURA의 차별성 확인

PLURA는 단순 차단 장비가 아니라, 전체 로그·탐지 로그·사후 탐지 로그를 포함한 다층적 분석 정보를 제공함으로써 공격의 실체를 보다 구체적으로 파악할 수 있게 합니다.

### 6-4) 가시성 중심 운영의 필요성 확인

금융 보안 운영에서는 “막았는가”만으로 충분하지 않습니다.  
“어떤 공격이, 어떤 흐름으로, 어느 수준까지 진행되었는가”를 빠르게 이해할 수 있어야 하며, 이번 PoC는 이러한 가시성 기반 운영의 필요성을 확인한 과정이었습니다.

## 7️⃣ 제안 방향

신한은행과 같은 금융권 환경에서는, 기존 보안 장비의 차단 기능만으로 모든 공격에 대응하기 어렵습니다.  
특히 웹 서비스 구간에서는 정상 트래픽과 유사한 형태의 공격, 크리덴셜 스터핑, SQL 인젝션, 지능형 공격 흐름 등에 대해 보다 정교한 탐지와 분석이 필요합니다.

이에 따라 향후 방향은 다음과 같이 정리할 수 있습니다.

### 7-1) 웹 구간 전체 가시성 강화

웹 요청과 응답에 대한 충분한 분석 데이터를 확보하여, 단순 차단 여부를 넘어 공격의 실제 흐름을 파악할 수 있어야 합니다.

### 7-2) 탐지와 차단 사이의 운영 연결 강화

공격 탐지 이후 실제 차단까지의 운영 절차를 단축하고, 심각도 판단과 대응 우선순위를 빠르게 정할 수 있는 체계가 필요합니다.

### 7-3) 대시보드 기반 상황 인식 강화

APT 및 이상행위의 진행 상황을 시각화하여, 보안 운영자가 단일 이벤트가 아니라 전체 공격 흐름을 입체적으로 파악할 수 있어야 합니다.

### 7-4) 기존 보안 체계 보완형 도입 검토

PLURA는 기존 보안 체계를 대체하기보다, 부족한 가시성과 분석 능력을 보완하는 방향으로 검토할 수 있습니다.  
이는 금융권 환경에서 현실적인 도입 방식이 될 수 있습니다.

---

## 8️⃣ 한 문장 정리

> 이번 신한은행 PoC는 웹 패킷 분석 기반의 추가 가시성이 기존 IPS 중심 운영의 한계를 보완할 수 있음을 보여주었고, 크리덴셜 스터핑 탐지, 공격 흐름 시각화, 실제 공격 상황에서의 운영 판단 지원 측면에서 PLURA의 실효성을 확인한 검증 과정이었습니다.

---

## 9️⃣ 최종 결론

이번 PoC는 단순한 기능 검증을 넘어, 실제 금융 서비스 환경에서 다음 사항을 확인한 의미 있는 사례였습니다.

- 웹 패킷 분석 기반 보안의 실효성
- 크리덴셜 스터핑 공격 탐지 가능성
- APT 공격 흐름 시각화 가능성
- 실제 공격 상황에서 기존 운영 체계의 대응 한계
- PLURA의 차별적 분석 가치와 도입 필요성

즉, 신한은행 환경에서 PLURA는 기존 침입차단시스템(IPS)을 보완하는 실질적인 보안 분석 체계로서 검토 가능한 수준임이 확인되었습니다.
