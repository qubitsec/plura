# 1. 왜 지금 서울교육청 사이버안전 대전환인가

학교는 더 이상 칠판과 교실만으로 운영되지 않습니다.  
출결, 성적, 생활기록, 급식, 상담, 학부모 소통, 원격수업, AI 디지털교과서, 에듀테크 서비스까지 교육의 많은 영역이 디지털 시스템 위에서 운영되고 있습니다.

그만큼 교육 분야의 사이버공격은 단순한 전산 장애가 아닙니다.  
학생 개인정보 유출, 교육행정 업무 차질, 학교 홈페이지 변조, 교직원 계정 탈취, 학부모 민원 시스템 장애, 디지털 수업 중단으로 이어질 수 있는 교육 안전사고입니다.

서울교육청 사이버안전 정책은 중앙 교육행정정보시스템과 경쟁하거나 이를 대체하려는 정책이 아닙니다.  
중앙 시스템은 교육부·KERIS와 협력하고, 서울교육청은 학교 홈페이지, 교직원 계정, 교육청 업무시스템, 에듀테크, AI 교육서비스, 외부 연동 영역처럼 학교 현장에서 실제로 발생하는 보안 공백을 책임 있게 보완하겠습니다.

특히 교육기관은 다음과 같은 특성을 가지고 있습니다.

| 교육청·학교 특성 | 보안상 위험 |
|---|---|
| 중앙 시스템과 학교 현장 시스템 병행 사용 | 권한·계정·외부 연동 관리 미흡 위험 |
| 교사·행정직원 계정 다수 | 계정 탈취와 권한 오남용 위험 |
| 학교별 홈페이지 운영 | 웹 취약점과 악성코드 유포 위험 |
| 다양한 에듀테크 사용 | 외부 서비스와 공급망 보안 위험 |
| 학부모 소통 시스템 운영 | 계정 도용, 피싱, 사칭 위험 |
| 디지털교과서·AI 교육 확대 | 학생 데이터 처리와 AI 보안 위험 |
| 학교 현장 보안 인력 부족 | 사고 대응 지연과 책임 전가 위험 |
| 방학·입학·수능 등 특정 시기 집중 | 공격자가 노리기 쉬운 시기 발생 |

교육청의 사이버보안은 서울시청과 다릅니다.  
서울시청처럼 모든 기관을 단순히 하나로 묶는 방식만으로는 어렵습니다.  
학교는 수가 많고, 현장의 정보화 수준도 다르며, 교사와 행정직원이 보안 업무까지 감당하기 어렵습니다.

따라서 서울교육청의 사이버안전 정책은 다음 방향이어야 합니다.

```text
학교가 알아서 막는 보안이 아니라
교육청이 기준을 만들고, 플랫폼을 제공하고, 사고 대응을 지원하는 보안
```

학교 현장에 책임을 떠넘기지 않겠습니다.  
교사와 행정직원이 복잡한 보안장비를 직접 운영하지 않아도, 교육청이 표준 기준과 통합 지원 체계를 통해 학교를 보호하겠습니다.

학생 개인정보는 한 번 유출되면 회복이 어렵습니다.  
미성년자의 개인정보, 학습 이력, 상담 기록, 생활기록, 건강 정보, 보호자 정보는 일반 행정정보보다 더 조심스럽게 보호되어야 합니다. 학생의 데이터는 단순한 행정자료가 아니라, 학생의 현재와 미래에 영향을 줄 수 있는 보호 대상입니다.

이제 서울교육청은 사이버보안을 단순 전산관리 업무가 아니라, 학생 안전과 교육권을 지키는 **교육 사이버안전 정책**으로 전환해야 합니다.  
교사는 보안 전문가가 아니라 교육 전문가로 일할 수 있어야 하고, 행정직원은 복잡한 보안 책임을 혼자 떠안지 않아야 합니다.

## 3. 학교 홈페이지와 웹서비스 보안 강화

하지만 학교 홈페이지는 공격자에게도 쉬운 표적이 될 수 있습니다.

서울교육청은 학교 홈페이지와 웹서비스를 교육청 차원에서 보호하겠습니다.

| 분야 | 내용 |
|---|---|
| 학교 홈페이지 보안점검 | 취약점, 관리자 계정, 파일 업로드, 게시판 보안 점검 |
| 웹방화벽 지원 | 학교 웹서비스에 대한 웹 공격 탐지·차단 지원 |
| 웹 제로데이 대응 | 알려진 규칙이 없는 이상 요청과 비정상 응답 탐지 |
| 관리자 계정 보호 | 관리자 로그인, 비밀번호, 다중인증, 접속 위치 관리 |
| 게시물 변조 대응 | 홈페이지 변조, 악성 링크 삽입, 피싱 페이지 생성 탐지 |

## 5. AI 디지털교과서와 에듀테크 보안 검증 강화

| 삭제 권리 | 계약 종료 후 학생 데이터 삭제 가능 여부 확인 |

- 학생 데이터 처리 기준 표준화

## 8. 긴급 상황 선조치·후보고 체계 도입

사이버공격은 보고 절차를 기다려주지 않습니다.  
학생 개인정보 유출, 교육행정 업무 차질, 학교 홈페이지 악성코드 유포가 의심되는 경우에는 빠른 조치가 필요합니다.

서울교육청은 긴급 상황에서 피해 확산을 막기 위해 **선조치·후보고** 체계를 마련하겠습니다.  
다만 선조치가 권한 남용이나 수업 방해로 이어지지 않도록 지정 책임자 승인, 조치 이력 기록, 사후 검증 절차를 함께 두겠습니다.

| 항목 | 내용 |
|---|---|
| 악성 IP 차단 | 학교 시스템 공격 출발지 차단 |
| 위험 계정 잠금 | 탈취 의심 교직원·관리자 계정 임시 정지 |
| 학교 홈페이지 임시 보호 | 악성코드 유포·변조 의심 시 임시 차단 |
| 위험 세션 종료 | 비정상 접속 세션 차단 |
| 관할 시스템 접근 제한 | 서울교육청 관할 시스템의 대량 조회·다운로드 의심 시 임시 제한 |
| 중앙 시스템 협력 요청 | 중앙 시스템 관련 이상징후는 교육부·KERIS와 즉시 협력 요청 |
| 긴급 패치 | 취약점 악용 가능성이 높은 시스템 우선 패치 |
| 로그 보존 | 사고 원인 분석을 위한 기록 보존 |
| 에듀테크 연동 중지 | 외부 서비스 유출 의심 시 연동 임시 중단 |

### 승인과 사후 검증 기준

| 구분 | 기준 |
|---|---|
| 긴급 조치 승인 | 교육감이 지정한 사이버안전 책임자 또는 교육지원청 책임자가 승인 |

| 상세 보고 | 조치 후 72시간 이내 |
| 재발 방지 보고 | 7일 이내 |
| 학생·학부모 안내 | 개인정보 또는 수업 영향 발생 시 별도 안내 |
| 사후 검증 | 교육감 직속 사이버안전 자문위원회 또는 지정 검증팀이 적정성 확인 |

정해진 기준에 따라 학생 피해를 막기 위해 조치한 담당자에게는 무리한 책임 전가보다 사후 검증과 제도 개선을 우선하겠습니다.  
다만 조치 권한이 남용되지 않도록 승인권자, 기록, 검증 절차를 명확히 하겠습니다.

학교 홈페이지 변조, 악성 파일 업로드, 관리자 계정 탈취, 학부모 대상 피싱을 교육청 차원에서 막겠습니다.

서울교육청 관할 시스템의 다중인증, 휴면 계정 정리, 전보·퇴직자 권한 회수, 비정상 로그인 탐지를 강화하고, 중앙 시스템 권한관리 개선 사항은 교육부·KERIS와 협의하겠습니다.

학생 개인정보 유출이나 학교 시스템 피해가 우려되는 경우, 서울교육청 관할 영역은 필요한 조치를 먼저 하고, 중앙 시스템 관련 사안은 교육부·KERIS와 즉시 협력하는 체계를 만들겠습니다.

일곱째, 교육감 직속 사이버안전 자문위원회를 운영하겠습니다.  
전산 담당자만의 보안이 아니라, 학생 개인정보, 학교 현장, 에듀테크, AI 보안, 학부모 관점이 함께 반영되는 정책을 만들겠습니다.

여덟째, 학생·교사·학부모 디지털 안전교육을 강화하겠습니다.  
딥페이크, 사이버폭력, 피싱, 개인정보 유출로부터 학생을 보호하는 실질적인 교육을 하겠습니다.

저는 학생 개인정보와 학교 현장을 지키는 교육청을 만들겠습니다.

디지털 교육은 신뢰 위에서만 지속될 수 있습니다.

**학생을 지키는 보안, 학교를 지원하는 교육청, 학부모가 안심하는 디지털 교육환경을 만들겠습니다.**

**학교 홈페이지와 웹서비스를 교육청 차원에서 보호하겠습니다. 홈페이지 변조, 악성 파일 업로드, 관리자 계정 탈취, 학부모 대상 피싱을 막겠습니다.**

**AI 디지털교과서와 에듀테크 서비스는 도입 전부터 보안과 개인정보 보호를 검증하겠습니다. 학생 데이터가 안전하게 사용되고 삭제되는지 끝까지 관리하겠습니다.**

**교직원 계정 보호를 강화하겠습니다. 중앙 시스템 권한관리 개선은 교육부·KERIS와 협력하고, 학교 홈페이지와 교육청 업무시스템 계정은 서울교육청이 직접 점검하겠습니다.**

**긴급 상황에서는 선조치·후보고가 가능하도록 하겠습니다. 다만 지정 책임자 승인, 조치 이력 기록, 사후 검증을 통해 학생 피해를 막으면서도 권한 남용을 방지하겠습니다.**

**교육감 직속 사이버안전 자문위원회를 운영하겠습니다. 학생 개인정보, 학교 현장, 에듀테크, AI 보안 전문가가 함께 교육청 사이버안전 정책을 점검하겠습니다.**

**딥페이크, 사이버폭력, 피싱, 개인정보 유출로부터 학생을 보호하는 디지털 안전교육을 강화하겠습니다. 한 번 유출되면 돌이킬 수 없는 학생의 미래를 지키겠습니다.**

**연차별 로드맵과 성과지표를 공개 가능한 범위에서 점검하겠습니다. 선언으로 끝나는 공약이 아니라, 학교 현장의 부담이 실제로 줄어드는지 확인하겠습니다.**

**학교 현장의 부담은 줄이고 교육청의 책임은 높이겠습니다. 중앙 교육행정정보시스템은 교육부·KERIS와 협력하고, 서울교육청은 학교 홈페이지·교직원 계정·에듀테크·AI 교육서비스·외부 연동 보안을 책임지는 서울형 교육 사이버안전 체계를 만들겠습니다.**
