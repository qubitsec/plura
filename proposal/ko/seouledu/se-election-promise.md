# 1. 왜 지금 서울교육청 사이버안전 대전환인가

학교는 더 이상 칠판과 교실만으로 운영되지 않습니다. 출결, 성적, 생활기록, 급식, 상담, 학부모 소통, 원격수업, 학교 홈페이지, AI·디지털 교육자료, 학습지원 소프트웨어, 에듀테크 서비스까지 교육의 많은 영역이 디지털 시스템 위에서 운영되고 있습니다.

그만큼 교육 분야의 사이버공격은 단순한 전산 장애가 아닙니다. 학생 개인정보 유출, 교육행정 업무 차질, 학교 홈페이지 변조, 교직원 계정 탈취, 학부모 대상 피싱, 디지털 수업 중단으로 이어질 수 있는 **교육 안전사고**입니다.

서울교육청 사이버안전 정책은 나이스 등 전국 공동 교육정보시스템을 서울교육청이 단독으로 대체하려는 정책이 아닙니다. 전국 공동 교육정보시스템은 교육부·KERIS·시도교육청 협력체계와 법령상 권한에 맞춰 개선하고, 서울교육청은 학교 현장에서 실제로 빈틈이 생기기 쉬운 영역을 보완하겠습니다.

특히 서울교육청은 이미 보안관제, 침해대응, 개인정보 점검, 서울교육 웹서비스 및 학교누리집 통합 유지관리 등 다양한 정보보호 업무를 운영하고 있습니다. 이 공약은 기존 체계를 부정하거나 새로 포장하는 것이 아니라, 기존 체계를 학교 현장 지원형으로 고도화하고, AI·디지털 교육 확대에 맞춰 보안 사각지대를 줄이는 정책입니다.

## 교육청·학교의 디지털 위험

| 교육청·학교 특성 | 보안상 위험 |
|---|---|
| 전국 공동 교육정보시스템과 학교 현장 시스템 병행 사용 | 권한·계정·외부 연동 관리 공백 |
| 교사·행정직원 계정 다수 | 계정 탈취, 권한 오남용, 피싱 피해 |
| 학교별 게시·관리 업무와 교육청 통합 유지관리 병행 | 홈페이지 취약점, 파일 업로드, 개인정보 노출 |
| 다양한 에듀테크와 외부 SaaS 사용 | 학생 데이터 처리, 공급망 보안, 계약 종료 후 삭제 문제 |
| AI·디지털 교육자료와 학습지원 소프트웨어 확대 | AI 학습 활용, 데이터 보관 위치, 자동화 판단 위험 |
| 학부모 문자·메일·알림 서비스 운영 | 학교 사칭, 스미싱, 개인정보 오발송 |
| 학교 현장 보안 인력 부족 | 사고 대응 지연, 담당자 책임 부담 증가 |
| 입학·시험·성적 처리·수능 등 특정 시기 집중 | 공격자가 노리기 쉬운 시기 발생 |

교육청의 사이버보안은 일반 행정기관의 보안과 다릅니다. 학교는 수가 많고, 현장의 정보화 수준이 다르며, 교사와 행정직원이 수업과 행정 업무를 수행하면서 보안 업무까지 모두 감당하기 어렵습니다.

따라서 서울교육청 사이버안전 정책은 다음 방향이어야 합니다.

```text
학교가 알아서 막는 보안이 아니라,
교육청이 기준을 만들고, 플랫폼을 제공하고,
교육지원청과 함께 사고 대응을 지원하는 보안
```

학생 개인정보는 한 번 유출되면 회복이 어렵습니다. 미성년자의 학습 이력, 상담 기록, 생활기록, 건강 정보, 보호자 정보는 일반 행정정보보다 더 조심스럽게 보호되어야 합니다. 학생 데이터는 단순한 행정자료가 아니라 학생의 현재와 미래에 영향을 줄 수 있는 보호 대상입니다.

이제 서울교육청은 사이버보안을 단순 전산관리 업무가 아니라, 학생 안전과 교육권을 지키는 **교육 사이버안전 정책**으로 전환해야 합니다. 교사는 보안 전문가가 아니라 교육 전문가로 일할 수 있어야 하고, 행정직원은 사고가 난 뒤에야 책임을 떠안는 구조에서 벗어나야 합니다.

## 1. 전국 공동 교육정보시스템과 연계한 학생정보 보호 협력체계 구축

서울교육청은 학생정보 보호를 **전국 공동 교육정보시스템 협력 영역**과 **서울교육청·학교 현장 보완 영역**으로 구분해 추진하겠습니다.

나이스 등 전국 공동 교육정보시스템은 교육부·KERIS·시도교육청 협력체계와 법령상 권한에 맞춰 운영·개선되는 영역입니다. 서울교육청은 이 체계를 존중하면서, 서울 학교 현장의 권한관리, 접속기록 활용, 사용자 불편, 사고 대응 요구가 반영되도록 적극 건의하겠습니다.

서울교육청은 전국 공동 시스템을 단독으로 대체하지 않습니다. 대신 학교가 실제로 어려움을 겪는 교직원 계정 보호, 권한 변경, 외부 서비스 연동, 사고 대응, 사용자 교육을 책임 있게 보완하겠습니다.

## 2. 기존 정보보호 체계를 현장지원형 사이버안전 플랫폼으로 고도화

서울교육청은 이미 운영 중인 사이버 보안관제, 침해대응, 개인정보 점검, 학교누리집 통합 유지관리, 정보화사업 보안성 검토 체계를 바탕으로 **학교 현장 지원형 사이버안전 플랫폼**을 고도화하겠습니다.

이 플랫폼은 학교를 감시하기 위한 도구가 아닙니다. 학교가 복잡한 보안 업무를 혼자 떠안지 않도록, 교육청이 위험을 먼저 확인하고 필요한 조치를 안내하며 사고 대응을 지원하는 체계입니다.

### 주요 기능

| 기능 | 내용 |
|---|---|
| 학교 홈페이지 보안 관제 | 홈페이지 공격, 변조, 악성 파일 업로드, 웹쉘 의심 행위 탐지 |
| 서울교육 웹서비스 보호 | 교육청·교육지원청·직속기관 웹서비스 보안 이벤트 통합 확인 |
| 교직원 계정 위험 탐지 | 비정상 로그인, 반복 실패, 관리자 계정 오남용, 휴면 계정 탐지 |

| 학부모 안내 부담 | 개인정보 또는 수업 영향 발생 시 안내문 표준안 제공 |

학교에는 복잡한 보안 용어와 책임만 내려보내지 않겠습니다. 교육청이 기준을 만들고, 플랫폼을 제공하고, 현장을 지원하겠습니다.

## 8. 긴급 상황 선조치·후보고 체계 도입

사이버공격은 보고 절차를 기다려주지 않습니다. 학생 개인정보 유출, 교육행정 업무 차질, 학교 홈페이지 악성코드 유포가 의심되는 경우에는 빠른 조치가 필요합니다.

서울교육청은 긴급 상황에서 피해 확산을 막기 위해 **선조치·후보고** 체계를 마련하겠습니다. 다만 선조치가 권한 남용이나 수업 방해로 이어지지 않도록 지정 책임자 승인, 조치 이력 기록, 사후 검증 절차를 함께 두겠습니다.

선조치는 서울교육청 관할 시스템과 사전에 정한 외부 연동 영역을 중심으로 시행하고, 나이스 등 전국 공동 교육정보시스템 관련 사안은 운영기관과의 비상연락·공동대응 절차에 따르겠습니다.

### 선조치 가능 항목

| 항목 | 내용 |
|---|---|
| 악성 IP 차단 | 학교 홈페이지와 서울교육 웹서비스 공격 출발지 차단 |
| 위험 계정 잠금 | 탈취 의심 교직원·관리자 계정 임시 정지 |
| 홈페이지 임시 보호 | 악성코드 유포·변조 의심 시 임시 차단 또는 보호 모드 전환 |
| 위험 세션 종료 | 비정상 접속 세션 차단 |
| 관할 시스템 접근 제한 | 서울교육청 관할 시스템의 대량 조회·다운로드 의심 시 임시 제한 |

### 승인과 사후 검증 기준

## 9. 교육감 직속 사이버안전 자문위원회 운영

교육청 사이버안전은 전산부서만의 일이 아닙니다. 학생 개인정보, 수업권, 학교 현장, 학부모 신뢰, 에듀테크 산업, AI 교육정책이 모두 연결되어 있습니다.

서울교육청은 **교육감 직속 사이버안전 자문위원회**를 운영하겠습니다. 자문위원회는 정책 방향, 제도 개선, 공개 가능한 성과지표, 사고 대응의 적정성을 검토합니다. 구체적 취약점, 침해사고 세부 로그, 관제 규칙 등 보안상 민감한 정보는 비공개 원칙으로 관리하겠습니다.

### 자문위원회 구성

| 분야 | 참여 전문가 |
|---|---|
| 침해사고 대응 | 실제 사고 분석과 대응 경험이 있는 전문가 |
| 개인정보 보호 | 학생 개인정보와 미성년자 정보 보호 전문가 |
| 학교 현장 | 교장, 교사, 행정실장 등 현장 대표 |
| 학부모 대표 | 학부모 신뢰와 안내 기준 자문 |
| 에듀테크 보안 | AI·디지털 교육자료, SaaS, 외부 서비스 보안 전문가 |
| 웹보안 | 학교 홈페이지, 웹방화벽, 웹 취약점 전문가 |
| 클라우드 보안 | 교육 클라우드와 데이터 보호 전문가 |
| 법률 전문가 | 개인정보, 교육 법령, 책임 기준 검토 |
| 청소년 보호 | 딥페이크, 사이버폭력, 디지털 피해 대응 전문가 |

### 역할

- 교육감 정기 사이버안전 보고
- 주요 사고 발생 시 긴급 자문
- 학생 개인정보 보호 기준 검토
- AI·디지털 교육자료와 에듀테크 보안 기준 검토
- 학교 현장 부담 완화 방안 검토
- 선조치·후보고 사례의 적정성 검증
- 공개 가능한 집계 성과지표 점검
- 보안상 민감하지 않은 우수 대응 사례 확산

## 11. 학생·교사·학부모 디지털 안전교육 강화

사이버안전은 기술만으로 완성되지 않습니다. 학생, 교사, 학부모가 함께 안전한 디지털 문화를 만들어야 합니다.

서울교육청은 보안교육을 형식적인 안내가 아니라 실제 생활과 연결된 교육으로 바꾸겠습니다. 개인정보 보호, AI 윤리, 딥페이크 피해 예방, 사이버폭력 대응, 학교 사칭 피싱 예방을 통합적으로 추진하겠습니다.

### 교육 내용

| 대상 | 교육 내용 |
|---|---|
| 학생 | 비밀번호 관리, 피싱 대응, 개인정보 보호, 딥페이크 피해 예방, 사이버폭력 대응 |
| 교사 | 학생정보 보호, 계정 보안, 수업용 에듀테크 안전 사용, AI 도구 활용 유의사항 |
| 행정직원 | 교육행정 시스템 사용자 보안, 개인정보 보호, 사고 보고와 증적 보존 |
| 학부모 | 학교 사칭 문자·메일 대응, 자녀 개인정보 보호, 딥페이크·사이버폭력 신고 안내 |
| 관리자 | 학교 홈페이지, 계정 권한, 위탁업체 관리, 사고 발생 시 조치 기준 |

- 학생 개인정보 자기보호 교육

## 12. 사이버안전 우수학교·우수 대응 포상제도 도입

보안은 사고가 난 뒤에만 평가받아서는 안 됩니다. 사고를 예방하고, 조기에 발견하고, 피해 확산을 막은 학교와 담당자도 평가받아야 합니다.

서울교육청은 **사이버안전 우수학교·우수 대응 포상제도**를 도입하겠습니다. 단순히 사고가 없는 학교만 포상하지 않겠습니다. 이상징후를 조기에 신고하고, 피해 확산을 막고, 재발방지 개선에 기여한 사례를 우수 대응으로 평가하겠습니다.

### 포상 대상

| 구분 | 내용 |
|---|---|
| 선제 탐지 | 이상 접속, 계정 탈취, 홈페이지 변조 시도 조기 발견 |
| 신속 신고 | 사고를 숨기지 않고 교육지원청·교육청에 신속히 공유 |
| 피해 예방 | 학생 개인정보 유출 가능성을 사전에 차단 |
| 우수 운영 | 계정관리, 패치, 홈페이지 보안, 에듀테크 보안 우수 |
| 교육 우수 | 학생·교사·학부모 디지털 안전교육 우수 |
| 협업 우수 | 학교, 교육지원청, 교육청이 함께 대응한 사례 |
| 제도 개선 | 현장 경험을 바탕으로 보안 기준 개선에 기여 |

### 운영 원칙

- 사고가 없어도 예방 성과를 평가합니다.
- 사고를 숨긴 학교가 아니라 조기 신고와 피해 확산 방지에 기여한 학교를 인정합니다.
- 개인뿐 아니라 학교와 팀 단위 포상을 병행합니다.
- 우수 사례는 보안상 민감한 정보를 제외하고 다른 학교와 공유합니다.
- 교육감 표창, 예산 지원, 컨설팅 우선 지원과 연계합니다.
