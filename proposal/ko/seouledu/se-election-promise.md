# 1. 왜 지금 서울교육청 사이버안전 대전환인가

학교는 더 이상 칠판과 교실만으로 운영되지 않습니다.  
출결, 성적, 생활기록, 급식, 상담, 학부모 소통, 원격수업, AI 디지털교과서, 에듀테크 서비스까지 교육의 많은 영역이 디지털 시스템 위에서 운영되고 있습니다.

따라서 교육 분야의 사이버공격은 단순한 전산 장애가 아닙니다.  
학생 개인정보 유출, 학사행정 마비, 학교 홈페이지 변조, 교직원 계정 탈취, 학부모 민원 시스템 장애, 디지털 수업 중단으로 이어질 수 있는 교육 안전사고입니다.

특히 교육기관은 다음과 같은 특성을 가지고 있습니다.

| 교육청·학교 특성 | 보안상 위험 |
|---|---|
| 학생 개인정보 보유 | 미성년자 개인정보 유출 위험 |
| 교사·행정직원 계정 다수 | 계정 탈취와 권한 오남용 위험 |
| 학교별 홈페이지 운영 | 웹 취약점과 악성코드 유포 위험 |
| 다양한 에듀테크 사용 | 외부 서비스와 공급망 보안 위험 |
| 학부모 소통 시스템 운영 | 계정 도용, 피싱, 사칭 위험 |
| 디지털교과서·AI 교육 확대 | 학생 데이터 처리와 AI 보안 위험 |
| 학교 현장 보안 인력 부족 | 사고 대응 지연과 책임 전가 위험 |
| 방학·입학·수능 등 특정 시기 집중 | 공격자가 노리기 쉬운 시기 발생 |

교육청의 사이버보안은 서울시청과 다릅니다.  
서울시청처럼 모든 기관을 단순히 하나로 묶는 방식만으로는 어렵습니다. 학교는 수가 많고, 현장의 정보화 수준도 다르며, 교사와 행정직원이 보안 업무까지 감당하기 어렵습니다.

따라서 서울교육청의 사이버안전 정책은 다음 방향이어야 합니다.

```text
학교가 알아서 막는 보안이 아니라
교육청이 기준을 만들고, 플랫폼을 제공하고, 사고 대응을 지원하는 보안
```

학교 현장에 책임을 떠넘기지 않겠습니다.  
교사와 행정직원이 복잡한 보안장비를 직접 운영하지 않아도, 교육청이 표준 기준과 통합 지원 체계를 통해 학교를 보호하겠습니다.

학생 개인정보는 한 번 유출되면 회복이 어렵습니다.  
미성년자의 개인정보, 학습 이력, 상담 기록, 생활기록, 건강 정보, 보호자 정보는 일반 행정정보보다 더 조심스럽게 보호되어야 합니다.

이제 서울교육청은 사이버보안을 단순 전산관리 업무가 아니라, 학생 안전과 교육권을 지키는 **교육 사이버안전 정책**으로 전환해야 합니다.

## 1. 학생 개인정보 보호 중심의 서울교육청 표준기록 체계 도입

보안의 출발점은 기록입니다.  
서울교육청은 학교, 교육지원청, 직속기관, 교육행정 시스템, 학교 홈페이지, 에듀테크 서비스에서 발생하는 주요 보안 기록을 표준화하겠습니다.

단순히 로그를 많이 모으는 것이 아니라, 사고가 발생했을 때 다음 질문에 답할 수 있도록 하겠습니다.

```text
누가 접속했는가?
어떤 학생 정보에 접근했는가?
어떤 계정이 사용되었는가?
어떤 시스템에서 이상행위가 발생했는가?
피해 학생과 학교는 어디까지인가?
어떤 조치가 이루어졌는가?
재발 방지는 어떻게 할 것인가?
```

| ----------- | ------------------------------- |

| 교직원 계정 기록   | 로그인, 실패, 비정상 접속, 관리자 권한 사용      |

이 플랫폼은 학교를 감시하기 위한 도구가 아니라, 학교를 보호하고 지원하기 위한 체계입니다.

하지만 학교 홈페이지는 공격자에게도 쉬운 표적이 될 수 있습니다.

서울교육청은 학교 홈페이지와 웹서비스를 교육청 차원에서 보호하겠습니다.

서울교육청은 학생 개인정보를 특별 보호 대상으로 지정하고, 접근·조회·수정·다운로드 기록을 강화하겠습니다.

## 5. AI 디지털교과서와 에듀테크 보안 검증 강화

AI 디지털교과서와 에듀테크 서비스는 교육 혁신의 중요한 도구입니다.  
그러나 학생 데이터를 다루는 서비스인 만큼, 보안과 개인정보 보호가 함께 검증되어야 합니다.

서울교육청은 에듀테크를 무조건 막지 않겠습니다.  
대신 안전하게 도입하고, 운영 중에도 계속 점검하겠습니다.

### 에듀테크 보안 검증 기준

| 항목           | 내용                         |
| ------------ | -------------------------- |
| 학생 데이터 수집 범위 | 필요한 정보만 수집하는지 확인           |
| 데이터 보관 위치    | 국내외 저장 위치와 접근 통제 확인        |

| AI 학습 활용 여부  | 학생 데이터가 AI 학습에 사용되는지 확인    |
| 삭제 권리        | 계약 종료 후 학생 데이터 삭제 가능 여부 확인 |
| 접속 기록        | 누가 언제 학생 정보에 접근했는지 확인      |
| 보안 패치        | 취약점 발생 시 신속 패치 가능 여부 확인    |
| 사고 대응        | 유출 사고 발생 시 통보와 복구 절차 확인    |

### 추진 내용

* 에듀테크 도입 전 보안성 검토 의무화
* 학생 데이터 처리 기준 표준화
* AI 디지털교과서 보안 점검 체계 운영
* 외부 SaaS 서비스 보안 체크리스트 제공
* 학교가 개별적으로 계약·점검하지 않도록 교육청 공통 기준 마련
* 안전성이 검증된 서비스 목록 제공
