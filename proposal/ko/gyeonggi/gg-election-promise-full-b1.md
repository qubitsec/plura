# 1. 왜 지금 경기도 AI 사이버보안 대전환인가

경기도는 대한민국에서 가장 많은 인구와 가장 넓은 산업 기반을 가진 광역자치단체입니다.  
도청, 31개 시군, 산하기관, 공공기관, 복지·의료·교통·재난안전 시스템, 산업 지원 플랫폼, 민원 서비스가 모두 디지털 시스템 위에서 운영되고 있습니다.

경기도의 디지털 행정은 도민의 일상과 직접 연결되어 있습니다. 복지 신청, 교통 서비스, 공공 예약, 재난 안내, 민원 처리, 기업 지원, 의료·복지 데이터, 청년 지원 서비스가 모두 정보시스템과 네트워크를 통해 제공됩니다.

따라서 사이버공격은 더 이상 단순한 전산 장애가 아닙니다. 도민의 개인정보, 행정서비스, 지역 산업, 공공 안전을 동시에 위협하는 새로운 형태의 안전사고이자 디지털 재난입니다.

특히 AI 기반 해킹 공격은 이미 현실화되고 있습니다. AI는 취약점을 찾고, 공격 경로를 조합하며, 계정 탈취를 자동화하고, 악성코드를 변형하며, 웹·앱·API를 빠르게 공격할 수 있습니다. 사람이 모든 경보를 확인하고, 기관별로 따로 대응하는 방식만으로는 AI 해킹의 속도를 따라가기 어렵습니다.

경기도형 사이버보안의 핵심은 **AI 에이전트 기반 능동 대응체계**입니다.

AI 에이전트는 보안 로그와 공격 흔적을 실시간으로 분석하고, 공격 흐름을 추적하며, 위험도를 판단하고, 대응 우선순위와 조치 방안을 제안합니다. 사람이 놓칠 수 있는 공격 신호를 AI 에이전트가 먼저 정리하고, 담당자는 더 빠르고 정확하게 판단할 수 있게 됩니다.

다만 AI 에이전트가 사람을 대체하는 것은 아닙니다. AI 에이전트는 실무자가 더 빠르고 정확하게 판단하도록 돕는 책임 행정형 보조 체계입니다. 중요한 차단, 격리, 계정 잠금, 서비스 중단은 명확한 권한과 승인 체계 안에서 수행하고, 모든 판단 근거와 조치 이력은 로그로 남기겠습니다.

이 대전환의 출발점은 **경기도 표준로깅 시스템**입니다. 로그가 없으면 공격 원인도 알 수 없고, 피해 규모도 산정할 수 없으며, 도민에게 어떤 정보가 영향을 받았는지도 설명할 수 없습니다.

경기도는 이제 다음 단계로 나아가야 합니다.

**경기도청, 31개 시군, 산하기관, 공공기관, 복지시설, 도립의료원, 지역 공공서비스의 보안 기록을 하나의 기준으로 통일하고, 이를 기반으로 경기도 통합 사이버보안 플랫폼을 구축하겠습니다.**

이 플랫폼은 단순한 보안장비의 집합이 아닙니다. 경기도 전체의 로그, 위협, 취약점, 패치, 사고 대응, AI 에이전트 판단, 선조치·후보고 이력을 하나의 기준으로 관리하는 **경기도형 사이버안보 운영체계**입니다.

이것은 단순한 기술 통합이 아니라 **책임 행정의 기반**입니다. 사이버사고가 발생했을 때 도민은 “어디에서 시작됐는지”, “어떤 피해가 있었는지”, “누가 어떤 조치를 했는지”, “앞으로 어떻게 막을 것인지”를 요구합니다. 경기도청, 시군, 산하기관이 서로 다른 기준으로 기록하고 대응하면 도민에게 책임 있게 설명할 수 없습니다.

복잡하면 실패합니다. 누구도 제대로 알지 못하고, 책임 있게 설명할 수 없으며, 실제 공격 대응에 기여하지 못하는 레거시 체계는 과감히 정리해야 합니다. 인증 항목을 채우기 위한 보안이 아니라, 실제 AI 해킹을 막고 도민에게 결과를 설명할 수 있는 보안으로 전환하겠습니다.

**AI가 공격하는 시대, 경기도는 AI 에이전트로 대응하겠습니다.**  
**기록하고, 분석하고, 즉시 대응하는 경기도.**  
**AI 에이전트, 국가대표 AI, 경기도형 소버린 사이버안보와 함께 AI 공격에도 흔들리지 않는 안전한 디지털 경기도를 만들겠습니다.**

# 3. 경기도 사이버보안 13대 공약

## 1. 기반 혁신
### 표준 로그와 통합 플랫폼으로 완성하는 책임 행정

경기도 사이버안보 대전환의 첫 번째 방향은 **기록과 통합**입니다. 보안 사고가 발생했을 때 도민에게 원인, 피해 규모, 조치 내역, 재발 방지 대책을 설명하려면 경기도청, 31개 시군, 산하기관, 공공기관이 같은 기준으로 기록하고 같은 플랫폼에서 분석해야 합니다.

### 1) 경기도 표준로깅 시스템 도입

보안의 출발점은 기록입니다. 경기도청, 31개 시군, 산하기관, 공공기관의 보안 로그를 하나의 기준으로 수집·보존·분석하겠습니다.

웹 요청, 인증, 계정, 서버 이벤트, 개인정보 접근, 관리자 행위, 패치, 장애, 사고 대응, AI 에이전트 분석 로그까지 표준화하여 공격 원인 분석, 피해 규모 산정, 재발 방지 대책 수립이 가능하게 하겠습니다.

### 2) 경기도 통합 사이버보안 플랫폼 구축

표준화된 로그를 기반으로 경기도 전체 통합 사이버보안 플랫폼을 구축하겠습니다. 경기도청, 31개 시군, 산하기관, 공공기관이 하나의 기준으로 협의하고 하나의 절차로 대응할 수 있도록 하겠습니다.

이 플랫폼은 단순한 장비의 집합이 아니라 로그, 위협, 취약점, 패치, 사고 대응, AI 에이전트 판단, 선조치·후보고 이력을 하나의 기준으로 관리하는 경기도형 사이버안보 운영체계입니다.

### 3) 31개 시군 공동 대응체계 구축

경기도 사이버보안은 도청만의 문제가 아닙니다. 31개 시군이 서로 다른 기준으로 대응하면 공격 원인 분석과 피해 확산 차단이 늦어질 수 있습니다.

경기도는 시군별 보안 수준의 차이를 줄이고, 공동 대응 기준을 마련하겠습니다. 공통 로그 기준, 사고 보고 기준, 긴급 차단 기준, 패치 이행 기준, AI 에이전트 분석 연계 기준을 마련하여 경기도 전체가 하나의 방어망처럼 움직이도록 하겠습니다.

### 4) 레거시 정보보안 체계 제거

기관별 개별 장비, 수동 보고, 중복 관제 중심의 낡은 보안체계를 단계적으로 제거하겠습니다. 레거시 제품과 절차는 효과성 평가를 통해 정리하고, 인증 항목 충족만을 이유로 실효성 없이 유지되는 제품은 과감히 걷어내겠습니다.

다만 시군과 산하기관의 예산, 계약, 폐쇄망 환경을 고려해 유예 기간, 전환 로드맵, 우수 전환 기관 인센티브를 제공하겠습니다.

## 2. AI 에이전트 방어
### AI 공격에는 AI 에이전트로 대응

두 번째 방향은 **AI 에이전트 기반 능동 대응체계**입니다. AI가 공격하는 시대에는 AI로 방어해야 합니다. 다만 AI가 모든 결정을 대신하는 것이 아니라, AI 에이전트가 분석과 판단을 지원하고 사람은 책임 있는 최종 판단을 수행하는 구조로 운영하겠습니다.

### 5) AI 에이전트 기반 해킹 공격 대응체계 구축

경기도 통합 사이버보안 플랫폼 위에 AI 에이전트를 적용하겠습니다.

AI 에이전트는 다음을 지원합니다.

| 기능 | 내용 |
|---|---|
| 경보 요약 | 대량 경보를 공격 흐름 중심으로 요약 |
| 공격 흐름 분석 | 웹 요청, 계정 행위, 서버 이벤트, 개인정보 접근 로그를 연결 |
| 공격자 행위 분석 | 공격자 IP, 계정, 세션, 요청 패턴을 상관분석 |
| 제로데이 의심 탐지 | 알려진 규칙에 없는 이상 요청과 비정상 응답 흐름 분석 |
| 크리덴셜 스터핑 탐지 | 로그인 실패·성공 패턴, 자동화 시도, 세션 변화를 분석 |
| 데이터 유출 가능성 분석 | 대량 조회, 다운로드, 외부 전송 의심 행위 분석 |
| 영향 범위 추정 | 관련 시스템, 계정, 개인정보 영향 범위 추정 |
| 대응 우선순위 제안 | 차단, 격리, 계정 잠금, 패치, 로그 보존 우선순위 제시 |
| 선조치·후보고 지원 | 선조치 근거와 후보고 초안 작성 |
| 사고 보고서 초안 | 원인, 피해, 조치, 재발 방지 대책 초안 작성 |

### 6) 미토스급 AI 해킹 대응체계 구축

미토스급 AI 해킹은 사람이 직접 수행하던 공격 과정을 AI가 자동화하는 위협입니다.

경기도는 다음과 같은 공격을 고위험 AI 해킹으로 분류하고 대응하겠습니다.

- 웹·앱·API 취약점 자동 탐색
- 제로데이 취약점 연쇄 공격
- 자동화된 크리덴셜 스터핑
- 계정 탈취 후 내부 이동
- 악성코드 변형 및 우회 시도
- 데이터 유출 경로 자동 탐색
- 보안 장비 탐지를 회피하는 반복 공격
- 지역 산업과 공공서비스 대상 공급망 공격

기존의 사후 보고 중심 대응으로는 이런 공격을 막기 어렵습니다. 경기도는 AI 에이전트가 공격 흔적을 실시간으로 분석하고, 위험도를 분류하며, 담당자에게 즉시 대응 방안을 제안하는 체계를 구축하겠습니다.

### 7) 국가대표 AI와 연계한 경기도형 AI 사이버 방어체계 구축

정부의 국가대표 AI 정책, K-사이버 보안 LLM, 국내 AI 보안 기술과 연계하여 경기도형 AI 사이버 방어체계를 만들겠습니다.

# 4. 핵심 공약 1: 경기도 표준로깅 시스템 도입

## “보안의 출발점은 기록입니다.”

사이버보안에서 가장 중요한 것은 로그입니다. 공격이 발생했을 때 무엇이 들어왔고, 어떤 계정이 사용되었고, 어떤 서버에서 어떤 행위가 일어났으며, 어떤 데이터가 접근·변경·유출되었는지를 확인할 수 있어야 합니다.

그러나 현재 많은 기관의 정보보안 체계는 로그를 충분히 남기지 못하거나, 남기더라도 기관별·시스템별 기준이 달라 공격의 전체 흐름을 재구성하기 어렵습니다.

로그 기준이 부실하면 다음과 같은 문제가 발생합니다.

| 문제 | 결과 |
|---|---|
| 로그 수집 기준 부재 | 어떤 이벤트를 남겨야 하는지 기관마다 다름 |
| 로그 형식 불일치 | 경기도청, 시군, 산하기관 간 통합 분석 어려움 |
| 중요 로그 누락 | 공격 원인 분석 실패 |
| 보존 기간 불명확 | 사고 이후 증거 확보 실패 |
| 개인정보 접근 로그 미흡 | 피해 규모 산정 실패 |
| 웹 요청·계정·서버 로그 분리 | 공격 흐름 파악 실패 |
| 수동 보고 중심 | 대응 방안 도출 지연 |
| AI 분석 로그 부재 | AI 에이전트 판단 근거와 책임 검증 어려움 |

결과적으로 사고가 발생해도 **공격 원인 분석, 피해 규모 산정, 재발 방지 대책 수립, 책임 소재 확인, 도민 안내**가 모두 늦어질 수밖에 없습니다.

따라서 경기도는 통합 사이버보안 플랫폼 구축에 앞서, 그 기반이 되는 **경기도 표준로깅 시스템**을 도입해야 합니다.

# 7. 표준로깅 시스템의 핵심 원칙

## 1. 공격 흐름을 재구성할 수 있어야 합니다

단일 로그만으로는 공격을 이해할 수 없습니다. 웹 요청, 계정 행위, 서버 이벤트, 개인정보 접근 기록이 하나의 흐름으로 연결되어야 합니다.

```text
외부 공격 요청
→ 로그인 시도
→ 계정 탈취
→ 관리자 권한 상승
→ 서버 명령 실행
→ 내부 이동
→ 개인정보 조회
→ 외부 전송
→ 차단 및 복구
```

이 흐름을 확인하지 못하면 사고 대응은 추측에 의존하게 됩니다.

## 2. 경기도 전체가 같은 로그 기준을 사용해야 합니다

경기도청은 A 방식, 시군은 B 방식, 산하기관은 C 방식으로 로그를 남기면 통합 분석이 불가능합니다.

| 항목 | 표준화 내용 |
|---|---|
| 로그 항목 | 반드시 남겨야 할 필수 로그 정의 |
| 로그 형식 | 기관 간 비교 가능한 공통 포맷 적용 |
| 로그 시간 | 표준 시간 동기화 |
| 로그 보존 | 중요도별 보존 기간 기준 마련 |
| 로그 무결성 | 위변조 방지 및 원본 보존 |
| 로그 접근권한 | 열람·수정·삭제 권한 통제 |
| 로그 검색 | 사고 시 즉시 검색 가능한 구조 |

## 3. 피해 규모를 산정할 수 있어야 합니다

보안사고 이후 가장 중요한 질문은 다음입니다.

```text

```

# 8. 통합 사이버보안 플랫폼과 표준로깅 시스템의 관계

## “표준로깅 시스템이 있어야 통합 플랫폼이 완성됩니다.”

경기도 통합 사이버보안 플랫폼은 표준로깅 시스템 위에서 완성됩니다.

로그 기준이 없으면 통합 플랫폼은 단순히 여러 장비의 알림을 모아 보는 수준에 머물게 됩니다. 반대로 표준로깅 시스템이 구축되면 경기도 전체의 보안 상황을 하나의 기준으로 분석할 수 있습니다.

```text
경기도 표준로깅 시스템
→ 경기도청·31개 시군·산하기관 로그 표준화
→ 통합 사이버보안 플랫폼 연계
→ AI 에이전트·국가대표 AI·경기도형 AI 사이버 방어 연계
→ 소버린 사이버보안 기업 공동 대응
→ 공격 흐름 분석
→ 피해 규모 산정
→ 등급별 대응
→ 선조치·후보고
→ 재발 방지
→ 도민 신뢰 회복
```

따라서 공약의 구조는 다음처럼 추진합니다.

```text
1단계: 경기도 표준로깅 시스템 도입
2단계: 경기도 통합 사이버보안 플랫폼 구축
3단계: AI 에이전트 기반 능동 대응체계 구축
4단계: 국가대표 AI와 연계한 경기도형 AI 사이버 방어체계 구축
5단계: 경기도형 소버린 사이버보안 생태계 구축
6단계: 미토스급 해킹 등급별 대응 체계 운영
7단계: 선조치·후보고 제도화
8단계: 경기도형 사이버보안 인재 양성
9단계: 분기별 점검과 공개 토론으로 제도 정착
```

# 12. 핵심 공약 5: 레거시 정보보안 체계 제거

경기도의 정보보안은 더 이상 기관별 장비 도입, 개별 관제, 수동 보고, 사후 점검 방식에 머물러서는 안 됩니다.

| 평가 항목 | 내용 |
|---|---|
| 실제 탐지 효과 | 최신 공격과 미토스급 자동화 공격에 대응 가능한가 |
| 로그 연계성 | 경기도 표준로깅 시스템과 연계 가능한가 |
| 통합 플랫폼 연계성 | 통합 사이버보안 플랫폼에서 분석·대응 가능한가 |
| AI 에이전트 연계성 | AI 기반 이상징후 탐지와 분석에 필요한 데이터를 제공하는가 |
| 자동 대응 가능성 | 차단, 격리, 계정 잠금 등 대응 자동화에 기여하는가 |
| 운영 이해 가능성 | 실무자가 구조와 역할을 설명하고 책임질 수 있는가 |
| 운영 부담 | 실무자에게 과도한 수동 운영 부담을 주지 않는가 |
| 중복 여부 | 다른 시스템과 기능이 중복되지 않는가 |
| 인증 의존성 | 인증 항목 충족 외에 실질적 보안 가치가 있는가 |
| 비용 대비 효과 | 예산 대비 보안 효과가 명확한가 |

## 추진 원칙

1. **신규 레거시 보안장비 도입 제한**  
   통합 플랫폼과 연계되지 않는 개별 장비 도입을 제한합니다.

2. **기존 보안장비 효과성 평가**  
   기존 장비는 즉시 폐기하는 것이 아니라, 실제 탐지·분석·대응 효과를 평가합니다.

3. **인증 항목 목적 제품 정리**  
   인증 항목을 채우기 위한 목적으로만 유지되는 제품은 통합·대체·폐지합니다.

4. **이해 불가능한 구조 정리**  
   누구도 운영 구조를 설명하지 못하거나 사고 때 책임소재가 불명확한 레거시 구조는 단계적으로 제거합니다.

5. **기존 장비 단계적 통합**  
   필요한 장비는 로그·정책·대응 절차를 통합 플랫폼으로 흡수합니다.

6. **유예 기간과 전환 인센티브 부여**  
   시군과 산하기관의 예산·계약·폐쇄망 환경을 고려해 합리적 유예 기간을 두고, 우수 전환 기관에는 예산·평가 인센티브를 제공합니다.

# 13. 핵심 공약 6: 미토스급 해킹 등급별 대응 체계 구축

## “공격 속도에 맞는 행정 대응 체계를 만들겠습니다.”

미토스급 AI 해킹은 사람이 회의하고 보고서를 올리는 속도보다 빠릅니다. 따라서 경기도는 사이버 위협을 등급화하고, 등급별 대응 권한과 절차를 명확히 하겠습니다.

공약에서 말하는 **미토스급 AI 해킹**은 단순히 “AI가 만든 공격”이라는 뜻이 아닙니다. 다음과 같은 고속·자동화·연쇄형 공격을 의미합니다.

| 유형 | 설명 |
|---|---|
| 제로데이 취약점 연쇄 공격 | 공개되지 않은 취약점 또는 패치 전 취약점을 빠르게 탐색·악용 |
| 웹·API 취약점 자동 탐색 | 대민 서비스, 민원 시스템, API의 취약점을 자동 탐색 |
| 자동화된 크리덴셜 스터핑 | 유출 계정 정보를 이용해 대량 로그인 시도 |
| 계정 탈취 후 내부 이동 | 정상 계정처럼 접속한 뒤 권한 상승과 내부 확산 |
| 랜섬웨어·데이터 유출 결합 | 침투 후 암호화, 협박, 정보 유출을 연계 |
| 방어 우회형 변종 공격 | 탐지 규칙을 피하기 위해 공격 패턴을 빠르게 변형 |
| 지역 산업 대상 공격 | 제조업, 중소기업, 첨단산업 지원 시스템을 겨냥한 공급망 공격 |

경기도는 이러한 공격을 **MITRE ATT&CK 프레임워크**에 따라 전술·기술·절차 단위로 분류하고, 표준 로그와 통합 플랫폼을 통해 공격 흐름을 재구성하겠습니다.

# 15. 핵심 공약 8: 사이버보안 실무·기술 담당관 지정과 도지사 직속 사이버보안 자문위원회 운영

경기도는 **사이버보안 실무·기술 담당관**을 지정하고, 같은 분야에서 전문성을 축적할 수 있도록 제도화하겠습니다. 정책 담당 부서는 조례·예산·계약·기관 조정 역할을 맡고, 실무·기술 담당관은 표준로깅, MITRE ATT&CK 매핑, 웹 공격 대응, AI 에이전트 운영, 소버린 기업 공동 대응, 상시 패치, 기술 검증을 책임지도록 하겠습니다.

또한 **도지사 직속 사이버보안 자문위원회**를 신설·운영하겠습니다. 도지사가 정기적으로 보고받고, 긴급 사고와 주요 정책 전환은 자문위원회 검토를 거치도록 해 사이버보안을 도지사가 직접 챙기는 구조를 만들겠습니다.

# 16. 핵심 공약 9: 선제적 공격 대응 포상제도와 우수 사례 공유

## “잘 막은 보안도 평가받고 보상받아야 합니다.”

사이버보안은 사고가 난 뒤에만 주목받는 분야가 되어서는 안 됩니다. 공격을 선제적으로 탐지하고, 피해 확산을 막고, 도민 피해를 예방한 성공 사례도 행정 성과로 인정해야 합니다.

사이버보안관제 인력은 평상시에는 존재감이 잘 드러나지 않습니다. 그러나 공격이 발생하면 가장 먼저 경보를 보고, 가장 늦게까지 로그를 확인하며, 야근과 특근, 긴급 대응과 사후 분석을 감당하는 조직입니다.

군인이 있어 도민이 밤에도 편히 잠들고 주말을 안심하고 보낼 수 있듯이, 사이버보안관제 인력은 경기도의 디지털 행정과 도민의 일상을 지키는 **보이지 않는 방어선**입니다.

경기도는 사이버보안 분야에 **선제적 공격 대응 포상제도**를 도입하겠습니다. 미토스급 AI 해킹, 웹 제로데이 공격, 크리덴셜 스터핑, 랜섬웨어, 개인정보 유출 시도를 조기에 탐지·차단한 담당자와 기관을 포상하겠습니다.

| 구분 | 내용 |
|---|---|
| 선제 탐지 | 공격 징후를 조기에 발견해 피해를 예방한 사례 |
| 선제 차단 | 악성 IP, 위험 세션, 탈취 의심 계정, 취약 서비스를 신속히 차단한 사례 |
| 피해 확산 방지 | 랜섬웨어, 개인정보 유출, 내부 이동을 조기에 막은 사례 |
| 표준로깅 우수 운영 | 공격 원인 분석과 피해 규모 산정에 필요한 로그를 정확히 확보한 사례 |
| AI 에이전트 활용 | AI 분석 결과를 검증해 신속한 대응과 후보고를 가능하게 한 사례 |
| 웹 공격 대응 | 웹 제로데이, 크리덴셜 스터핑, API 공격, 봇 공격을 조기에 탐지·차단한 사례 |
| 제조사 공동 대응 | 소버린 사이버보안 기업과 협력해 긴급 패치·차단·복구를 성공시킨 사례 |
| 제도 개선 | 사고 대응 경험을 정책·기술 기준 개선으로 연결한 사례 |
| 협업 우수 | 경기도청, 시군, 산하기관이 함께 대응해 피해를 줄인 사례 |

# 17. 핵심 공약 10: MITRE ATT&CK 기반 운영체계와 웹 공격 대응 강화

경기도 사이버안보 체계는 개별 장비 도입이나 인증 항목 충족에 머물러서는 안 됩니다. 공격자가 실제로 사용하는 전술·기술·절차를 기준으로 방어체계를 설계해야 합니다.

경기도는 글로벌 표준인 **MITRE ATT&CK 프레임워크**를 기준으로 공격 흐름을 분석하고, 탐지·차단·포렌식·재발 방지 체계를 정비하겠습니다.

경기도의 대민 행정서비스는 대부분 웹, 모바일앱, API를 통해 도민과 연결됩니다. 따라서 웹 공격 대응은 경기도 사이버안보의 최전선입니다.

| 대응 분야 | 강화 내용 |

| 웹 제로데이 대응 | 알려진 규칙이 없는 이상 요청, 비정상 응답, 공격 흐름 탐지 |
| 크리덴셜 스터핑 대응 | 로그인 요청 본문, 실패·성공 패턴, 세션 행위, 대량 자동화 시도 분석 |
| API 보안 | 비인가 API 호출, 과도한 요청, 권한 우회, 데이터 대량 조회 탐지 |
| 봇·자동화 공격 대응 | 비정상 속도, 반복 시도, 세션 재사용, 자동화 도구 징후 분석 |
| 개인정보 접근 보호 | 대량 조회·다운로드·외부 전송 이상행위 탐지 |
| MITRE ATT&CK 매핑 | 웹 침투 이후 계정 탈취, 권한 상승, 내부 이동까지 공격 흐름 연결 |

# 18. 핵심 공약 11: 소프트웨어 구성명세서와 상시 패치 의무화

경기도가 사용하는 소프트웨어와 시스템은 외부 업체가 개발·납품·운영하는 경우가 많습니다. 따라서 공급망 보안 없이는 경기도 보안도 완성될 수 없습니다.

경기도 주요 정보화 사업에는 다음을 의무화하겠습니다.

| 항목 | 내용 |

| 소프트웨어 구성명세서 제출 | 납품 소프트웨어의 구성 요소, 오픈소스, 라이브러리, 버전 정보 제출 |
