# PLURA-XDR 기술 비교 제안서
## 쿠팡 환경에서 본 SentinelOne 대비 PLURA-XDR의 차별성

---

## 핵심 요약

SentinelOne은 엔드포인트 중심 탐지와 자동화에 강한 글로벌 제품입니다.  
반면 PLURA-XDR은 웹 요청·응답 구간, 인증/세션 흐름, 서버 행위, 자동 대응, 포렌식을 하나의 흐름으로 연결해 분석하는 구조를 기반으로, 쿠팡과 같은 대규모 웹/API 서비스 환경에서 필요한 침해 해석과 운영 판단에 더 직접적으로 대응합니다.

이번 비교의 핵심은 단순한 기능 수가 아닙니다.  
실제 비교 기준은 다음과 같습니다.

- 크리덴셜 스터핑과 같은 대규모 인증 공격을 어떤 근거로 식별하는가
- 인증키 유출 이후 정상처럼 보이는 공격을 어떻게 탐지하는가
- 제로데이 또는 비정형 웹 공격을 어느 구간에서 포착하는가
- 웹 요청과 응답, 인증 결과, 서버 후속 행위, 자동 대응, 포렌식까지 하나의 사건 흐름으로 연결할 수 있는가

PLURA-XDR은 이 기준에서  
**웹 요청 본문 + 웹 응답 본문/결과 신호 + 서버 감사 로그 및 행위 로그**를 결합해 분석한다는 점에서 차별성이 있습니다.

특히 다음 기술 요소를 함께 갖습니다.

- **웹 요청·응답 본문 분석 기술**: 한국·미국 특허 보유
- **서버 감사 정책 기반 로그 분석 기술**: 특허 보유
- **웹부터 서버, 자동 대응, 포렌식까지 자동 연결되는 운영 구조**

이 문서의 목적은 두 제품을 단순 비교하는 것이 아니라,  
쿠팡 환경에서 어떤 제품이 더 직접적으로 공격 표면을 설명하고 통제할 수 있는지를 비교 관점에서 정리하는 데 있습니다.

## 비교의 기준

쿠팡 환경에서 이 비교는 다음 기준으로 읽는 것이 적절합니다.

### 1. 탐지의 출발점
공격을 어떤 데이터에서 먼저 포착하는가

### 2. 탐지 근거의 깊이
왜 공격이라고 판단했는지 어느 수준까지 설명 가능한가

### 3. 공격 흐름 복원 능력
웹 요청, 인증 결과, 서버 후속 행위, 자동 대응, 포렌식을 하나의 사건으로 연결할 수 있는가

### 4. 운영 적용성
대규모 서비스 환경에서 보안팀이 실질적으로 운영 가능한 구조인가

### 5. 대응 연계성
탐지 이후 제한, 차단, 격리, 포렌식으로 자연스럽게 이어질 수 있는가

## SentinelOne과 PLURA-XDR의 구조적 차이

SentinelOne은 엔드포인트 중심 EDR/XDR로서 다음 영역에 강점을 가집니다.

- 행동 기반 엔드포인트 탐지
- Storyline 기반 사건 흐름 정리
- 자동 대응 및 롤백
- 생성형 AI 기반 조사 보조
- AI SIEM 및 데이터 파이프라인 확장

즉, SentinelOne은 엔드포인트와 자동화 중심의 매우 성숙한 제품입니다.

다만 쿠팡과 같은 환경에서는 다음 질문이 별도로 필요합니다.

- 로그인 및 인증 공격의 실제 입력을 볼 수 있는가
- HTTP 200 응답 내부의 실패/거부/추가 인증 요구를 구분할 수 있는가
- 인증키가 유효한 상태의 Brute Force 또는 자동화 호출을 구분할 수 있는가
- 웹 침해의 시작점과 서버 후속 행위, 그리고 포렌식 확보까지 하나의 흐름으로 설명할 수 있는가

PLURA-XDR은 바로 이 지점에 초점을 둡니다.  
즉, 엔드포인트에서 결과를 보는 데 그치지 않고,  
**웹 요청 본문과 응답 결과 신호를 먼저 보고, 이를 인증/세션 흐름과 서버 로그, 자동 대응, 포렌식으로 연결**합니다.

이 차이가 크리덴셜 스터핑, 인증키 기반 Brute Force, 제로데이, 데이터 유출 징후 해석에서 의미를 가집니다.

## 브루트포스 공격과 인증키 탐지 관점의 차이

쿠팡 환경에서 Brute Force는 단순 로그인 실패 누적만을 의미하지 않습니다.  
특히 인증키, 서명키, API Key, 토큰이 외부로 노출된 이후에는 헤더나 서명 수준에서는 정상 요청처럼 보일 수 있습니다.

따라서 실제 방어는 다음 두 층위로 나누어 봐야 합니다.

### 1. 1차 방어선: 키 교체/폐기
키 교체와 폐기는 필수입니다.  
다만 현실적으로 다음 공백이 남습니다.

- 교체 적용 전 시간차
- 조직 내부 공유 지연
- 이미 유출된 키를 사용하는 자동화 공격 지속
- 개발 변경이나 오프보딩 정보가 보안팀에 늦게 도착하는 문제

즉, 키 교체만으로는 방어가 완성되지 않습니다.

### 2. 2차 방어선: 행위 기반 Brute Force 탐지·차단
PLURA-XDR은 이 2차 방어선의 필요성을 전제로 합니다.

핵심은 Access Log 표면이 아니라 다음입니다.

- **요청 본문**: 실제 인증 시도 값, 파라미터, 토큰 사용 방식
- **응답 결과 신호**: 성공/실패, 실패 사유, 추가 인증 요구, 토큰 발급 여부
- **세션 흐름**: 동일 키·계정·세션 내 반복과 자동화 패턴
- **행위 편차**: 정상 대비 성공률/실패율 변화, 시간 밀도, 반복성

즉, 인증키가 유효하더라도 다음과 같은 조건에서 탐지 가능합니다.

- 동일 **키/계정/세션**에서 실패·재시도가 정상 범위를 초과
- 세션 내에서 반복적이고 자동화된 인증 또는 토큰 발급 흐름이 관측
- IP가 수천 개로 분산되어도 공격 본질인 **반복성·시간 밀도·자동화 패턴**이 남음

이 구조는 단순 IP 차단과 다릅니다.  
PLURA-XDR은 **계정/키/세션/엔드포인트 단위 행위**로 Brute Force를 설명하고 차단할 수 있습니다.

즉, 인증키 기반 Brute Force 탐지에서 PLURA-XDR의 차별성은  
“유효한 키를 가진 공격자”까지 포함해 설명 가능한 2차 방어선이라는 점입니다.

## 크리덴셜 스터핑 관점의 차이

쿠팡과 같은 대규모 서비스 환경에서 크리덴셜 스터핑은  
단순 IP 차단 문제로 보기 어렵습니다.

실제 운영에서 중요한 것은 다음과 같습니다.

- 어떤 계정/세션/엔드포인트에서 반복 시도가 발생했는가
- 요청 본문에서 어떤 인증 시도 값이 들어왔는가
- 응답 결과가 실제 실패인지, 추가 인증 요구인지, 성공인지 구분되는가
- 자동화된 반복 패턴과 정상 사용자 오류를 분리할 수 있는가

PLURA-XDR은 이 지점에서  
**요청 본문 + 응답 결과 신호 + 세션 흐름**을 결합합니다.  
따라서 크리덴셜 스터핑을 단순 트래픽 이상이 아니라 실제 인증 공격 흐름으로 설명할 수 있습니다.

반면 SentinelOne은 엔드포인트와 행위 기반 탐지 측면에서는 강점이 있으나,  
크리덴셜 스터핑의 출발점인 웹 요청 본문과 응답 결과 신호 자체를 제품의 핵심 강점으로 보기는 어렵습니다.  
따라서 쿠팡 환경에서는 PLURA-XDR 쪽이 더 직접적인 설명력을 가집니다.

## 제로데이 관점의 차이

제로데이 대응에서도 구조적 차이가 있습니다.

SentinelOne은 엔드포인트 측면에서  
비정상 프로세스, 메모리 행위, 파일 행위, 권한 변화 등을 기반으로  
의심스러운 후속 행위를 탐지하는 데 강점이 있습니다.  
즉, 결과적으로 서버 또는 단말에서 나타난 행위를 잘 봅니다.

PLURA-XDR은 여기에 더해  
**웹 요청 본문과 웹 응답 본문/결과 신호를 먼저 분석**하고,  
그 후 서버 행위와 연결합니다.

따라서 제로데이 또는 비정형 웹 공격에 대해 다음과 같은 설명이 가능합니다.

- 어떤 비정상 입력이 들어왔는가
- 응답 결과가 어떤 변화를 보였는가
- 이후 서버에서 어떤 행위가 이어졌는가
- 그 행위에 대해 어떤 자동 대응이 작동했는가
- 포렌식 증적이 어떻게 함께 확보되었는가

즉, 제로데이를 단순히 “후속 악성 행위”로만 보는 것이 아니라  
**웹 구간의 이상 징후와 서버 구간의 후속 행위, 자동 대응, 포렌식을 함께 해석**할 수 있다는 점에서 차이가 있습니다.

## 웹부터 포렌식까지 자동 연결되는 구조

PLURA-XDR의 강점은 단순히 웹 분석이나 서버 분석 각각에 있지 않습니다.  
핵심은 이들이 **하나의 운영 구조 안에서 자동 연결**된다는 점입니다.

구조를 단순화하면 다음과 같습니다.

1. **웹 구간**
   - 요청 본문 분석
   - 응답 본문 또는 결과 신호 분석
   - 제로데이 및 크리덴셜 스터핑 징후 식별

2. **XDR 상관분석**
   - 웹 이벤트와 인증/세션 이상행위 연결
   - 기존 보안 이벤트와 상관분석
   - MITRE ATT&CK 기준 행위 해석

3. **서버 구간**
   - 프로세스, 파일, 네트워크, 계정 행위 분석
   - 감사 정책 기반 로그 분석
   - 웹셸, 확산, 랜섬웨어/APT 후속 행위 식별

4. **자동 대응**
   - 제한
   - 추가 인증
   - 차단/격리
   - 알림 연계

5. **포렌식**
   - 탐지와 동시에 증적 확보
   - 사고 이후 별도 수집이 아니라 운영 흐름 안에서 증적 정리

즉, PLURA-XDR은  
**웹 → 인증/세션 → 서버 → 자동 대응 → 포렌식**을 하나의 운영 체계로 연결하는 구조입니다.

이 점은 단일 도구를 나열한 구조와 다릅니다.  
쿠팡과 같은 환경에서는 경보가 많다는 사실보다  
이 흐름이 실제로 자동 연결되어 운영자가 사건 단위로 볼 수 있는가가 중요합니다.

## 서버 감사 정책 기반 로그 분석의 의미

이번 비교에서 추가로 봐야 할 부분은 서버의 감사 정책 기반 로그 분석입니다.

쿠팡과 같은 환경에서 웹 공격은 종종 서버 행위로 이어집니다.

- 프로세스 실행
- 파일 생성 또는 변경
- 권한 상승 시도
- LOLBAS 악용
- 내부 확산 전조

PLURA-XDR은 서버 행위 분석에서  
일반 이벤트 수집 수준이 아니라  
**감사 정책 기반 로그 분석**을 통해 운영체제 수준의 행위를 해석하는 구조를 함께 설명할 수 있습니다.

그리고 이 영역 역시 특허 보유 기술로 제시할 수 있습니다.

이 부분은 과도하게 강조할 성격은 아니지만,  
제품 비교 관점에서는 의미가 있습니다.  
즉, PLURA-XDR은 웹 앞단만 보는 제품이 아니라  
웹 구간과 서버 구간을 모두 설명하는 구조를 갖추고 있다는 뜻입니다.

## 비교표에서 도출되는 해석

이 비교에서 도출되는 해석은 비교적 명확합니다.

### 1. SentinelOne은 엔드포인트 중심으로 매우 강한 제품입니다
이 점은 전제로 두어야 합니다.  
행동 기반 탐지, 자동화, 생성형 조사 보조 측면에서 성숙도가 높습니다.

### 2. 그러나 쿠팡의 핵심 공격 표면은 엔드포인트만이 아닙니다
쿠팡 환경에서는 로그인, 인증, 세션, API, 데이터 응답, 서버 후속 행위를 함께 봐야 합니다.

### 3. PLURA-XDR은 그 앞단과 연결 구간을 더 깊게 봅니다
PLURA-XDR은 웹 요청 본문, 웹 응답 본문/결과 신호, 인증/세션 흐름, 서버 감사 로그를 연결해 사건을 해석합니다.

### 4. 크리덴셜 스터핑, 인증키 기반 Brute Force, 제로데이에서 차이가 더 분명해집니다
크리덴셜 스터핑과 인증키 기반 Brute Force는 요청 본문과 응답 결과를 봐야 하고,  
제로데이는 웹 구간의 비정형 입력과 서버 후속 행위를 함께 봐야 합니다.  
이 점에서 PLURA-XDR은 쿠팡 환경에 더 직접적인 설명 구조를 가집니다.

### 5. 웹부터 포렌식까지의 자동 연결이 운영 차이를 만듭니다
탐지 자체보다 더 중요한 것은  
그 탐지가 자동 대응과 포렌식 확보까지 이어지는가입니다.  
PLURA-XDR은 이 연결 구조를 제품의 핵심 축으로 가집니다.

## 최종 정리

SentinelOne은 엔드포인트 중심 탐지와 자동화에 강한 글로벌 제품입니다.  
이 점은 분명합니다.

반면 PLURA-XDR은 쿠팡과 같은 대규모 웹/API 서비스 환경에서 중요한 다음 요소를 하나의 흐름으로 연결해 설명할 수 있습니다.

- 웹 요청 본문
- 웹 응답 본문 또는 결과 신호
- 인증/세션 흐름
- 서버 행위 및 감사 정책 기반 로그
- 자동 대응
- 포렌식

특히 다음 항목에서 차별성이 명확합니다.

- **크리덴셜 스터핑**: 요청 본문 + 응답 결과 + 세션 흐름 기반 분석
- **인증키 기반 Brute Force**: 키/계정/세션 단위 임계치 + 세션 행위 기반 탐지
- **제로데이**: 웹 구간의 비정형 입력과 서버 후속 행위의 연결 분석
- **기술 신뢰 요소**: 웹 요청·응답 본문 분석 특허, 서버 감사 정책 기반 로그 분석 특허
- **운영 구조**: 웹부터 자동 대응, 포렌식까지 이어지는 통합 연결 구조
- **운영 적합성**: 쿠팡형 웹 중심 서비스 환경에 대한 직접적인 설명력

따라서 이 비교의 결론은  
어느 제품이 더 유명한가가 아니라,  
**쿠팡의 실제 공격 표면과 운영 구조를 어느 제품이 더 직접적으로 설명하고 통제할 수 있는가**에 있습니다.

그 기준에서 PLURA-XDR은  
엔드포인트 중심 강점을 가진 글로벌 제품과 다른 위치에서,  
쿠팡 환경에 더 밀접한 분석 축과 운영 구조를 제공하는 제품으로 정리할 수 있습니다.
