# PLURA-XDR 기술 제안서
## 쿠팡 대규모 서비스 환경을 위한 제품 비교 중심 제안

---

## 제안의 출발점

쿠팡과 같은 대규모 서비스 환경에서 보안 제품의 평가는  
단순히 “탐지가 되는가”만으로 끝나지 않습니다.

실제로 중요한 것은 다음입니다.

- 웹 요청의 실제 입력을 볼 수 있는가
- 응답 결과를 해석할 수 있는가
- 그 결과가 서버에서 어떤 후속 행위로 이어졌는가를 복원할 수 있는가
- 이를 하나의 사건 흐름으로 설명할 수 있는가
- 탐지 이후 차단, 격리, 포렌식까지 운영 흐름으로 연결할 수 있는가

즉, 쿠팡이 필요로 하는 것은 단순 경보 수집이 아니라,  
**웹 요청 본문, 응답 결과 신호, 서버 행위를 하나의 흐름으로 연결해 실제 침해를 더 깊게 설명하고 통제할 수 있는 보안 운영 체계**입니다. 이 점이 이번 제안의 핵심 출발점입니다. 

---

## 0️⃣ 제안 요약

PLURA-XDR은 단순한 EDR이나 일반적인 XDR 제품이 아닙니다.  
핵심은 **웹 요청·응답 분석 + 계정/세션 이상행위 분석 + 서버 행위 분석 + 자동 대응 + 포렌식 연계**를 하나의 운영 체계로 제공한다는 점입니다. 

쿠팡 환경에서 특히 중요한 이유는 분명합니다.

- 대규모 로그인 및 인증/API 구간을 보호해야 함
- 단순 IP 차단이 아니라 계정/키/세션 단위 행위 분석이 필요함
- 키 탈취 이후 “정상처럼 보이는 공격”까지 고려해야 함
- 개발 조직이나 인사 이벤트에 의존하지 않고 보안 조직이 독자적으로 탐지·통제할 수 있어야 함

따라서 이번 제안의 목적은 글로벌 제품을 단순히 대체하자는 것이 아니라,  
**글로벌 EDR/XDR이 강한 엔드포인트 중심 가시성 위에, 쿠팡이 실제로 더 필요한 웹 기반 가시성과 설명력을 추가하자**는 데 있습니다. 

---

## 1️⃣ 비교의 기준

이번 제안에서 제품 비교는 다음 기준으로 진행합니다.

### 1-1. 탐지 범위
무엇을 수집하고 무엇을 탐지하는가

### 1-2. 탐지 근거의 깊이
왜 공격이라고 판단했는지 설명 가능한가

### 1-3. 사건 복원 능력
웹 요청부터 서버 후속 행위까지 하나의 침해 흐름으로 연결 가능한가

### 1-4. 운영 현실성
대규모 서비스 환경에서 실제 운영 가능한가

### 1-5. 대응 연계성
탐지 이후 차단, 격리, 포렌식으로 이어지는가

이 기준은 단순 기능 비교가 아니라,  
**쿠팡 관계자가 실제 운영과 사고 대응 관점에서 제품을 읽을 수 있도록 하기 위한 기준**입니다. 

---

## 2️⃣ 쿠팡 환경에서 먼저 봐야 할 핵심 질문

쿠팡과 같은 대규모 서비스 환경에서는 다음 질문이 특히 중요합니다.

### 2-1. Access Log만으로 충분한가

아닙니다.  
실제 Brute Force, 크리덴셜 스터핑, 인증 우회, API 악용의 핵심 증거는  
대부분 URL이나 상태코드가 아니라 **요청 본문(POST Body)** 과 **응답 결과 신호** 안에 존재합니다.  
응답코드가 200이어도 응답 본문 안에서는 실패, 권한 거부, 추가 인증 요구가 기록될 수 있습니다. 

### 2-2. 키가 유효하면 정상으로 보이지 않는가

표면상으로는 그렇게 보일 수 있습니다.  
그러나 실제 공격은 반복성, 시간 밀도, 자동화 패턴, 성공률/실패율 편차, 특정 자원 집중 같은 행위 흔적을 남깁니다.  
즉, **키는 통과해도 행위는 숨기기 어렵습니다.** 

### 2-3. 보안팀이 개발 통지를 기다려야 하는가

그렇게 설계하면 구조적으로 공백이 생깁니다.  
보안팀은 개발 변경, 오프보딩, 권한 변경, 키 교체 통지에 의존하지 않고도,  
관측·탐지·통제 체계를 통해 독자적으로 침투 및 유출 시도를 통제할 수 있어야 합니다. 

---

## 3️⃣ 제품 비교의 큰 그림

글로벌 EDR/XDR 제품은 분명한 강점을 가지고 있습니다.

- 엔드포인트 중심 행위 탐지
- 행동 기반 AI/ML
- 자동 대응 및 롤백
- 생성형 AI 기반 조사 보조
- 대규모 글로벌 운영 경험

특히 SentinelOne은 행동 기반 탐지, 자동 대응, Purple AI 기반 생성형 조사 보조를 함께 제공하는 다층 구조의 제품으로 이해하는 것이 맞습니다. 단순히 “전통 AV”로 보는 것은 부정확합니다. 

다만 쿠팡과 같은 웹 중심 대규모 서비스 환경에서는 여전히 별도의 질문이 남습니다.

> 엔드포인트 중심 분석만으로  
> 웹 요청의 실제 입력, 응답 결과, 세션 흐름, 그리고 서버 후속 행위를  
> 하나의 침해 흐름으로 충분히 설명할 수 있는가?

PLURA-XDR은 바로 이 지점에서 차별화됩니다.  
즉, **웹 요청 본문 + 응답 결과 신호 + 서버 행위**를 결합해,  
웹 기반 침해의 시작과 성공 여부, 후속 영향까지 더 깊게 설명할 수 있는 구조를 제공합니다.

## 4️⃣ SentinelOne과 PLURA-XDR 비교

아래 표는 쿠팡 관계자가 읽을 때 가장 중요하게 봐야 하는 비교 항목을 중심으로 정리한 것입니다.

| 비교 항목 | SentinelOne | PLURA-XDR |
| --- | --- | --- |
| 제품 출발점 | 엔드포인트 중심 EDR/XDR | 웹-계정-서버 연계형 XDR |
| 핵심 강점 | 행동 기반 엔드포인트 탐지, Storyline, 자동 대응, Purple AI | 요청 본문 분석, 응답 결과 신호 분석, 웹-서버 상관분석, 자동 대응, 포렌식 연계 |
| AI의 성격 | 행동 기반 탐지 AI/ML + 자동화 + 생성형 AI 보조(Purple AI) | 탐지·분석·상관분석·자동 대응 중심 구조, AI 신위협 대응은 웹/서버 연계 관점에서 설명 |
| 주 분석 입력 | 엔드포인트/보안 이벤트 중심 | 웹 요청 본문, 응답 결과 신호, 인증/세션 흐름, 서버 행위 |
| 웹 요청 본문 분석 | 일반적 핵심 강점으로 보기 어려움 | 핵심 차별점 |
| 응답 결과 신호 분석 | 제한적 또는 별도 연계 필요 | 핵심 차별점 |
| 웹 이벤트와 서버 후속 행위의 단일 흐름 복원 | 가능하더라도 기본 강점은 엔드포인트 중심 | 제품 핵심 구조 |
| 크리덴셜 스터핑/대규모 인증 공격 설명력 | 엔드포인트/행위 탐지 관점은 강점 | 요청 본문 + 응답 결과 + 세션 흐름까지 결합 가능 |
| 키 탈취 이후 “정상처럼 보이는 공격” 탐지 설명 | 행위 탐지 관점 접근 | 계정/키/세션/엔드포인트 단위 행위 분석으로 직접 설명 가능 |
| 민감정보 유출 징후 분석 | 서버/엔드포인트 중심 | 웹 응답 구간 포함 설명 가능 |
| MITRE ATT&CK 기반 설명 | 가능 | 웹 공격부터 계정 공격, 내부 확산, 유출, 랜섬웨어/APT까지 하나의 흐름으로 설명 |
| 자동 대응 | 강점 | 강점 |
| 포렌식 연계 | 가능 | 탐지-대응-포렌식 연계가 제품 설명의 핵심 축 |
| 쿠팡형 대규모 웹/API 서비스 적합성 | 엔드포인트 측면 강점 | 웹 중심 침해 설명력 측면 강점 |
| 한국어/영어 지원 | 글로벌 영어 중심 운영 친화적 | 한국어/영어 동시 지원, 로컬 운영 밀착 대응 |
| 로컬 정책 조정 및 현장 피드백 반영 속도 | 일반적으로 글로벌 프로세스 의존 | 로컬 기술 조직 기반으로 빠른 조정 가능 |

이 비교의 핵심은 SentinelOne을 낮추는 데 있지 않습니다.  
오히려 SentinelOne은 **행동 기반 탐지 + 자동화 + 생성형 분석 보조**라는 분명한 강점을 가진 제품입니다. 다만 쿠팡 환경에서는 그 강점만으로 충분하지 않을 수 있으며, 특히 **웹 요청 본문과 응답 결과 신호까지 포함한 침해 설명력**이 별도로 요구됩니다. 바로 그 지점에서 PLURA-XDR의 차이가 발생합니다.

## 5️⃣ 왜 PLURA-XDR이 쿠팡 환경에서 더 명확하게 읽히는가

### 5-1. 요청 본문 기반 분석

대규모 로그인, 인증, 토큰 발급, API 호출 환경에서는  
공격의 실질적 증거가 POST Body 안에 존재하는 경우가 많습니다.

PLURA-XDR은 다음을 분석 대상으로 삼습니다.

- 인증 시도 값
- 비정상 파라미터 조작
- 자동화된 대량 호출 패턴
- 악성 입력 시도
- 일반 Access Log로 드러나지 않는 시도 내용

이는 단순히 URL, IP, 응답코드 수준의 표면 분석과 본질적으로 다릅니다. 

### 5-2. 응답 결과 신호 기반 분석

탐지는 시도만 봐서는 부족합니다.  
실제로 그 시도가 성공했는지, 실패했는지, 어떤 사유로 실패했는지를 알아야  
정확한 차단과 낮은 오탐을 동시에 달성할 수 있습니다.

PLURA-XDR은 다음 신호를 해석 대상으로 삼습니다.

- 로그인 성공/실패
- 실패 사유
- 토큰 발급 여부
- 권한 부여 결과
- 추가 인증 요구 여부

즉, **탐지의 정확도와 안전한 자동 차단의 기반**이 더 분명합니다.

### 5-3. 웹-서버 상관분석

쿠팡 관계자가 가장 중요하게 볼 부분은 이것입니다.

PLURA-XDR은

- 특정 웹 요청
- 해당 세션 또는 계정 흐름
- 서버 프로세스/파일/네트워크 행위
- 내부 확산 또는 후속 이상 징후

를 하나의 흐름으로 연결해 해석합니다.

따라서 개별 경보를 나열하는 것이 아니라,  
**웹에서 시작된 침해가 실제 서버 행위로 이어졌는가**를 더 빠르게 설명할 수 있습니다. 

### 5-4. 보안팀 독자 운영 모델

쿠팡과 같은 조직에서는 보안팀이 항상 개발 변경이나 인사 이벤트를 먼저 알 수 없습니다.

PLURA-XDR은 다음 원칙을 전제로 합니다.

- 종료/복호화 지점에서 요청 본문 수집
- 응답 결과 신호 정규화
- 임계치 + 세션 행위 기반 탐지
- 단계적 자동 통제 적용

즉, “누가 알려주면 반응하는 보안”이 아니라,  
**보안팀이 지금부터 독자적으로 시작할 수 있는 운영 모델**을 지원합니다.

## 6️⃣ 제품 비교 요약

### 6-1. SentinelOne이 강한 영역

SentinelOne은 다음 영역에서 분명한 강점을 가집니다.

- 엔드포인트 중심 행동 기반 탐지
- Storyline 기반 사건 흐름 정리
- 자동 대응 및 롤백
- Purple AI 기반 자연어 조사 보조
- AI SIEM 및 데이터 파이프라인 확장

즉, **엔드포인트와 운영 자동화 관점에서 매우 성숙한 글로벌 제품**입니다. 

### 6-2. PLURA-XDR이 더 강하게 설명할 수 있는 영역

PLURA-XDR은 다음 지점에서 더 직접적인 설명력을 가집니다.

- 웹 요청 본문 분석
- 응답 결과 신호 분석
- 크리덴셜 스터핑/Brute Force의 근거 제시
- 키 탈취 이후 행위 기반 탐지 설명
- 웹 요청부터 서버 후속 행위까지 단일 흐름 복원
- 웹 기반 데이터 유출 징후 설명
- 탐지 이후 자동 대응과 포렌식 연계

즉, **웹 중심 대규모 서비스 환경에서 실제 침해를 더 깊게 설명해야 하는 지점**에서 차별성이 분명합니다. 

---

## 7️⃣ 한국어·영어 지원 로컬 회사라는 장점

PLURA-XDR의 장점은 기술만이 아닙니다.

쿠팡과 같은 환경에서는  
국내 실무 운영팀과 글로벌 감사/컨설팅/보고 체계가 동시에 존재합니다.  
이때 중요한 것은 단순 언어 지원이 아니라, **실제 운영 품질과 대응 속도**입니다.

PLURA-XDR은 다음 장점을 가집니다.

- 한국어·영어 동시 지원
- 로컬 서비스 환경에 대한 높은 이해
- 정책 조정과 현장 튜닝의 빠른 반영
- 시차 없는 협업
- 국내 운영 실무에 맞는 상세 대응

즉, 로컬 회사라는 장점은 감성 포인트가 아니라,  
**운영 민첩성과 사고 대응 품질의 차이**로 설명하는 것이 맞습니다. 이 방향은 기존 제안서에서 반복된 “운영 가능한 통합 구조”라는 메시지와도 일치합니다.

## 8️⃣ 쿠팡 관점의 최종 정리

쿠팡이 이번 비교에서 봐야 할 핵심은 단순합니다.

### 첫째
SentinelOne은 엔드포인트 중심 탐지, 자동 대응, 생성형 분석 보조 측면에서 강한 글로벌 제품입니다. 

### 둘째
그러나 쿠팡과 같은 대규모 웹/API 서비스 환경에서는  
엔드포인트 중심 가시성만으로는  
웹 요청의 실제 입력, 응답 결과, 세션 흐름, 데이터 유출 징후를 충분히 설명하기 어렵습니다. 

### 셋째
PLURA-XDR은 요청 본문, 응답 결과 신호, 서버 행위를 연결해  
웹 기반 침해의 전후 맥락을 더 깊고 더 명확하게 복원할 수 있습니다.

### 넷째
이는 단순한 기능 차이가 아니라,  
**조사 속도, 설명 가능성, 오탐 제어, 대응 결정 품질의 차이**로 이어집니다. 

### 다섯째
PLURA-XDR은 한국어와 영어를 모두 지원하는 로컬 기술 조직 기반으로,  
쿠팡의 실제 운영 환경에 맞춘 빠른 기술 대응과 정책 조정을 제공할 수 있습니다. 

---

## 9️⃣ 결론

이번 제안의 목적은 글로벌 제품을 부정하는 것이 아닙니다.

오히려 결론은 더 명확합니다.

- SentinelOne은 엔드포인트 중심 탐지와 자동화에 강점이 있습니다.
- PLURA-XDR은 쿠팡과 같은 대규모 웹 서비스 환경에서 더 필요한  
  **요청 본문 + 응답 결과 신호 + 서버 행위의 통합 분석**에 강점을 가집니다.
- 따라서 PLURA-XDR의 가치는 “같은 제품의 대체”가 아니라,  
  **쿠팡 환경에서 실제 침해를 더 깊게 설명하고 더 빠르게 통제할 수 있는 보안 운영 플랫폼**이라는 데 있습니다.

즉, 쿠팡 관점에서 PLURA-XDR은  
단순히 또 하나의 XDR이 아니라,  
**웹 중심 대규모 서비스 환경에서 글로벌 제품이 상대적으로 약할 수 있는 가시성과 설명력을 보완하고 강화하는 플랫폼**으로 읽는 것이 가장 정확합니다.
