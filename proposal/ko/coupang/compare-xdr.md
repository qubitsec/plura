# PLURA-XDR 기술 제안서
## 쿠팡 대규모 서비스 환경을 위한 글로벌 제품 대비 기술적 차별성 설명

---

## 0. 문서 목적

본 문서는 쿠팡과 같은 대규모 서비스 환경에서,  
PLURA-XDR이 글로벌 보안 제품 대비 어떤 기술적 차별성을 가지며,  
그 차이가 실제 침해 대응과 운영 효율 측면에서 왜 중요한지를 설명하기 위한 기술 문서입니다.

이 문서는 단순한 제품 소개서가 아닙니다.  
특히 읽는 사람이 Mandiant 컨설턴트와 같은 사고 대응 및 침해 분석 관점의 전문가라는 점을 전제로,  
다음 질문에 명확히 답하는 것을 목표로 합니다.

- 무엇을 더 볼 수 있는가
- 무엇을 더 정확히 설명할 수 있는가
- 무엇을 더 빠르게 조사할 수 있는가
- 왜 그 차이가 실제 보안 운영에서 중요한가

PLURA-XDR의 핵심 가치는 단순히 “기능이 많다”는 데 있지 않습니다.  
핵심은 **웹 요청 본문, 응답 결과 신호, 서버 행위를 하나의 흐름으로 연결하여 침해 전후 맥락을 더 깊게 복원할 수 있다**는 점입니다. 이는 기존 글로벌 EDR/XDR 계열 제품이 상대적으로 강한 엔드포인트 행위 탐지와는 다른 차별성입니다. 

---

## 1. 평가 기준의 출발점

Mandiant와 같은 침해사고 대응 관점에서는 제품 평가의 기준이 단순하지 않습니다.  
중요한 것은 “탐지가 되는가”만이 아니라, 그 탐지가 **충분히 설명 가능하고, 실제 조사와 대응으로 이어질 수 있는가**입니다.

따라서 본 제안서는 다음 기준 위에서 작성됩니다.

### 1-1. 탐지가 가능한가
공격 또는 이상 행위를 실제로 식별할 수 있어야 합니다.

### 1-2. 탐지 근거가 충분한가
운영자 또는 분석가가 “왜 이것이 공격인가”를 이해할 수 있어야 합니다.

### 1-3. 원인 분석과 범위 산정이 가능한가
침해의 시작점, 성공 여부, 후속 행위, 영향 범위를 복원할 수 있어야 합니다.

### 1-4. 운영 조직이 지속 운용 가능한가
특정 개발 일정, 조직 이벤트, 별도 통지 체계에 과도하게 의존하지 않아야 합니다.

### 1-5. 기존 글로벌 솔루션과 중복이 아니라 보완 또는 우위가 있는가
글로벌 제품이 이미 잘하는 부분을 단순 반복하는 것이 아니라,  
실제 운영에서 남는 공백을 메우거나 더 깊은 가시성을 제공해야 합니다.

## 2. 글로벌 보안 제품의 일반적 강점

글로벌 EDR/XDR 제품은 분명한 강점을 가지고 있습니다.  
특히 SentinelOne과 같은 제품은 다음 요소를 결합해 제공합니다.

- 행동 기반 탐지 AI/ML
- 룰 및 정책 기반 자동 대응
- 생성형 AI 기반 조사 보조 기능
- 엔드포인트 중심 행위 분석
- 대규모 배포 및 운영 경험

즉, 글로벌 제품은 단순한 시그니처 기반 도구가 아니라,  
**행동 기반 탐지 + 자동화 + 생성형 분석 보조**를 포함한 다층 구조로 발전하고 있습니다.  
따라서 이를 단순히 “전통 제품”으로만 분류하는 것은 부정확합니다. 

하지만 이 강점이 곧 모든 영역에서 충분한 가시성을 보장한다는 뜻은 아닙니다.  
특히 대규모 웹 서비스 환경에서는 별도의 질문이 필요합니다.

> 엔드포인트 중심 행위 분석만으로,  
> 웹 요청의 실제 입력과 응답 결과, 그리고 그 이후 서버 행위까지  
> 하나의 침해 흐름으로 충분히 설명할 수 있는가?

바로 이 지점에서 PLURA-XDR의 차별성이 시작됩니다.

## 3. 글로벌 제품만으로는 남을 수 있는 공백

### 3-1. 웹 요청 본문은 Access Log 표면에 드러나지 않습니다

실제 인증 우회, Brute Force, 크리덴셜 스터핑, API 악용, 비정상 파라미터 조작은  
대부분 URL보다 **요청 본문(POST Body)** 안에서 발생합니다.

일반적인 웹 Access Log에는 보통 다음 정도만 남습니다.

- 시간
- IP
- URL
- 메서드
- 응답코드

그러나 이것만으로는 실제 시도 값이 무엇이었는지,  
정상 사용자의 실패인지 자동화 공격인지,  
혹은 단순 트래픽 급증인지까지 분리하기 어렵습니다. 

### 3-2. 응답코드만으로는 결과를 설명하기 어렵습니다

운영 환경에서는 HTTP 200을 유지하면서도  
실패 사유나 권한 부여 결과를 응답 본문(JSON 메시지, 결과 코드) 안에 담는 경우가 많습니다.

이 경우 응답코드만 보면 정상으로 보일 수 있습니다.  
하지만 실제로는 다음이 응답 결과 신호에 들어 있습니다.

- 로그인 성공/실패
- 실패 사유
- 토큰 발급 여부
- 권한 부여 결과
- 추가 인증 요구 여부

즉, **응답 결과 신호가 없으면 실제 공격을 정상 흐름으로 오판할 가능성**이 있습니다. 

### 3-3. 웹 침투와 서버 후속 행위가 분리되면 조사 경로가 길어집니다

많은 제품은 웹 구간과 서버 구간을 각각 잘 볼 수는 있어도,  
그 둘을 하나의 사건 흐름으로 설명하는 데 한계가 있습니다.

하지만 실제 침해는 다음처럼 이어집니다.

- 웹 요청
- 인증 시도 또는 우회
- 세션 또는 토큰 확보
- 서버 후속 행위
- 내부 확산
- 데이터 수집
- 유출 또는 랜섬웨어/장기 잠복

따라서 개별 경보를 많이 보는 것보다 중요한 것은  
**공격의 앞단과 뒷단을 하나의 흐름으로 연결해 설명하는 능력**입니다.  
PLURA-XDR은 이 흐름을 웹-서버 상관분석 구조로 풀어내는 데 집중합니다.

## 4. PLURA-XDR의 핵심 차별성

PLURA-XDR의 핵심은 단순한 “추가 기능”이 아닙니다.  
핵심은 **글로벌 EDR/XDR이 상대적으로 강한 엔드포인트 중심 가시성 위에,  
웹 요청 본문과 응답 결과 신호까지 결합해 더 깊은 침해 맥락을 복원할 수 있다**는 점입니다.

### 4-1. 요청 본문 기반 분석

대부분의 중요한 웹 공격 증거는 요청 본문에 존재합니다.

- 인증 시도 값
- 파라미터 조작
- 비정상 요청 구조
- 대량 자동화 호출 패턴
- 악성 입력 시도

PLURA-XDR은 이 입력 자체를 분석 대상으로 삼습니다.  
이는 단순 URL, IP, 응답코드 수준의 표면 분석과는 본질적으로 다릅니다. 

### 4-2. 응답 결과 신호 기반 분석

탐지의 정확도를 높이려면 시도 자체만이 아니라  
그 시도가 어떤 결과를 만들었는지도 함께 봐야 합니다.

PLURA-XDR은 응답 결과 신호를 통해 다음을 더 명확히 판단할 수 있습니다.

- 공격 시도가 실제 성공했는가
- 실패가 반복되는가
- 정상 사용 실패와 자동화 공격을 구분할 수 있는가
- 안전한 자동 차단이 가능한가

즉, **정확한 탐지와 낮은 오탐 사이의 균형**을 잡는 데 응답 결과 신호가 중요한 역할을 합니다. 

### 4-3. 웹-서버 상관분석

PLURA-XDR은 웹 이벤트와 서버 행위를 별도 로그가 아니라  
하나의 침해 흐름으로 연결해 해석합니다.

예를 들어,

- 특정 요청 이후
- 어떤 세션이 생성되었고
- 서버에서 어떤 프로세스 또는 파일 행위가 이어졌으며
- 그 결과가 어떤 확산 또는 유출 시도로 연결되었는지

를 분석할 수 있습니다.

이는 사고 조사 시 “경보 목록”이 아니라  
**사건 단위의 설명 구조**를 제공한다는 의미입니다. 

### 4-4. MITRE ATT&CK 기반 설명력

PLURA-XDR은 탐지 항목을 기능 중심이 아니라  
MITRE ATT&CK 기준으로 해석 가능한 구조를 지향합니다.

따라서 단순히 “이상하다”가 아니라,

- 어떤 초기 접근이 있었고
- 어떤 자격 증명 시도가 있었으며
- 어떤 권한 상승 또는 LOLBAS 악용이 있었고
- 어떤 내부 확산 또는 영향 단계로 이어졌는지

를 설명 가능한 형태로 정리할 수 있습니다. 

### 4-5. 자동 대응과 포렌식 연계

탐지는 그 자체로 끝나면 의미가 약합니다.  
실제 운영에서는 탐지 이후 다음으로 이어져야 합니다.

- 알림
- 제한
- 추가 인증
- 차단
- 격리
- 증적 확보
- 사후 분석

PLURA-XDR은 탐지 이후 조치와 포렌식 증적 확보까지 같은 운영 흐름 안에서 연결하는 구조를 지향합니다.

## 5. 쿠팡 환경에서 왜 이 차이가 더 중요해지는가

쿠팡과 같은 대규모 서비스 환경에서는 단일 솔루션의 기능 여부보다,  
**어떤 신호를 얼마나 깊게 수집하고, 그 신호를 실제 운영 의사결정으로 얼마나 빠르게 연결할 수 있는가**가 중요합니다.

### 5-1. 대규모 인증/API 환경에서는 행위 기반 탐지가 필수입니다

대규모 서비스에서는 공격자가 단순한 한 번의 시도가 아니라,  
분산된 IP와 자동화된 호출 패턴을 이용해 지속적인 인증 시도, 토큰 발급 시도, API 악용을 반복할 수 있습니다.

이때 핵심은 IP 수가 아니라 다음입니다.

- 반복성
- 시간 밀도
- 성공률/실패율 편차
- 세션 단위 지속성
- 자동화된 호출 패턴

즉, 방어의 기준은 IP 블랙리스트보다  
**계정/키/세션/엔드포인트 단위의 행위 분석**으로 옮겨가야 합니다.  
PLURA-XDR은 이 지점을 Layer-2 방어선 관점으로 설명합니다. 

### 5-2. 키 탈취 이후에는 “정상처럼 보이는 공격”이 가능해집니다

서명키, 인증키, API Key, 토큰이 외부로 노출되면  
헤더 또는 서명 수준에서는 정상 요청처럼 보일 수 있습니다.

하지만 공격자는 여전히 다음 흔적을 남깁니다.

- 비정상적인 반복 호출
- 자동화된 인증 시도
- 정상 기준선 대비 급격한 편차
- 실패와 재시도의 연쇄
- 특정 자원에 대한 비정상적인 집중

즉, **키가 유효해도 행위는 숨기기 어렵습니다.**  
PLURA-XDR은 바로 이 행위 신호를 기반으로 탐지·차단을 수행하는 구조를 제안합니다. 

### 5-3. 개발 조직 의존도를 줄이는 보안 운영이 필요합니다

대규모 조직일수록 다음이 늘 완벽하게 공유되기는 어렵습니다.

- 오프보딩
- 권한 변경
- 키 교체
- 배포 일정
- 개발 변경 사항

따라서 “누군가 알려주면 반응하는 보안”은 구조적으로 공백이 생길 수 있습니다.

쿠팡에 필요한 것은  
개발 조직이나 인사 이벤트에 과도하게 의존하지 않고도,  
보안 조직이 **관측 + 탐지 + 통제**를 독자적으로 운영할 수 있는 체계입니다.  
PLURA-XDR은 이 원칙을 분명히 전제합니다.

## 6. 글로벌 제품과 PLURA-XDR의 비교 관점

이번 제안의 목적은 글로벌 제품을 낮춰 평가하는 것이 아닙니다.  
오히려 각 제품의 강점과 한계를 구분한 뒤,  
쿠팡 환경에서 필요한 가시성과 설명력의 관점에서 PLURA-XDR의 위치를 명확히 하려는 것입니다.

### 6-1. 글로벌 제품의 일반적 강점

- 엔드포인트 중심 행위 탐지
- 자동 대응 및 자동화
- 생성형 AI 기반 조사 보조
- 대규모 운영 경험
- 폭넓은 배포 레퍼런스

### 6-2. 글로벌 제품이 상대적으로 약할 수 있는 지점

- 웹 요청 본문 자체에 대한 심층 분석
- 응답 결과 신호를 포함한 정밀 탐지
- 웹 이벤트와 서버 후속 행위의 단일 흐름 복원
- 대규모 로컬 서비스 환경 맞춤형 빠른 운영 조정
- 한국어 기반 현장 실무와 영어 기반 보고 체계의 동시 충족

### 6-3. PLURA-XDR의 차별적 위치

PLURA-XDR은 글로벌 EDR/XDR의 강점을 부정하는 대신,  
그 위에 다음 가치를 더합니다.

- 웹 구간 입력 데이터의 깊이
- 응답 결과 해석 기반 정확도
- 웹-서버 단일 사건 흐름 복원
- MITRE ATT&CK 기준 설명력
- 자동 대응과 포렌식 연계
- 로컬 운영 민첩성

즉, PLURA-XDR은 단순 대체재가 아니라,  
**쿠팡과 같은 웹 중심 대규모 서비스 환경에서 더 깊은 분석과 설명이 필요한 영역을 담당하는 기술 플랫폼**으로 이해하는 것이 맞습니다.

## 7. 한국어·영어 동시 지원 로컬 회사라는 장점의 의미

PLURA-XDR의 또 하나의 중요한 장점은  
한국어와 영어를 모두 지원하는 로컬 기술 조직 기반이라는 점입니다.

이 점은 단순히 “국내 회사라 편하다”는 수준의 장점이 아닙니다.  
실제 보안 운영에서는 다음 차이를 만듭니다.

### 7-1. 국내 실무팀과 글로벌 이해관계자 모두를 지원할 수 있습니다

대규모 기업에서는 다음 두 축이 동시에 존재합니다.

- 한국어 중심의 운영 실무팀
- 영어 보고 또는 글로벌 컨설팅/감사 체계

PLURA-XDR은 이 둘을 동시에 지원할 수 있는 구조를 가집니다.

### 7-2. 정책 조정과 피드백 반영 속도가 빠릅니다

글로벌 제품은 기능이 강력하더라도  
고객별 세부 요구, 로컬 운영 관행, 특정 서비스 흐름에 대한 빠른 조정에는 시간이 걸릴 수 있습니다.

반면 로컬 기술 조직은 다음에서 강점을 가집니다.

- 빠른 룰/정책 조정
- 실환경 기반 튜닝
- 현장 피드백 반영
- 시차 없는 협업
- 한국어 기반 상세 대응

### 7-3. 사고 대응 품질이 달라집니다

사고는 결국 문서가 아니라 사람과 기술이 함께 대응하는 과정입니다.  
언어 장벽, 시차, 지원 경로의 복잡성은 실제 대응 품질에 영향을 줍니다.

PLURA-XDR은 한국어와 영어를 모두 지원하는 로컬 기술 조직 기반으로,  
국내 운영 환경에 대한 이해와 빠른 기술 대응을 동시에 제공합니다.  
이는 단순한 편의성이 아니라, **사고 대응과 PoC 운영, 정책 최적화 속도에서 실질적인 차이**를 만듭니다.

## 8. 제안의 핵심 메시지

이번 제안서의 핵심은 다음과 같이 요약할 수 있습니다.

### 첫째
글로벌 EDR/XDR 제품은 엔드포인트 행위 탐지와 자동화 측면에서 강점을 가집니다.

### 둘째
그러나 쿠팡과 같은 대규모 웹 서비스 환경에서는  
웹 요청 본문과 응답 결과 신호를 포함한 깊은 가시성이 없으면,  
실제 침해의 시작점과 결과를 충분히 설명하기 어렵습니다.

### 셋째
PLURA-XDR은 요청 본문, 응답 결과 신호, 서버 행위를 연결하여  
웹 기반 침해의 전후 맥락을 더 풍부하게 복원할 수 있습니다.

### 넷째
이는 단순한 탐지 기능의 차이가 아니라,  
**조사 속도, 설명 가능성, 오탐 제어, 대응 결정 품질의 차이**로 이어집니다.

### 다섯째
PLURA-XDR은 한국어와 영어를 모두 지원하는 로컬 기술 조직 기반으로,  
국내 대규모 서비스 환경에 맞춘 빠른 기술 대응과 운영 조정을 제공할 수 있습니다.

## 9. 결론

쿠팡과 같은 대규모 서비스 환경에서 중요한 것은  
단순히 더 많은 경보를 받는 것이 아닙니다.

중요한 것은 다음입니다.

- 웹 요청의 실제 입력을 볼 수 있는가
- 응답 결과를 해석할 수 있는가
- 그 결과가 서버에서 어떤 행위로 이어졌는가를 복원할 수 있는가
- 그 전체를 하나의 침해 흐름으로 설명할 수 있는가
- 그리고 그 설명이 실제 차단, 격리, 포렌식으로 이어질 수 있는가

PLURA-XDR의 차별성은 바로 여기에 있습니다.

PLURA-XDR은 글로벌 제품이 강한 엔드포인트 중심 탐지의 가치를 부정하지 않습니다.  
대신, 쿠팡과 같은 대규모 웹 서비스 환경에서 실제로 더 필요한  
**요청 본문 + 응답 결과 신호 + 서버 행위의 통합 분석**을 통해,  
침해의 시작과 진행, 후속 영향을 더 명확하게 설명하고 대응할 수 있도록 합니다. 

또한 한국어와 영어를 모두 지원하는 로컬 기술 조직 기반으로,  
국내 운영 환경에 적합한 빠른 정책 조정과 기술 대응을 제공할 수 있습니다.

즉, PLURA-XDR은 단순한 로컬 대체재가 아니라,  
**쿠팡과 같은 대규모 서비스 환경에서 글로벌 제품이 놓치기 쉬운 웹 기반 침해 맥락을 더 깊게 설명할 수 있는 보안 운영 플랫폼**입니다.

이것이 본 제안의 핵심입니다.
