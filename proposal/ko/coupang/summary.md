# PLURA-XDR 제안 — Executive Brief

*키 유출 및 퇴사/변경 가시성 공백 상황에서 Brute Force 인증 공격을 막기 위한 자율적 Layer-2 방어*

> **비즈니스 효과:** 대규모 크리덴셜 스터핑 및 정보 유출 시도를 조기에 차단·격리하여 재무적 손실과 법적/컴플라이언스 리스크(예: 보고 대상 사고)를 줄입니다.  
> 개발팀 리소스를 소모하지 않고(개발 사이클 낭비 없이) 보안을 강화할 수 있습니다.  
> **CISO 현실:** 보안이 “무엇이 바뀌었는지”를 누군가 알려줘야만 움직인다면, 정작 중요한 것을 놓치게 됩니다. 서명이 정상이라고 해서 악의가 아닌 것은 아닙니다.  
> **의사결정 요청(Ask):** 보안팀 주도 파일럿과 3가지 운영 승인(가시성, 데이터 통제, 단계적 통제)을 승인해 주십시오.

## 왜 지금 중요한가 (CISO 관점)

- 입·퇴사(오프보딩), 권한 변경, 배포/개발 변경은 보안 조직이 **늦게 인지**하는 경우가 많아, 구조적 공백이 발생합니다.
- 서명키/인증키가 통제된 환경 밖으로 나가면, 공격자는 **헤더/서명 수준에서는 정상처럼 보이는 요청**을 보낼 수 있습니다.
- Access Log(URL/응답코드/IP)만으로는 결정적인 증거가 부족한 경우가 많습니다. 신호는 대개 **요청 페이로드(POST Body)**와 **응답 결과**에 있습니다.

## 무엇을 제안하는가

보안 조직이 완벽한 협업을 기다리지 않고도 시스템 행위 신호로 Brute Force/자동화 시도를 탐지·통제할 수 있도록,  
자율 운영 표준인 **“관측(Observability) + 통제(Enforcement)”**를 구축합니다.

- 1차(필수): 키 교체/폐기(Key rotation / revocation)로 노출 범위를 축소
- 2차(제안): 행위 기반 탐지 + 단계적 통제(제한 → 추가 인증 → 차단/격리)로, **유효한 키로 서명된 요청**까지 포함해 Brute Force를 차단

## 한눈에 보기: 무엇이 달라지는가

| 키 교체만 하는 경우 | PLURA-XDR Layer-2 도입 시 |
| --- | --- |
| 교체 전·후 공백, 지연, 자동화 공격 지속 등 **노출 창(window)**이 남습니다. | 공백/지연 상황에서도 **행위 신호**로 조기 격리가 가능합니다. |
| 서명된 요청은 정상처럼 보일 수 있고, IP 분산은 IP 기반 룰을 약화시킵니다. | **임계치 + 세션 행위**를 키/계정/세션/엔드포인트로 상관분석하여(IP가 아닌) 통제합니다. |
| 빠른 트리아지 및 범위 산정을 위한 증거가 제한적입니다. | **요청 본문 + 결과 신호**로 신뢰도·트리아지 속도·조사 품질을 개선합니다. |

## 핵심 아이디어: 요청 본문 + 응답 결과 신호

- HTTPS 종료/복호화 지점(웹 서버 / WAF / API Gateway)에서 **요청 본문(POST Body)**과 **응답 결과 신호(성공/실패 + 사유)**를 수집합니다.
- 응답 본문 전체 보관이 불필요한 경우, **구조화된 결과 필드**만 수집하여 개인정보/성능/비용 리스크를 낮추면서도 통제 정확도를 높입니다.

## 탐지 방식(상위 수준)

- 1단계 — **임계치 트리거:** 동일 키/계정/세션 기준으로 실패·재시도가 정상 기준선을 초과
- 2단계 — **세션 행위 확증:** 세션 내에서 반복·자동화된 인증 또는 토큰 발급 흐름이 관측됨
- 통제는 **단계적**(제한 → 추가 인증 → 차단/격리)으로 적용하여 오탐을 줄이고 서비스 안정성을 유지합니다.

## 보안팀 주도 도입(개발 의존 최소)

본 접근은 애플리케이션 기능 개발을 기다리지 않고, 종료/복호화 지점의 설정과 정책 중심으로 보안 조직이 주도 실행하도록 설계됩니다.  
개발 조직의 참여는 “시작 조건”이 아니라, 이후 최적화를 위한 “선택 사항”입니다.

- 파일럿 범위: 고위험 엔드포인트(로그인 / 인증 / 토큰 발급)부터 시작
- 1–2주차: 종료 지점에서 수집 활성화(필드 마스킹/해싱, 접근 권한 분리, 보관기간 통제 포함)
- 2–3주차: 정상 기준선 수립, 임계치 + 세션 행위 탐지 룰 적용
- 3–4주차: 단계적 통제 적용, 오탐 통제(예외/롤백) 튜닝
- 파일럿 산출물: 정상 기준선 프로파일, 탐지·통제 정책 세트, SOC/IR 운영 플레이북

## 최소 승인(결정) 항목

- 종료/복호화 지점에서의 수집(요청 본문 + 결과 신호)을 **가시성 기반(visibility foundation)**으로 승인
- 민감정보 처리 원칙(마스킹/해싱, 접근 권한 분리, 보관기간) 승인으로 컴플라이언스 리스크 통제
- 단계적 통제 범위(제한 → 추가 인증 → 차단/격리) 승인으로, 운영 안전성을 확보하면서 신속 격리 가능

## 기대 효과

- 유효한 키로 서명된 요청까지 포함해 Brute Force 자동화를 탐지·격리
- 인사/배포 이벤트 인지에 대한 의존도를 낮추고, 행위 신호 기반으로 독자 운영 가능
- 구조화된 요청/결과 증거로 조사 품질 및 범위 산정 능력 향상
- WAF/SIEM/봇 방어 등 기존 체계를 대체하는 것이 아니라, 이들이 의존하는 **가시성 기반을 강화**하여 상호 보완
