## 1. 핵심력 강화 – 제품 포지셔닝 및 고도화 (AI 에이전트 포함)

### ✅ 1-1. AI 에이전트 기능 강화

* **보안 운영 자동화 에이전트(SOC Copilot)**
  웹로그·시스템 로그·이벤트 기반으로 수집된 데이터를 분석하여, AI 기반의 **이상 징후 판단 → 자동 티켓 생성 → 대응 가이드 제시**까지 자동화합니다.

  * ChatGPT API 또는 자체 AI 모델 연동으로 **자연어 질의/응답형 분석 인터페이스** 제공
  * “이 사용자 계정 이상행위 있었어?” 등 비전문가도 활용 가능한 **NLQ(Natural Language Query) 보안 분석 도우미** 제공

* **자동조치(Auto Response) 에이전트 탑재**

  * 위험도 등급에 따라 **방화벽 차단, 계정 잠금, 프로세스 종료, 관리자 통보** 등의 조치를 자동 실행
  * 시나리오 기반 플레이북을 사용자 정의할 수 있도록 지원하여, EDR 수준의 **실행력** 확보

* **PLURA Forensic + AI 상관분석 연계**
  기존 포렌식 모듈과 연결되어, AI가 판단한 위험 이벤트에 대해 **관련 증거 자동 수집 및 시각화 리포트**를 생성

---

### ✅ 1-2. 통합 보안 플랫폼 포지셔닝 강화

* **올인원 구조 정립**
  PLURA-XDR을 WAF, EDR, SIEM, Forensic, Auto Response를 하나로 통합한 **클라우드 기반 SECaaS**로 명확히 포지셔닝
  → “보안 인력이 없어도 대응 가능한 AI 기반 자동 보안 운영 플랫폼”이라는 메시지 전달

* **기능 분리형 서비스화**
  고객 규모에 따라 필요한 모듈만 선택 가능한 구조 도입 (예: WAF only / Forensic only / AI Agent only 등), SMB 공략에 유리

* **클라우드-온프레미스 혼합 구조 대응력 강화**
  다양한 환경(공공기관 폐쇄망 등)에 맞는 Agent 구성 옵션 제시

### ✅ 4-1. 퍼블릭 클라우드 MSP 대상 전략적 연동

* **MSP 파트너 연동 목적**
  AWS, Azure, GCP 등 주요 퍼블릭 클라우드에서 **고객의 워크로드 보호 및 클라우드 보안 강화 수요**가 확대되고 있음
  PLURA-XDR을 MSP의 보안 운영 패키지에 내장시켜 **Managed SECaaS(Security-as-a-Service)** 형태로 공급

* **적용 방식**

  * MSP가 운영 중인 \*\*보안 매니지드 상품(SOC, MDR 등)\*\*에 PLURA-XDR을 기술 스택으로 연동
  * 멀티 테넌시 구조를 지원하여 **다수 고객에 대한 로그 수집, AI 분석, 자동 대응**이 가능하도록 SaaS 방식 배포
  * 고객사 클라우드 인프라에 Agent 설치 또는 API 연동을 통해 비침투형 적용 가능

  * PLURA-XDR의 **Agentless 로그 분석 기반** 탐지 기술은 중앙 집중형 관리에 적합
  * NIA, NIRS, KISA 산하 공공 클라우드 및 보안 사업에 참여

### ✅ 5-2. 국방부 및 방위사업청 전략

> **폐쇄망 환경**, **공급망 보안**, **산하기관 및 방산업체 연계 분석 역량**이 핵심

#### 🔹 국방부 산하기관 대상

| 기관                        | 주요 보안 수요                      | PLURA-XDR 적용 방향                       |
| ------------------------- | ----------------------------- | ------------------------------------- |
| **국방부 본부 및 사이버작전사령부**     | 군사망/업무망 이원화, 내부 계정 남용 및 침해 탐지 | **웹로그·시스템로그 기반 내부 위협 탐지 시스템 구축**      |
| **육·해·공군 각군 본부 및 통합전산센터** | 시스템 간 비인가 접속, 계정 도용, 파일 탈취 등  | **공격 시나리오 기반 AI 분석 및 포렌식 보고서 자동화**    |
| **정보체계관리단, 군수사**          | 장비 수명관리 시스템 및 무기체계 제어 시스템 보안  | **비정상 행위 시그니처 탐지 + 외부 연결 탐지 (백도어 등)** |

#### 🔹 방위사업청(방사청) 대상

* **무기체계 개발·구축사업에 포함되는 정보보호 요구 대응**

  * PLURA-XDR은 **탑재형 무기체계가 아닌, 구축형 체계**의 부대 연계 시스템 보안 구성 시 적용 가능
  * 탐지 범위: 협력사 개발망 → 시범부대 연동망 → 실운영망 간 연결 구간에서의 이상행위 탐지

* **무기체계 공급망 보안 대응 방안**

  * 한화, LIG넥스원, 현대로템 등 **방산업체와 군 사이의 연계 시스템 로그 분석**
  * 예: 무기개발 설계포털, 문서교환시스템, 형상관리 서버 등의 웹 기반 시스템에 대한 로그 기반 AI 이상행위 탐지
  * **공급망 공격 (협력사 계정 탈취 → 군 내부 시스템 접근)** 시나리오 대응

### ✅ 5-3. 방위산업체 협력 연계

* PLURA-XDR을 적용한 **한화 디펜스, 현대로템 등 방산업체 대상 공급망 보안 모델** 구축
* 방산 ERP, PLM, 설계 협업툴 등에 대한 AI 기반 웹 접근 분석
* 국방부와 방산업체 간 파일 공유, 인증 모듈, 사용자 세션 로그 등에 대한 **비인가 접근/권한 상승 시도 탐지**
