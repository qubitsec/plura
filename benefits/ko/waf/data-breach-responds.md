# PLURA-XDR — **요청/응답 본문 기반 실시간 데이터 유출 탐지/차단/증적화**

> **한 줄 요약**: PLURA-XDR은  
> 단순 접속 로그만이 아니라 **웹 요청 본문과 응답 본문을 직접 분석**해  
> 개인정보·기밀정보가 외부로 새어 나가는 순간을 **정확히 잡고**,  
> **자동 차단 + 시나리오 묶음 + 증적화**까지 완료하는 **운영형 데이터 유출 방어 모듈**입니다.

---

# 왜 PLURA-XDR 데이터 유출 탐지인가

## **1) “정상 200 응답” 속 유출도 잡는 — 요청/응답 본문 정탐**

쿠팡 사례처럼 인증 취약점·내부자 악용으로
DB가 통째로 조회되어도 응답은 **HTTP 200 정상**일 수 있습니다.
또한, 로그인/검색/파일 업로드 등 **요청 본문** 안에서
이미 대량 계정·민감정보가 흘러들어갈 수도 있습니다.

PLURA는:

* **웹 요청/응답 본문을 직접 분석**
* 이름·전화번호·주소·이메일·주민번호 등 **민감정보 패턴 탐지**
* SQLi·디렉터리 리스팅·WebShell 결과 등 **데이터 노출 패턴 식별**
* 탐지 즉시 **차단·속도제한·알림 정책 자동 연계**

👉 “수개월 간 조용히 빠져나간 대량 조회”도
**“초기 몇 분~몇 시간 안에” 이상 요청/응답 패턴으로 포착**할 수 있음.

---

## **2) 대량 조회를 “데이터 유출 인시던트”로 자동 묶어내는 시나리오 엔진**

PLURA-XDR은 단일 이벤트가 아니라 **패턴을 조합**합니다.

* 비정상적으로 큰 응답 크기(Resp-Size)
* 짧은 시간 안의 **수만·수십만 건 조회 패턴**
* 요청/응답 본문 내 **민감정보 패턴 포함 여부**

이 신호들을 결합해,

> **“데이터 유출 시나리오”로 자동 묶음 → 경보 → 정책 기반 차단**

관리자 분석 없이도 **시스템이 직접 차단**하도록 설계.

👉 쿠팡과 같은 “해외 서버 기반 대량 조회”는
**요청/응답 본문 + 조회량 + 국가/경로 + 패턴**으로 즉시 인시던트 생성.

---

## **3) 사고 후에도 ‘언제·무엇이·얼마나’가 보이는 증적화**

PLURA의 강점은 탐지뿐 아니라 **사후 증거(Forensic Evidence)** 확보입니다.

* 최초 유출 시점
* 유출된 계정 수
* 유출된 필드(name/email/address 등)
* 공격 IP/국가/경로
* 응답 본문 일부(마스킹 적용)
* 전체 타임라인(Log Timeline)

이 모든 증거를 **플랫폼 내에서 자동 보관·시각화**하여,

> “유출 여부 → 경로 차단 → 유출 범위 산정(증적화)”
> 를 **한 번에 처리**할 수 있음.

👉 수사·규제기관 보고·고객 통지에서
**정확한 근거 데이터 제공**이 가능해짐.

---

# 3대 핵심 가치 (정탐↑ · 대응속도↑ · 증거품질↑)

1. **정탐↑ — 요청/응답 본문 기반 실시간 민감정보 탐지**
2. **대응속도↑ — 해외 IP·대량 응답·패턴을 자동 시나리오화 및 차단**
3. **증거품질↑ — 유출 타임라인·필드·범위까지 완전 증적화**

---

# 1페이지 요약 (슬라이드용 3줄)

1. **정상처럼 보이는 HTTP 200 응답에서도, 요청/응답 본문을 분석해 민감정보 유출을 즉시 포착합니다.**
2. **해외 IP + 대용량 응답 + 반복 조회를 ‘데이터 유출 시나리오’로 자동 묶어 차단까지 수행합니다.**
3. **유출 시점·경로·범위를 완전 증적화하여 보고·사후 대응을 단일 플랫폼에서 처리합니다.**
