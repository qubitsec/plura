# PLURA-XDR 지자체·공공기관 제품 정보

**웹·서버·PC의 공격을 통합적으로 탐지·차단·분석하는 클라우드 기반 XDR 플랫폼**

**문서 성격:** 이 문서는 지자체·공공기관을 대상으로 PLURA-XDR의 주요 기능과 도입 가치를 설명하고, 현재 확인된 경쟁 솔루션의 등록 설명과 비교하기 위한 제품 안내 자료입니다.

**활용 대상:**

- 지자체·공공기관 정보보호 담당자
- 정보화·전산 운영 담당자
- 보안관제 담당자
- 조달·구매 담당자
- Sales, 파트너 영업 및 제안 담당자

**핵심 메시지:**

> “PLURA-XDR은 기존 보안 솔루션의 로그를 모아 위험을 확인하는 수준을 넘어, 웹·서버·PC에서 발생하는 공격을 직접 탐지하고 차단하며, 사고 발생 시 원본 로그와 포렌식 증거까지 확인할 수 있는 통합 사이버보안 플랫폼입니다.”

## 1. 지자체·공공기관 대상 솔루션 요약

### 1.1 왜 지자체·공공기관에 필요한가

지자체와 공공기관은 홈페이지, 대민 서비스, 업무 시스템, 서버, 사용자 PC와 개인정보 처리 시스템을 함께 운영합니다.

공격자는 하나의 경로만 이용하지 않습니다.

- 홈페이지와 대민 웹서비스의 취약점 공격
- 관리자·사용자 계정 탈취
- 서버 침투와 권한 상승
- 악성코드와 랜섬웨어 실행
- 내부 시스템으로의 횡적 이동
- 개인정보와 중요 자료의 외부 유출
- 정상 사용자 또는 내부 계정으로 위장한 이상 행위

각 영역의 보안 제품과 로그가 서로 분리되어 있으면, 개별 경보는 확인할 수 있어도 공격이 어디서 시작되어 어디까지 진행되었는지 전체 흐름을 파악하기 어렵습니다.

PLURA-XDR은 웹·서버·PC에서 발생하는 보안 이벤트를 하나의 플랫폼으로 연결하여 탐지, 차단, 분석, 포렌식과 대응을 지원합니다.

### 1.2 한 문장 소개

> “PLURA-XDR은 지자체와 공공기관의 웹서비스, 서버, PC에서 발생하는 해킹 시도를 한눈에 보고 탐지·차단·분석하는 클라우드 기반 통합 사이버보안 플랫폼입니다.”

조금 더 짧게 설명할 때는 아래 문장을 사용합니다.

> “PLURA-XDR은 보이지 않던 공격을 보이게 하고, 공격의 시작부터 대응 결과까지 하나의 흐름으로 확인하게 합니다.”

### 1.3 주요 제공 기능

| 구분 | 주요 기능 | 지자체·공공기관 활용 |
|---|---|---|
| WAF | 웹 공격 탐지 및 차단 | 홈페이지와 대민 웹서비스의 취약점 공격, 변형 공격과 비정상 요청 대응 |
| EDR | 서버·PC 이상 행위 탐지 및 대응 | 악성 프로세스, 계정 오남용, 권한 상승, 악성코드와 랜섬웨어 징후 확인 |
| SIEM | 보안 로그 통합 수집·분석 | 웹·서버·PC와 주요 시스템 로그의 통합 검색 및 상관분석 |
| XDR | 공격 흐름 통합 분석 | 웹 공격 이후 서버·PC에서 이어지는 침해 행위를 하나의 흐름으로 분석 |
| Forensic | 사고 흔적과 증거 분석 | 사고 발생 시 원본 로그, 행위 기록과 포렌식 증거 확보 |
| SOAR | 자동 대응 | 공격자 IP 차단 등 반복 대응 업무 자동화 |
| VAS | 취약 설정과 위험 요소 점검 | 서버와 시스템의 취약한 설정 및 보안 위험 확인 |
| sysMon | 시스템 리소스 모니터링 | CPU, 메모리, 디스크와 네트워크 이상을 장애·공격 징후와 함께 확인 |
| SOC | 원격보안관제 | 전문 보안 인력이 공격 징후를 함께 확인하고 대응을 지원 |

### 1.4 지자체·공공기관에 제공하는 핵심 가치

#### 1) 웹서비스 공격 탐지와 차단

지자체 홈페이지와 대민 서비스는 외부에 항상 공개되어 있어 지속적인 공격 대상이 됩니다.

PLURA-XDR은 웹 요청과 공격 코드를 분석하여 SQL Injection, XSS, 파일 업로드 공격, 웹셸 시도, 취약점 공격과 변형·제로데이 의심 공격을 탐지하고 차단할 수 있습니다.

#### 2) 서버와 PC의 침해 행위 확인

웹방화벽을 통과한 공격이나 계정 탈취 이후에는 서버와 PC에서 실제 행위가 발생합니다.

PLURA-XDR은 로그인, 프로세스, 파일, 권한, 네트워크 연결과 감사 로그를 분석하여 서버 침해, 내부 이동, 악성코드와 랜섬웨어 징후를 확인합니다.

#### 3) 개인정보 침해와 데이터 유출 대응

개인정보 처리 시스템에 대한 비정상 접근과 대량 조회, 비정상 응답, 외부 전송 징후를 분석하여 개인정보 침해 가능성을 확인할 수 있습니다.

웹 요청뿐 아니라 응답 데이터와 서버 행위를 함께 확인함으로써 공격 성공 여부와 실제 유출 가능성을 판단하는 근거를 제공합니다.

#### 4) 사고 대응 증거 확보

보안 사고가 발생하면 단순 경보만으로는 원인과 피해 범위를 설명하기 어렵습니다.

PLURA-XDR은 원본 로그, 탐지 결과, 차단 이력과 포렌식 정보를 함께 제공하여 내부 보고, 감사, 사고 조사와 재발 방지 대책 수립에 필요한 증거를 확보할 수 있도록 지원합니다.

#### 5) 보안 인력 부족 보완

지자체와 공공기관은 제한된 인력으로 다양한 시스템을 운영해야 합니다.

PLURA-XDR과 원격보안관제를 함께 이용하면 전문 보안 인력이 웹·서버·PC의 공격 징후를 함께 확인하고 대응을 지원하므로, 내부 담당자의 운영 부담을 줄일 수 있습니다.

### 2.3 핵심 차이

#### AhnLab XDR

제공된 등록 설명을 기준으로 보면, AhnLab XDR은 기존에 구축·운영 중인 보안 솔루션과 이메일 등에서 생성되는 로그를 수집하여 리스크의 우선순위를 파악하고 관리하는 데 초점을 둡니다.

즉, 여러 시스템에서 발생한 경보와 로그 가운데 어떤 위험을 먼저 확인하고 조치해야 하는지를 판단하는 **리스크 관리형 XDR**로 설명할 수 있습니다.

#### PLURA-XDR

PLURA-XDR은 웹·서버·PC에서 발생하는 원본 데이터를 직접 수집·분석하고, 공격 탐지와 차단부터 사고 흔적 분석과 자동 대응까지 하나의 플랫폼에서 제공합니다.

즉, 기존 보안 솔루션의 로그를 통합하는 것뿐 아니라 WAF, EDR, SIEM, Forensic, SOAR, VAS와 sysMon 기능을 직접 제공하는 **탐지·차단·분석·대응 통합형 XDR**입니다.

### 2.4 지자체 제안 시 강조할 차별점

#### 1) 로그 우선순위 관리에서 실제 공격 대응까지

> “경쟁 제품이 여러 보안 시스템의 로그를 바탕으로 확인해야 할 리스크의 우선순위를 관리한다면, PLURA-XDR은 웹·서버·PC에서 발생하는 공격을 직접 탐지하고 차단하며, 사고가 발생했을 때 원본 로그와 포렌식 증거까지 확인할 수 있습니다.”

#### 2) WAF와 EDR이 포함된 통합 플랫폼

> “PLURA-XDR은 외부 보안 솔루션의 로그 연동만을 전제로 하지 않습니다. 웹방화벽과 서버·PC 보안 기능을 플랫폼 안에서 함께 제공하여 공격의 시작부터 내부 행위까지 연결해서 볼 수 있습니다.”

#### 3) 웹 공격 성공 여부와 침해 범위 확인

> “웹 공격 경보만 확인하는 것이 아니라, 공격 이후 서버에서 어떤 프로세스와 파일, 계정 행위가 발생했는지 확인하여 실제 침해 여부와 영향 범위를 판단할 수 있습니다.”

#### 4) 개인정보 유출과 사고 조사 지원

> “지자체와 공공기관에서 중요한 것은 경보의 수가 아니라, 개인정보가 실제로 조회·유출되었는지와 어떤 대응을 수행했는지를 증거로 설명할 수 있는가입니다. PLURA-XDR은 원본 로그와 포렌식 정보를 함께 제공합니다.”

#### 5) 제품과 전문 관제의 결합

> “제품만 공급하는 방식이 아니라, 필요 시 전문 보안 인력이 고객 시스템의 공격 징후를 함께 확인하고 대응을 지원하는 원격보안관제 서비스를 제공합니다.”

### 2.5 경쟁 제품 비교 시 주의할 점

경쟁 제품을 설명할 때 지원 여부가 확인되지 않은 기능을 단정해서는 안 됩니다.

하지 말아야 할 표현:

- “AhnLab XDR은 웹 공격을 탐지하지 못합니다.”
- “AhnLab XDR에는 EDR이 없습니다.”
- “경쟁 제품은 대응 기능이 없습니다.”
- “PLURA-XDR이 모든 면에서 더 우수합니다.”

대신 아래처럼 설명합니다.

> “현재 등록된 상품 설명에서는 기존 보안 솔루션과 이메일 등의 로그를 기반으로 리스크 우선순위를 관리하는 기능이 중심으로 소개되어 있습니다. 웹방화벽, 포렌식, 취약점 점검과 원격보안관제 등의 구체적인 제공 범위는 별도 확인이 필요합니다.”
