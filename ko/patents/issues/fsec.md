## 2. 당시 제안의 핵심 목적

첨부된 금융보안원 대상 제안서는 제목과 구성상 **Cyber Security Cloud SaaS / Next Generation SIEM**, **금융권 Cloud SIEM SaaS**, **금융권 웹 통합보안관제** 방향의 제안서입니다. 제안서에는 금융ISAC의 정보공유체계 문제점으로, 금융ISAC이 개별 금융기관보다 수집 가능한 정보의 폭이 좁아 이상징후의 정상 여부를 판별하기 어렵다는 점이 기재되어 있고, 개별 금융기관 측에는 네트워크 트래픽, 네트워크 공격 시도, APT 공격 탐지 내역, 시스템 로그, 웹 서버 로그, 데이터베이스 로그, 기타 시스템 로그 등이 존재한다고 정리되어 있습니다. 

즉, 제안의 배경은 금융보안원이 기존처럼 제한적인 정보만으로 관제하는 것이 아니라, **금융회사 내부의 웹 서버 로그, 시스템 로그, 데이터베이스 로그 등 더 넓은 범위의 로그를 통합적으로 수집·분석해야 놓친 공격을 판별할 수 있다**는 문제의식으로 정리됩니다. 이는 첨부 제품소개서의 “실시간 로그분석을 통하여 해킹 탐지하는 차세대 SIEM - 프루라(PLURA)”라는 제품 설명과도 연결됩니다. 

## 3. “웹 요청 본문 분석” 제안이 명확히 포함된 부분

첨부 제품소개서에는 일반적인 웹 접속 로그 수준을 넘어, **웹 요청 본문에 해당하는 Post-Body를 로깅하고 분석하여 웹 공격을 탐지한다**는 내용이 반복적으로 제시되어 있습니다. 특히 제품소개서의 웹 로그 분석 파트는 “웹트래픽 분석을 통한 POST-BODY 로깅 지원으로 해킹 탐지”라고 명시하고, 웹방화벽이 웹쉘, SQL Injection, XSS 등을 막지 못하는 경우 공격 패킷이 내부 서버로 유입될 수밖에 없다고 설명합니다. 이어서 **HTTP 및 HTTPS에 대하여 Post-Body로 들어온 웹 공격을 로그로 남겨 웹 공격을 탐지한다**고 기재되어 있습니다. 

또한 동일 자료는 구현 방식도 구체화하고 있습니다. 에이전트 설치형 방식, Proxy 방식, 포트미러링 기반 Agentless 방식이 제시되어 있으며, 포트미러링 방식에서는 웹서버에 별도 프로그램을 설치하지 않고 별도 어플라이언스 서버에서 HTTP 웹트래픽을 분석하여 **POST-BODY 로그를 취합**한다고 설명합니다. 이 부분은 단순한 “웹보안 필요성” 수준이 아니라, **웹 요청 본문 수집·로깅·분석 방식 자체를 금융보안원에 제공된 자료에서 구체적으로 제안했다**는 근거로 정리할 수 있습니다. 

## 4. “놓친 공격을 탐지·차단할 수 있다”는 제안 취지

제품소개서는 기존 사후 감사 방식의 한계를 설명하면서, 침해사고 후 로그 분석은 이미 피해가 발생한 이후이며 로그 위·변조 가능성 및 분석 시간 문제가 있다고 지적합니다. 그 대안으로 “이제는 실시간으로 로그를 분석해야 한다”고 하며, PLURA는 실시간으로 로그를 분석하여 해킹을 탐지 및 방어하고 이메일과 앱 푸시로 위험과 방어를 보고한다고 설명합니다. 

또한 PLURA의 서비스 설명에는 **실시간 로그분석을 통하여 이상로그를 탐지하고 방어**한다고 되어 있고, 웹 서버 보안 영역에서는 인젝션, 인증 및 세션 관리 취약점, 민감 데이터 노출, XML 외부개체, 접근통제 취약점, 보안설정 오류, XSS, 불충분한 로깅 및 모니터링 등 OWASP Top 10 유형에 맞춰 웹 서버 로그를 분석한다고 정리되어 있습니다. 

나아가 기능 소개에서는 서버 시스템과 웹서버의 로그를 분석하여 종합적인 해킹 탐지를 수행하고, 시그니처와 통계 기반 복합분석으로 정확성을 높이며, 서버 방화벽 기능과 연동하여 위험 IP 주소를 자동 차단한다고 설명합니다. 따라서 당시 제안의 핵심은 **웹 요청 본문/Post-Body를 포함한 웹 로그를 수집·분석해야 웹방화벽 등 기존 장비가 놓친 공격을 탐지하고, 필요 시 차단까지 할 수 있다**는 내용으로 정리됩니다. 

## 5. 특허 및 선행 권리화 기록

제품소개서에는 제안 당시 이미 국내 특허 등록 및 글로벌 출원 현황이 명시되어 있습니다. 자료에는 등록번호 **10-1896267**, 특허등록일 **2018년 9월 3일**, 명칭 “실시간로그분석기반의 공격탐지시스템 및 방법”이 기재되어 있고, 해당 기술은 **시스템 로그와 웹 로그(Post-body)에 대한 실시간 로깅 기술을 이용하여 탐지**한다고 설명합니다. 또한 등록번호 **10-1909957**, 특허등록일 **2018년 10월 15일**, 명칭 “실시간웹해킹탐지를위한 웹트래픽로깅시스템 및 방법”이 기재되어 있으며, 실시간 웹 로깅 4가지 기법 및 포트미러링 기법이 포함된다고 되어 있습니다. PCT 출원번호 **PCT/KR2018/010450**도 함께 기재되어 있습니다. 

이 점은 중요합니다. 메일 송부일인 **2018년 11월 16일**보다 앞서, 관련 핵심 기술이 이미 국내 특허로 등록되어 있거나 PCT 출원되어 있었다는 자료가 금융보안원에 송부된 제품소개서 안에 포함되어 있었기 때문입니다. 따라서 기술탈취 또는 특허침해 주장을 검토할 때, “아이디어 제안 시점”뿐 아니라 “권리화 선행 시점”을 함께 주장할 수 있는 구조입니다.  

## 6. 증거기록용 요약 문안

아래 문안은 사건 정리서나 내부 기록에 그대로 사용할 수 있는 형식입니다.

> 큐비트시큐리티 주식회사는 2018년 11월 14일 전후 금융보안원을 대상으로 금융권 웹 통합보안관제 및 Cloud SIEM SaaS 관련 제안 발표를 진행하였고, 2018년 11월 16일 12:43경 금융보안원 도메인의 수신자 4인에게 “[큐비트시큐리티] 제품 소개서와 제안서 재중”이라는 제목으로 발표 시 사용한 회사 소개 자료와 제안서를 이메일로 송부하였다. 해당 메일에는 `20181114_프루라(PLURA)_제품소개서-29f.pdf` 및 `20181114_금융보안원_클라우드SaaS 금융권웹통합보안관제_제안서f.pdf` 두 개의 파일이 첨부되어 있었다. 
>
> 위 첨부자료에는 금융보안원의 기존 정보공유체계가 개별 금융기관보다 수집 가능한 정보의 폭이 좁아 이상징후 판별에 한계가 있다는 문제의식이 포함되어 있고, 금융기관 측의 웹 서버 로그, 시스템 로그, 데이터베이스 로그 등 다양한 로그를 통합적으로 수집·분석해야 한다는 취지가 제시되어 있다. 
>
> 특히 PLURA 제품소개서에는 웹방화벽이 막지 못한 웹쉘, SQL Injection, XSS 등 웹 공격이 내부 서버로 유입될 수 있으므로, HTTP 및 HTTPS의 Post-Body, 즉 웹 요청 본문으로 들어온 웹 공격을 로그로 남기고 분석하여 탐지해야 한다는 내용이 명시되어 있다. 또한 에이전트 설치형, Proxy 방식, 포트미러링 기반 Agentless 방식 등 Post-Body 로깅 및 웹트래픽 분석 구현 방식도 구체적으로 제시되어 있다. 
>
> 따라서 큐비트시큐리티가 금융보안원에 제안한 핵심은 단순 웹 관제 또는 일반 로그 수집이 아니라, 금융권 웹 통합보안관제 환경에서 웹 요청 본문(Post-Body)을 포함한 웹 로그를 실시간으로 수집·분석하여 기존 보안장비가 놓친 공격을 탐지하고, 필요 시 차단까지 수행하는 차세대 SIEM/Cloud SaaS 구조였다.  
>
> 또한 위 제품소개서에는 관련 특허인 등록번호 10-1896267 및 10-1909957, PCT/KR2018/010450이 명시되어 있어, 제안 당시 큐비트시큐리티가 웹 로그(Post-body) 실시간 로깅, 웹트래픽 로깅, 포트미러링 기반 탐지 기술에 관하여 이미 국내 특허 등록 및 국제 출원을 진행한 상태였음을 확인할 수 있다. 

## 7. 현재 주장과 첨부자료로 입증되는 범위

첨부자료만으로 명확히 입증되는 부분은 다음입니다.

첫째, 큐비트시큐리티가 2018년 11월 금융보안원에 제안 발표를 했고, 발표자료와 제안서를 금융보안원 수신자들에게 이메일로 송부했다는 점입니다. 둘째, 그 자료 안에 **웹 요청 본문(Post-Body) 로깅·분석**, **GET/POST 웹 로그 분석**, **웹 공격 탐지**, **실시간 탐지·방어**, **위험 IP 자동 차단**, **금융권 Cloud SIEM SaaS / 웹 통합보안관제** 구상이 포함되어 있었다는 점입니다. 셋째, 해당 핵심기술과 관련된 특허 등록·출원 정보가 제안 당시 제품소개서에 포함되어 있었다는 점입니다.   

다만, **금융보안원이 이후 동일 또는 유사 아이디어를 자사 서비스에 포함하여 금융권에 제공하고 있다는 점**, 그리고 **KB손해보험 김진규 상무/CISO가 유사 서비스를 금융보안원으로부터 제공받고 있다는 점**은 현재 첨부된 3개 문서 안에서는 직접 확인되지 않습니다. 이 부분은 별도 증거, 예를 들면 금융보안원 서비스 소개자료, KB손해보험 측 확인 메일·회의록·녹취·제안요청서·서비스 화면·계약/이용 내역 등을 추가로 확보하여 위 2018년 제안자료와 항목별로 대조하는 방식이 필요합니다.

법적 문안으로는 “기술탈취 및 특허침해가 확정된다”라고 단정하기보다, 현재 단계에서는 **“큐비트시큐리티가 2018년 11월 금융보안원에 제안·공개한 웹 요청 본문(Post-Body) 분석 기반 웹 공격 탐지·차단 기술 및 관련 특허기술과, 금융보안원이 이후 금융권에 제공 중인 것으로 확인되는 유사 서비스 사이에 기술적 동일·유사성이 존재할 경우, 기술탈취 및 특허침해 여부를 검토할 필요가 있다”**라고 정리하는 것이 증거기록상 안전합니다.
