# PLURA V6 업그레이드 제안

## 1. 제안 배경

현재 PLURA V5를 이용 중인 고객사를 대상으로 최신 버전인 **PLURA V6 업그레이드**를 제안드립니다.

최근 웹 공격, 취약점 공격, 자동화 공격 시도가 지속적이고 과도하게 발생하고 있으며, 알려진 취약점뿐 아니라 신규 취약점 및 제로데이성 공격으로 확대될 가능성도 높아지고 있습니다.

이에 따라 보다 세분화된 탐지, 자동 차단, AI 기반 분석, 포렌식 연계를 제공하는 **PLURA V6 기반의 자동화 대응 체계**로 전환할 필요가 있습니다.

## 2. PLURA V6 주요 강화 기능

| 구분      | 주요 강화 내용                                                              | 참고 문서                                                                                                                                                                                                                                                                                                             |
| ------- | --------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 웹방화벽    | 자동 IP 차단 기능 세분화, 공격 탐지 룰셋 업데이트, CVE 기반 취약점 공격 탐지, 제로데이 의심 공격 탐지 기능 강화 | [자동 IP 차단](https://docs.plura.io/ko/v6/fn/comm/ipblock/auto), [제로데이 탐지](https://docs.plura.io/ko/v6/fn/comm/sdetection/zeroday)                                                                                                                                                                                   |
| EDR/호스트 | 필터 탐지 시 자동 포렌식 연동, 기본 시스템 정보 조회, AV 및 Microsoft Defender 상태 조회        | [자동 포렌식](https://docs.plura.io/ko/v6/fn/comm/systemmgmt/system/forensic), [EDR](https://docs.plura.io/ko/v6/fn/comm/edr), [시스템 정보](https://docs.plura.io/ko/v6/fn/comm/systemmgmt/system#id-2), [SecureOS](https://docs.plura.io/ko/v6/fn/comm/secureos), [시스템 보안](https://docs.plura.io/ko/v6/fn/comm/systemsec) |

## 3. AI 기반 공격 자동화 대응

PLURA V6는 최신 **AI 기반 공격 자동화 대응 기능**을 통해 반복적이고 고도화되는 공격에 대해 보다 신속한 탐지와 대응이 가능하도록 개선되었습니다.

이를 통해 단순 탐지 중심의 운영에서 벗어나, 공격 이벤트에 대한 분석, 차단, 포렌식 연계까지 보다 자동화된 보안 운영 체계로 전환할 수 있습니다.

## 4. 사전 준비 사항

PLURA V6의 AI 기반 탐지 및 자동화 기능을 원활히 이용하기 위해서는 고객사 명의의 외부 서비스 계정 및 API Key 준비가 필요합니다.

| 서비스                 | 용도          | 준비 사항                                                                                           |
| ------------------- | ----------- | ----------------------------------------------------------------------------------------------- |
| VirusTotal          | PLURA-TI 연동 | 고객사 명의 계정 및 API Key 준비<br>[API 생성 가이드](https://github.com/QubitSecurity/howto/tree/main/api/ti) |
| OpenAI API Platform | PLURA-AI 연동 | 고객사 명의 계정 및 API Key 준비<br>[API 생성 가이드](https://github.com/QubitSecurity/howto/tree/main/api/ai) |

## 5. 업그레이드 시 주의사항

PLURA V6는 V5와 아키텍처가 상이하므로, 기존 V5 환경에서 이용 중인 시스템 에이전트는 모두 삭제한 후 V6 버전으로 새롭게 설치해야 정상적인 이용이 가능합니다.

서비스 가입 및 API Key 준비가 완료되면, 기존 V5 에이전트 삭제 및 PLURA V6 에이전트 설치 절차를 순차적으로 안내드립니다.

* [PLURA V6 회원가입 매뉴얼](https://docs.plura.io/ko/v6/signup)

## 6. 기대 효과

PLURA V6 업그레이드를 통해 고객사는 최신 공격에 대한 탐지 정확도 향상, 자동 차단 및 포렌식 연계 강화, AI 기반 분석 체계 도입을 통해 보다 신속하고 체계적인 보안 대응이 가능합니다.
