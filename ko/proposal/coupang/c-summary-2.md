# 쿠팡 개인정보 침해 재발방지 및 과징금 취소·감경 소송 지원을 위한

# **PLURA-XDR AI 기반 통합 방어·증거체계 구축 요약 제안서**

## 인증키·내부자 위협·위조 토큰·AI 해킹 대응과 법원 제출용 증거패키지의 일체화

> **“키를 가진 사람이 누구이든, 공격이 AI로 얼마나 빠르게 변형되든, 오남용은 탐지·차단되고 모든 대응은 법원이 검증할 수 있는 증거로 남는 체계”**

> **법률상 유보**  
> 본 제안서는 개인정보보호위원회의 공개자료와 처분사유에서 지적된 기술적 위험요인을 전제로 재발방지, 시정명령 이행, 증거체계 및 소송지원 방안을 제시합니다. 본문의 기술적 개선·시험·운영확약은 쿠팡이 행정소송에서 다투는 사실관계, 피해 규모, 인과관계, 법적 책임 또는 과징금 산정의 적법성을 인정하거나 자백하는 취지가 아닙니다. 실제 법원 제출 문구는 최종 처분서·의결서·조사기록·원시증거를 대입한 뒤 소송대리인이 확정합니다.

## 1. 제안의 결론

쿠팡에 필요한 것은 보안제품을 하나 더 구매하는 일이 아닙니다. 처분기관이 지적한 위험요인을 구조적으로 통제하고, 실제 운영효과를 변조 방지 원본증거로 남기며, 외부 전문가가 이를 독립적으로 검증하고, 그 결과를 법원 제출용 준비서면과 증거설명서로 완성하는 **통합 방어·증거·소송지원 프로젝트**가 필요합니다.

본 사업은 다음 네 가지 산출물을 하나의 사업으로 완성합니다.

| 최종 산출물 | 핵심 내용 |
|---|---|
| 통합 방어체계 | HSM·KMS, IAM·PAM, 인사·인증·API·WAF·EDR·데이터 로그와 PLURA-XDR AI 연계 |
| 법적 증거체계 | 원본 로그, 설정·모델 버전, 해시값, 시간동기화, 접근기록, 증거 인수인계 절차 |
| 독립 검증체계 | 위조 토큰·내부자 위협·저속 분산조회·미토스급 AI 공격 재현시험과 외부 전문가 검증 |
| 소송 문서체계 | 준비서면 초안, 쟁점별 입증표, 증거목록·증거설명서, 전문가 의견서와 변론자료 |

### 통합 방어·증거화 흐름

```text
키·권한 → 토큰 발급·사용 → API 호출 → 개인정보 응답 → 외부 통신
                 ↓ PLURA-XDR AI 상관분석
위험도·판단근거 → 차단·격리 → 원본 로그·해시 보전 → 독립검증
                 ↓
        준비서면·증거목록·증거번호
```

> **위험요인을 통제하고, 공격을 멈추고, 증거를 보전하며, 법률적 주장으로 입증합니다.**

# 2. 문제 인식과 재발방지 원칙

이번 개편은 하나의 키를 더 안전하게 보관하는 수준에 머물러서는 안 됩니다. 하나의 키, 하나의 계정 또는 하나의 통제가 실패하더라도 장기간의 대규모 개인정보 침해로 확대되지 않는 구조를 만들어야 합니다.

퇴직자 대응만으로도 충분하지 않습니다. 재직자·협력사가 고의로 키를 오남용하거나 외부 공격자에게 제공할 수 있고, 정상 계정과 단말이 탈취될 수도 있습니다. 따라서 사람의 신분이나 선의를 신뢰하는 보안이 아니라, **누가 키를 가졌더라도 실제 행위가 승인된 업무와 일치하는지를 지속적으로 검증하는 보안**으로 전환해야 합니다.

## 다섯 가지 전환

| 기존 접근 | 전환 방향 |
|---|---|
| 인증 성공 여부 중심 | 인증 이후의 토큰 사용·API 호출·데이터 조회까지 보는 행위 중심 보안 |
| 재직자·내부 계정에 대한 신뢰 | 정상 권한도 업무 목적·승인·기기·시간·사용량과 일치하는지 지속 검증 |
| 개별 보안장비 중심 | 키·계정·토큰·웹·서버·데이터·외부 통신을 하나의 사건으로 연결 |
| 구축계획·완료보고 중심 | 원시로그·차단이력·재현시험·독립감사로 운영효과를 증명 |
| 사람이 경보를 순차 검토 | PLURA-XDR AI가 공격 흐름을 실시간 상관분석하고 안전하게 대응 |

## 책임 있는 운영 확약

쿠팡이 정부·법원·고객에게 제시할 약속은 “앞으로 어떠한 해킹도 절대 발생하지 않는다”는 선언이 아닙니다.

> 처분기관이 지적한 기술적 위험요인을 법률상 다툼과 별개로 구조적으로 통제하겠습니다. 재직자·협력사·탈취 계정이 정상 권한을 사용하더라도 고위험 행위는 지속 검증하겠습니다. 하나의 통제가 실패해도 다음 통제에서 공격을 탐지·차단하고, 모든 조치의 이행 여부와 운영 결과를 원본 로그·재현시험·독립검증으로 증명하겠습니다.

핵심은 사고가 없을 것이라는 희망이 아니라, **동일한 실패가 반복되지 않도록 시스템으로 강제하겠다는 약속**입니다.

# 3. 지적사항과 미래 위협에 대한 직접 개선안

개선조치는 처분기관이 지적한 위험요인 및 현실적인 내부자·AI 위협과 일대일로 대응해야 합니다.

| 지적사항·위협 | 구축할 통제 | 법원·감사 증거 |
|---|---|---|
| 인증용 서명키의 평문 노출 가능성 | 운영키를 HSM·KMS에서 생성·보관하고 사람의 평문 열람·추출 경로 제거 | 키 전수목록, 구 키 폐기·거부시험, HSM·KMS 정책과 접근 로그 |
| 서명키 접근권한 관리 미흡 | 개인별 최소권한, MFA, JIT, 업무 티켓 연계, 이중 승인, 직무 분리 | IAM·PAM 변경이력, 승인 원장, 특권 세션 기록, 정기 재승인 결과 |
| 재직자·협력사·탈취 계정의 오남용 | 사람 계정의 직접 서명 호출 제한, 워크로드 신원 검증, 반출 징후와 후속 행위 상관분석 | 키 접근·파일·EDR·DLP·외부 통신·토큰·API 로그의 통합 타임라인 |
| 퇴직·전보 후 권한·키 미정비 | 인사 이벤트와 계정정지·권한회수·토큰폐기·키교체를 하나의 완료조건으로 연결 | HR-IAM-PAM-KMS 워크플로, 처리시간, 누락 경보와 대사 결과 |
| 위조 토큰 장기 미탐지 | 정상 토큰 발급원장과 실제 사용기록 대조, 미발급·폐기키·비정상 재사용 차단 | 토큰 지문·`jti`·`kid`, 발급·사용 상관로그, 폐기·차단 이력 |
| 비정상 개인정보 조회 탐지 부족 | 계정·토큰·세션·단말·고유 정보주체 수·응답량·시간대를 복합 분석 | WAF·API·DB·EDR 원본 로그, 탐지정책 버전, 경보·차단·롤백 기록 |
| 미토스급 AI가 상징하는 고속·변형·다단계 공격 | 요청·응답·서버·계정·토큰·데이터 흐름을 PLURA-XDR AI가 상관분석하고 위험도를 설명 | 원본 로그 링크, 모델·정책 버전, 판단근거, 자동·승인 대응 이력과 독립 시험 |

## 적용 원칙

HSM·KMS는 키를 보호하고, IAM·PAM은 사람과 권한을 통제합니다. PLURA-XDR AI는 이들 시스템과 인증·API·WAF·EDR·데이터 로그를 연결해 **각 통제가 실제로 작동했는지**, 그리고 실패가 다음 공격 단계로 이어졌는지를 지속 감시합니다.

PLURA-XDR AI가 키 관리시스템이나 권한관리시스템을 대체한다고 주장하지 않습니다. 기존 보안투자를 최대한 활용하면서, 분리된 이벤트를 하나의 공격 흐름과 법적 증거로 연결하는 역할에 초점을 둡니다.

# 4. PLURA-XDR AI 기반 7중 방어체계

## 제1방어선 — 키 보호

HSM·KMS, 평문 키 제거, 비밀정보 전수점검, 키 원문 추출 금지로 최초 위험을 줄입니다.

## 제2방어선 — 재직자·협력사 내부위협 통제

사람의 키 원문 접근을 차단하고, 고위험 키 작업에 JIT 권한·업무 티켓·이중 승인·직무 분리·특권 세션 기록을 적용합니다.

## 제3방어선 — 사람·권한·키의 생명주기 통제

입사·직무변경·전보·휴직·퇴직 시 계정·권한·토큰·키를 일관되게 변경하고, 누락을 자동 경보합니다.

## 제4방어선 — 토큰의 출처 검증

정상 발급원장과 실제 사용기록을 대조하여 암호학적으로 유효해 보이는 위조·오용 토큰도 식별합니다.

## 제5방어선 — PLURA-XDR AI 기반 신종·다단계 공격 탐지

웹 요청·응답, 서버 명령, 계정·토큰, 개인정보 조회와 외부 통신을 연결하여 변형·제로데이 의심·경보 홍수 공격의 흐름을 판단합니다.

## 제6방어선 — 데이터 접근행위 탐지

민감 API의 순차·대량·저속·분산 조회, 고유 정보주체 수 증가, 응답량 급증과 자동화 패턴을 분석합니다.

## 제7방어선 — 안전한 차단과 증거 보전

위험도에 따라 요청 제한, 추가인증, 토큰·세션 폐기, 계정·단말 격리, 키 긴급 교체를 수행하고 원본증거를 보존합니다.

### AI 자동대응의 안전등급

| 등급 | 대표 조치 | 실행 원칙 |
|---|---|---|
| 관찰 | 추가 로깅, 증거수집, 위험도 재평가 | 자동 수행, 판단근거 기록 |
| 저영향 통제 | 속도 제한, 추가인증, 특정 요청 차단 | 사전 승인 정책에 따라 자동 수행 |
| 가역적 격리 | 토큰 폐기, 세션 종료, 단일 IP·단말 차단 | 고신뢰 사건에 한해 자동 또는 신속 승인, 즉시 롤백 가능 |
| 고영향·비상조치 | 계정·서버 격리, 중요 권한 회수, 키 긴급 교체 | 원칙적으로 사람·이중 승인, 복구계획과 사후 독립검토 적용 |

> **퇴직자든 재직자든 외부 공격자든 AI 에이전트든, 누구 하나 또는 하나의 키가 전체 사고를 완성할 수 없도록 만듭니다.**

# 5. PLURA-XDR AI 운영·증거화 모델

본 사업은 개별 경보를 많이 생성하는 데 목적이 있지 않습니다. 공격의 처음부터 끝까지를 하나의 사건번호로 연결하고, 판단과 대응의 근거를 재현할 수 있게 만드는 것이 목적입니다.

## 1) 온전한 관찰

다음 로그를 시간 동기화하여 수집합니다.

- HSM·KMS의 키 생성·사용·정책변경·폐기 기록
- IAM·PAM·인사시스템의 계정·권한·승인·인사변동 기록
- 인증서버의 토큰 발급원장과 세션 정보
- WAF·API 게이트웨이의 요청·응답 메타데이터와 민감 API 호출
- EDR·서버의 프로세스·CommandLine·파일·설정 변경
- 데이터베이스·데이터 플랫폼의 개인정보 접근과 반환 범위
- DLP·프록시·네트워크의 외부 반출 징후와 통신기록

## 2) AI 상관분석과 설명

PLURA-XDR AI는 각 시스템의 경보를 따로 보지 않고 다음 질문에 답하도록 구성합니다.

> 어떤 키·계정·단말이 사용되었는가?  
> 정상 발급된 토큰인가?  
> 승인된 업무와 실제 행위가 일치하는가?  
> 몇 명의 개인정보를 어떤 속도와 방식으로 조회했는가?  
> 차단하지 않을 경우 예상 영향은 무엇인가?

중요 판단에는 사용된 원본 로그, 정상 기준에서 벗어난 요소, 공격 진행단계, 권고 대응과 적용된 룰·모델 버전을 함께 남깁니다.

## 3) 대응과 복구

확실한 악성행위는 신속히 차단하고, 서비스 영향이 큰 조치는 사람의 승인을 거칩니다. 자동조치마다 최대 적용시간, 승인자, 롤백 책임자와 정상 서비스 확인절차를 지정합니다.

## 4) 증거 보전과 개인정보 최소화

원본 로그와 분석·요약자료를 구분해 보존하고, 해시값·시간동기화·관리자 접근기록을 남깁니다. 개인정보 원문은 필요 최소한으로 수집하며 마스킹·가명처리·열람권한 분리를 적용합니다.

> 대시보드 화면은 설명자료입니다. 법원에서 핵심이 되는 것은 원본 로그, 수집 경로, 해시값, 시간기준, 설정 버전과 증거 인수인계 기록입니다.

## 시험 원칙

1. 실제 공격용 AI는 운영망에 연결하지 않고 독립 레드팀이 통제된 시나리오를 수행합니다.

3. 보완 후 동일 조건으로 재시험하여 개선 전·후 차이를 남깁니다.  
4. 사고 당시 로그와 사후 재현시험 로그를 명확히 구분합니다.  
5. 시험 결과는 대시보드 캡처와 함께 원본 로그·해시·설정 버전·승인·롤백 기록으로 제출합니다.

- **완료성:** 계획이 아니라 핵심 운영환경에 적용되고 예외·잔여위험이 명시될 것
- **실효성:** 재현시험에서 탐지·판단·차단·복구·증거보전이 실제 작동할 것
- **지속성:** 월별 성능검토, 정기 훈련, 독립감사와 경영진 보고가 계속될 것

# 11. 국내 산업 공공성·글로벌 성공사례 및 실행 요청

본 사업은 규제 대응에 그치지 않고, 실제 구축·운영·독립검증이 완료된 뒤 그 교훈을 산업 전체의 예방 지식으로 전환할 수 있습니다.

## 국내 산업에 공유할 공공재

고객 개인정보·영업비밀·구체적 취약점·탐지 우회정보를 제거한 뒤 다음 내용을 익명화하여 공유합니다.

- 인증키·토큰·내부자 위협·AI 공격을 연결하는 참조 아키텍처
- HSM·KMS, IAM·PAM과 PLURA-XDR AI의 연계 원칙
- 변형·다단계·저속 분산·경보 홍수 공격의 안전한 시험 시나리오
- 탐지·차단시간, 로그 완전성, 오탐·미탐, 서비스 영향과 롤백 공통지표
- CEO·CISO·CPO·법무·개발·운영조직이 함께 사용할 대응 체크리스트
- 중견·중소기업도 단계적으로 적용할 수 있는 최소 통제와 실행 순서

> **한 기업이 겪은 사고의 비용을 산업 전체의 예방 지식으로 전환하는 것이 본 사업의 가장 큰 공공성입니다.**

## 대한민국 보안기술의 글로벌 성공사례

국내 제품이라는 이유만으로 선택하거나 감경을 주장하지 않습니다. 동일 조건의 PoV와 독립검증을 통해 실효성·증거력·서비스 안정성을 확인한 뒤, 검증된 성과와 한계만 한글·영문 백서로 발간합니다. 미국 등 해외 고객·파트너·감사기관·보안 커뮤니티에는 대규모 플랫폼 환경에서 검증된 한국형 AI 사이버 방어와 증거화 모델로 전달합니다.

## 경영진 실행 요청

1. 90일 통합 방어·증거화 사업과 전사 공동 태스크포스 착수  
2. 법무·CPO·CISO·개발·운영·인사·감사·회계조직의 공동 증거보전 명령  
3. 운영용 인증키의 HSM·KMS 비추출 구조 전환과 특권권한 전수 정비  
4. PLURA-XDR AI 로그 연계, 재현시험, 독립감사와 법원 제출용 증거패키지 작성  
5. 91~180일 전사 확대계획 및 국내 사례공유·해외 확산의 조건부 승인

> **쿠팡은 사고가 없었던 기업으로 돌아갈 수는 없습니다. 그러나 처분기관이 지적한 위험요인을 가장 엄격하게 통제하고, 그 운영효과를 원본증거로 입증하며, 그 교훈을 산업과 나누어 AI 시대의 새로운 보안 기준을 만든 기업은 될 수 있습니다.**
>
> **그 변화가 법원에서 인정되기 위해서는 의지나 홍보가 아니라, 처분사유별 주장·원본증거·재현시험·독립검증이 일치하는 충실한 준비서면이 필요합니다.**
