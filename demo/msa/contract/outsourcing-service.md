# 소프트웨어 개발 계획서

## 쿠버네티스 환경에서 악성코드 분석 MSA 아키텍처 구축

---

## 1. 사업 개요

### 1.1 사업 배경

최근 사이버 공격은 단순한 악성코드 유포 단계를 넘어,
**공급망 공격, 자동화된 침투, 장기 잠복형 공격(APT)** 등 고도화된 형태로 진화하고 있습니다.
특히 다음과 같은 변화가 두드러지고 있습니다.

* 컨테이너·클라우드 환경을 직접 공격 대상으로 삼는 악성코드 증가
* 단일 악성코드가 아닌 **다단계 공격 체인** 형태 확산
* 공격자가 인프라 환경(Kubernetes, Docker)을 인지한 상태에서 공격 수행
* 악성코드 분석 대상 파일 수의 폭증

기존의 **단일 서버 기반 악성코드 분석 시스템**은 다음과 같은 구조적 한계를 가집니다.

* 분석 요청 증가 시 처리 지연 발생
* 분석 엔진 추가·변경 시 전체 시스템 영향
* 장애 발생 시 전체 서비스 중단 위험
* 테넌트 분리 및 리소스 격리가 어려움

이에 따라 본 사업은 **Kubernetes 기반 MSA 아키텍처**를 적용하여
**확장성, 안정성, 자동화, 운영 효율성**을 동시에 확보하는
차세대 악성코드 분석 플랫폼 구축을 목표로 합니다.

---

### 1.2 사업 목적

본 개발의 목적은 다음과 같습니다.

1. 악성코드 분석 기능을 **마이크로서비스 단위로 분리**
2. Kubernetes 기반의 **자동 확장 및 자원 격리 구조 구현**
3. 정적·동적 분석 파이프라인의 **유연한 확장 구조 확보**
4. 웹 기반 분석 요청·결과 조회 체계 구축
5. 향후 AI 기반 분석 및 외부 보안 시스템 연계를 고려한 기반 마련

---

### 1.3 사업 범위

본 사업은 다음 범위를 포함합니다.

* 시스템 아키텍처 설계
* Kubernetes 기반 인프라 구성
* 악성코드 분석 마이크로서비스 개발
* 웹 기반 관리 UI 개발
* 테스트, 문서화 및 인수 지원

---

## 2. 전체 시스템 구성 및 아키텍처

### 2.1 아키텍처 설계 방향

본 시스템은 **MSA 원칙**에 따라 다음 기준을 중심으로 설계합니다.

* 서비스 간 **강결합 제거**
* 독립 배포 및 독립 확장 가능 구조
* 장애 발생 시 영향 최소화
* 분석 기능 추가 시 기존 서비스 영향 최소화

---

### 2.2 전체 아키텍처 구성 요소

#### 주요 구성 요소

1. API Gateway
2. Malware Ingest Service
3. Static Analysis Service
4. Dynamic Analysis Service
5. Metadata / IOC Service
6. Result Aggregation Service
7. Web Frontend
8. Monitoring & Logging Stack

각 서비스는 **Docker 컨테이너**로 패키징되며
Kubernetes 상에서 **Pod 단위로 배포·운영**됩니다.

---

### 2.3 Kubernetes 적용 구조

* Namespace 기반 서비스 분리
* 분석 서비스별 독립 스케일링
* 분석 컨테이너 자동 생성/삭제
* 장애 Pod 자동 재기동
* 리소스 제한(CPU/Memory) 설정
