# 필상 시스템 — Day 54 CVD/PSIRT 운영(제보 접수·버그바운티·공지·패치) 표준

1. **날짜**

* 2024-10-25

2. **목표**

* 외부 연구자·고객·내부 제보를 아우르는 **CVD(Coordinated Vulnerability Disclosure)** 체계 확립
* **PSIRT**(Product Security Incident Response Team) 운영·버그바운티·보안 공지(Advisory/VEX) 발행을 **48h 내 초동 대응, 7d 내 중대 취약점 패치** 원칙으로 표준화

---

3. **내용**

### A. 범위 & 역할

* **PSIRT**: 총괄·분류·조정·고지(Advisory/VEX)·패치 오케스트레이션
* **제품팀/플랫폼팀**: 영향 분석·수정·테스트·배포
* **CSM/지원**: 고객 커뮤니케이션·완화 가이드 배포
* **법무/컴플라이언스**: Safe Harbor·라이선스·규제 보고 검토

### B. 접수 채널 & Safe Harbor

* **접수**: `security.txt`(/.well-known/security.txt), PGP 공개키, 전용 폼(익명 허용), 이메일 `security@`
* **Safe Harbor**: 선의의 테스트(비파괴·데이터 유출/서비스 중단 금지)·선공개 금지(협조기간 내)·보복 금지 명시
* **금지행위**: 사회공학, 랜섬/협박, 고객 데이터 접근·추출

### C. 분류(SLA) & 우선순위

| 심각도          | 기준(예시)             |   1차 응답 |   임시 완화 |    패치/종결 |
| ------------ | ------------------ | ------: | ------: | -------: |
| **Critical** | 인증우회/원격코드실행/주권 위반  | **24h** | **48h** |  **≤7d** |
| **High**     | 권한상승/대규모 정보노출 가능   | **48h** |  **4d** | **≤14d** |
| **Medium**   | 제한적 영향/사용자 상호작용 필요 |  **4d** |     필요시 | **≤30d** |
| **Low**      | 모범사례 위반/정보노출 최소    |  **7d** |      선택 | **≤90d** |

> 내부 기준(운영 정책 Day 48/52와 정합): **KEV/EPSS High↑**는 자동 Critical 재분류 가능.

### D. 보상(버그바운티) 정책(요약)

* **스코프**: 콘솔·공개 API·엣지 게이트웨이·에이전트 업데이트 채널(스테이징 제외)
* **보상 등급(USD 또는 KRW 등가)**

  * Critical: 3,000–10,000
  * High: 1,000–3,000
  * Medium: 300–1,000
  * Low: Hall of Fame + 스웩
* **중복/품질**: 최초 제보 우선, 재현 절차·PoC·영향 증빙 포함 시 상향

### E. 처리 플로우(엔드투엔드)

1. **수신/접수** → 티켓 생성(PSIRT 큐)·PGP 확인·재현 ID 할당
2. **검증/재현** → 영향 범위(서비스/리전/테넌트)·주권 여부·KEV/EPSS 매핑
3. **임시 완화** → WAF 룰/레이트리밋/플래그/정책(OPA) 적용·모니터링 강화
4. **패치 오케스트레이션** → RFC/CAB·테스트(회귀/성능)·KR→JP 순 카나리 배포
5. **공지/고객 안내** → **보안 공지(Advisory)** + **VEX**(Not Affected/Fixed/Under Investigation) 발행
6. **보상/종결** → 연구자 공로 표기(Hall of Fame)·보상/감사 메일·티켓 종결·지식베이스 반영

### F. 보안 공지(Advisory) & VEX 포맷

* **Advisory 필수 항목**: 식별자(PSIRT-YYYY-###), 영향 버전/컴포넌트, CWE/CVSS/EPSS, 임시 완화, 패치/업데이트 방법, 주권 영향, 타임라인
* **VEX**: `status ∈ {affected, not_affected, fixed, under_investigation}`, 근거(코드 경로/설정), 대체 완화
* **배포**: 웹·RSS·서명 PDF, API(`GET /v1/advisories`, `GET /v1/vex?cve=CVE-…`) 제공

### G. 자동화 & 인터페이스

* `POST /v1/psirt/report` — 제보 수신(파일/PGP 첨부)
* `GET /v1/psirt/case/{id}` — 진행상태/타임라인(연구자 조회 토큰)
* `POST /v1/psirt/triage { id, severity, kev, epss }` — 분류/재현 결과 등록
* `POST /v1/psirt/mitigate { id, rule_id|flag|policy }` — 임시 완화 적용
* `POST /v1/psirt/advisory` / `POST /v1/psirt/vex` — 공지/VEX 발행
* `POST /v1/psirt/bounty/payout { id, tier }` — 보상 집행(법무/재무 승인 연동)

.  


### H. 거버넌스·가드레일

* **선공개 금지 기간**: 패치 배포 + 30일 또는 합의된 일정까지
* **데이터 주권**: 재현·증거는 리전 내 저장, 외부 공유는 **서명 URL(10분)**
* **법적 준수**: 국제 제재·수출통제(암호기술)·개인정보 규정 검토
* **감사**: 모든 케이스 `security.audit.v1` 해시체인 고정, 릴리스 노트/Advisory와 링크

### I. SLO/지표

* **초동 응답**: Critical 24h/High 48h 준수율 ≥ **98%**
* **패치 리드타임**: Critical ≤ 7d, High ≤ 14d 준수율 ≥ **95%**
* **중복률**: 중복 제보 비율 ≤ **20%**(스코프·가이드 품질로 감소)
* **연구자 만족도**: 피드백 점수 ≥ **4.5/5.0**

### J. 운영 체크리스트

* [ ] `/.well-known/security.txt`·PGP 키 최신화
* [ ] 재현 템플릿(CWE별)·PoC 샌드박스 준비
* [ ] KEV/EPSS 자동 매핑·위험 점수화(Day 48 규칙 연계)
* [ ] Advisory/VEX 템플릿 i18n(ko/ja/en)·서명·워터마크
* [ ] Hall of Fame 페이지 월간 갱신·보상 집행 SLA 준수

4. **예시 설명 (PSIRT-2024-017 — 인증 우회, 6일 내 종결)**

1) 연구자 제보: 콘솔 특정 플로우에서 **JWT 검증 우회** 가능성(PoC 포함). PSIRT가 **Critical** 분류(24h 내 응답).
2) 재현/분석: 조건부 헤더 파싱 취약·CWE-345 매핑, 영향 버전 2.8.0–2.8.2(KR/JP).
3) 임시 완화: 게이트웨이 **헤더 정규화** 플래그 적용, 관련 경로 레이트리밋 강화.
4) 패치: 고정-검증 로직 수정 → KR-stg 카나리 → KR-prod/JP 순 배포, SLO/회귀 통과.
5) 공지: **Advisory**(CVSS 9.1, EPSS 0.79)·**VEX**(fixed) 동시 발행, 고객 가이드/업데이트 스크립트 제공.
6) 보상·종결: 연구자 **High Tier 상향(재현성/영향 증빙 우수)**, Hall of Fame 등재. 전 과정 **감사 해시체인** 고정.
   → 결과: **CVD·PSIRT·버그바운티**가 유기적으로 작동해 **48h 초동, 6일 내 패치 완료**를 달성하고, 고객/연구자 신뢰를 강화.
